## 5. Top gaps preventing 9.5+ on the 3 sub-9.5 criteria

### G13 (Governance, 9.45) -Deploy.s.sol leaves cross-singleton wiring incomplete
`Deploy.s.sol:77-127`: deploys Distributor (line 85) and Bridge proxy (line 116) but **never calls `Treasury.setDistributor(...)` or `Treasury.setBridge(...)`** (Treasury wiring fns @ `Treasury.sol:240-246`). Post-deploy state: `treasury.distributor() == address(0)`, `treasury.bridge() == address(0)`. `Treasury.mintEmissionsToDistributor` will revert until manual post-deploy tx. PoolFactory ownership also left at deployer (line 90) -no transfer to AC owner / multisig. DeployScript.t.sol does not detect this ∵ it asserts `assertTrue(a.distributor != 0)` but not `treasury.distributor() == a.distributor`.
**Fix:** add `setDistributor` + `setBridge` calls in Deploy.s.sol before broadcast end; assert wiring in DeployScript.t.sol.

### G14 (Governance, 9.45) -Treasury UUPS upgrade timelock has no unit test
`Treasury.sol:319-348`: `requestUpgrade` / `executeUpgrade` / `cancelUpgrade` + `_authorizeUpgrade` (line 348). Bridge mirrors pattern (`Bridge.sol:371`, `Router.sol:330-356`). **Zero tests** assert the timelock is enforced (`grep -rn "executeUpgrade" test/` → 0 hits). A buggy `TL.validate(pendingUpgradeOp)` swap would silently disable timelock with no detection.
**Fix:** `test/unit/TreasuryUpgrade.t.sol` covering request → fast-forward → execute happy path + early-execute revert + cancel path.

### G15 (Conciseness, 9.45) -Deploy.s.sol contains 35-line embedded comment + manual JSON concat
`Deploy.s.sol:91-104`: 14-line block-comment dictating the GovToken ↔ Treasury bootstrap reasoning. Useful for archaeology but belongs in ADR. JSON construction `_logAndPersist:144-161` is 17 lines of `string.concat`; `vm.serializeJson` (forge-std) would be 5 lines. Conciseness reviewer marks down.
**Fix:** move bootstrap rationale to ADR-002; replace string.concat block with `vm.serializeAddress` chain.

### G16 (Cross-repo, 9.45) -sdk has no ABI-freshness test
sdk barrel is complete but `sdk/test/` has no test like `forge inspect Pool abi == sdk/src/abis/Pool.json`. Contract change without ABI regen is silently accepted at sdk type-check time. Cross-repo reviewer (C, D) cannot lift past 9.4-9.5 without a CI-level freshness gate.
**Fix:** add `sdk/test/abi-freshness.test.ts` running `forge inspect <c> abi` for each ABI in barrel and diffing.

### G17 (Cross-repo, minor) -G12 fix unused; alm Vault async-redeem queue unrepresented in sdk
- `BridgeableERC20.supportsInterface` made `virtual` but `GovToken` does not override → fix is dead until consumer arrives. Not harmful but signals an unfinished refactor.
- alm `Vault.sol:236-287` exposes ERC-7540 async-redeem (`requestRedeem`, cohort blend by SHARES @ line 285). sdk barrel @ `dex/sdk` does not export Vault ABI (it's an alm contract; arguably out-of-scope, but cross-repo reviewer flags the asymmetry: dex tokens shipped, alm vault not).
**Fix:** decide: either (a) add alm Vault ABI to a separate `@btr-protocol/sdk-alm` or (b) ADR note declaring alm sdk separation.
