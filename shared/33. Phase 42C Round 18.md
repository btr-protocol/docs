### A3 -economic / MEV

#### A3.1 -Donate inflation attack on liquidityIndex
- `Pool.donate` rebalances `liquidityIndex` by `idx * (liabBefore + amt) / liabBefore`. Edge: `liabBefore == 0 ⇒ liquidityIndex unchanged` (early-return correct). Edge: `liabBefore == 1` + huge donate → uint64 truncation of liquidityIndex (F-A1-R14-2 documented; mitigated by deployer-seed-deposit pattern). Already documented INFO.

#### A3.2 -Sandwich on swap via internal oracle update
- `_oracle(q)` (Pool.sol:503-518) pushes hop prices to InternalOracle AFTER swap settles. Attacker post-trade flushes oracle = pushed-by-design. Subsequent ops within same block see updated oracle. Sandwich window = 1 block. Mitigated by `lastUpdate` guards in InternalOracle. Re-walked -no new finding.

#### A3.3 -Bridge inbound griefing
- Attacker sends LZ message with `amount` that exceeds inbound limit → falls to `_queueFailedMessage(... FC_RATE_LIMIT)`. Owner must `recoverFailedMessage`. Spam vector: many small failed msgs blow up `failedMessages` mapping = unbounded gas cost on owner ops. Owner responds via `pauseToken`. Acceptable trade-off.

#### A3.4 -Distributor merkle-replay across campaigns
- Leaf includes `campaignId` (Distributor.sol:244, R14 fix). Replay closed.

#### A3.5 -Stake-then-donate flash farm
- `donate` rebalances index → sLP holders gain on unstake. Attacker `stakeLP(small) → donate(large) → wait stakeLockDuration → unstakeLP`. Lock-duration ≥ 0 (default). If owner sets `stakeLockDuration = 0`, attack is atomic-block but still requires existing LP balance (donate must come from somewhere). Net: attacker pays into `donate` what they extract via index growth × their share. Zero-sum modulo other LPs' share. **Not exploitable** -donate appreciation is proportional to existing LP balance %; donating >> own share leaks value to other LPs.

## 3. New surfaces probed (not in R17 §4)

### 3.1 -`Pool.batchSwap` w/ duplicate NATIVE inputs
**Probe**: user calls `batchSwap` with input array containing 2+ entries each resolving to wnative (passed as C.NATIVE via `tk == $.wnative` branch in `_pull(tk == $.wnative ? C.NATIVE : tk, ...)`). 

**Trace** (Pool.sol:380, Base.sol:139-152):
- Iter 1: `_pull(NATIVE, amt1)` → `IWETH9.deposit{value: amt1}()` → refund `msg.value - amt1` → contract ETH balance = 0.
- Iter 2: `_pull(NATIVE, amt2)` → check `msg.value >= amt2` passes (msg.value is read-only, still original); `IWETH9.deposit{value: amt2}()` reverts on insufficient contract ETH (after refund consumed the surplus).

**Severity**: LOW (DoS only -no fund loss). Affects niche use-case (single-tx batchSwap with multiple NATIVE inputs is unusual; user would normally aggregate native into one entry). Practical impact ≈ 0; documenting.

**Disposition**: F-A4-R18-1 LOW (DoS / UX). Not Phase 42C blocker -ergonomics, not security.

### 3.2 -`Admin.setAc` non-idempotent despite docstring
Comment line 184 says "Idempotent" but body line 186-188 unconditionally overwrites. By-design per Base.sol:44-49 (rollback path requires non-idempotent). Docstring misnomer only. F-A2-R18-1 INFO doc-only.

### 3.3 -`Admin.setFlowCooldown` unbounded
Owner can set cooldown to type(uint16).max = 65535s ≈ 18.2hr. Bricks deposits/staking but recoverable in one tx. Owner-trust. F-A2-R18-2 INFO.

### 3.4 -`Distributor` over-published-tree last-claimer DoS
If manager publishes a Merkle tree whose Σleaves > totalAllocated, claimants succeed FIFO until totalClaimed reaches cap; remaining claimants revert via line 117 guard. Manager must `updateCampaignRoot` with corrected tree. Operator concern. F-A1-R18-1 INFO.

### 3.5 -`Treasury.burnGovToken` does not refund emissions cap
Burning gov tokens reduces `totalSupply` but does NOT decrement `emissionsSchedule.claimed`. This is intentional (claimed is a ratchet -minted-ever count, not net-outstanding count). Means circulation can be < cap-consumption indefinitely. Acceptable. Not a finding -design clarification only.
