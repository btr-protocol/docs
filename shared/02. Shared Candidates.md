## Stays-in-place (rejected from peripheral)

- **dex**: `LibPricing`, `LibSpline`, `LibAnchorTree`, `LibPoseidon`, `LibOracle` (B64 decode), `BaseV1` (ERC-7201 plumbing), `InternalOracleV1`, `BridgeV1`, all `tokens/`, all `modules/`, ERC-7201 storage slots in `LibConstants`. Reason: dex AMM / zk / diamond mechanics non-portable.
- **alm**: `Vault`, `adapters/Dex`, `adapters/Swapper`. Reason: vault NAV semantics + LP-adapter polymorphism non-portable.

## 3-expert debate (compressed; doc 03 carries the rest)

### Round 1 — Solidity architect proposal

Pkg layout: `peripheral/src/{access,oracle,errors,libs,factory,math}/`. Pin pragma `^0.8.29` (alm anchor). Solady as git submodule. Foundry repo w/ remappings emitted as `peripheral/remappings.txt` consumed via `@btr/peripheral=lib/peripheral/src/`.

**Critique (Gov lead)**: A lone shared `AccessControl` cannot satisfy both repos w/o degradation. alm needs immutable cluster-wide kill-cap; dex needs per-module pause + bridge-operator role. Forcing alm's design on dex strips role granularity; forcing dex's role-bitmask on alm bloats bytecode.

**Critique (Gas specialist)**: Shared `Err` library = +0 bytecode (selectors compile-time inline) **iff** consumers `import {Err} from "@btr/peripheral/errors/Errors.sol"` and use `revert Err.X()`. Inheritance-based `IErrors` model (dex current) leaks ABI surface. EIP-170 budget: dex `AdminV1.sol` near limit; pulling in shared AC adapter logic risks overflow → must split via library calls.

**Revised consensus**: AC = layered. Base `AccessControl` (alm semantics) + optional `IRoleBitmask` extension contract used by dex modules. Errors via library import only (no inheritance). Bytecode-impacting helpers stay as `internal` libraries (free-standing); cross-contract calls reserved for AC + PriceProvider singletons.

### Round 2 — Gov-lead proposal

Two-tier governance. Tier-1 = peripheral `AccessControl` (owner/keeper/treasury/swapper/factory + adapter registry). Tier-2 = consumer-local roles (alm: `MAX_ADAPTERS=8` adapter-state; dex: `freezeAsset`, bridge ops, oracle pushers). Tier-2 reads `ac.owner()` for root-of-trust; defines its own role mappings.

**Critique (architect)**: Per-pool dex storage owner (`_s().owner`) currently set on init must migrate. Either (a) `_s().owner = address(ac)` and modifiers query `ac.owner()` (cold sload + extcodecall per check), or (b) cache via `setOwnerSnapshot()` callable by ac on rotation. Pick (b) for hot-path gas; rotation triggers a fan-out call to all pools via factory registry.

**Critique (gas)**: Per-pool fan-out N pools × 21k gas. For 100+ pools = unacceptable user-borne rotation cost. Counter-proposal: pool reads `ac.owner()` lazily, cache per-tx via transient slot (LibTransientCache extension). Cold first-check ~3k gas, subsequent calls ~100 gas. Rotation = O(1) on-chain.

**Revised consensus**: dex pool stores `address ac` (immutable post-init). Modifier reads `ac.owner()` w/ transient cache. Removes `_s().owner` from PoolStorage; pool factory pins ac at deploy.

### Round 3 — Gas-specialist proposal

Library-internal-fn doctrine: every shared lib exposes `internal` only → callers inline → no DELEGATECALL hop. Exception: `AccessControl` + `PriceProvider` (state) → external contracts. EIP-170 audit: alm contracts all <24kB w/ Solady; dex `AdminV1` (625 LOC) at risk → split into `AdminConfigV1` + `AdminTimelockV1` post-refactor.

**Critique (architect)**: Splitting `AdminV1` = ABI-breaking. Defer to module-rotation timelock window in PoolProxyV1. Document split as Phase 42C addendum.

**Critique (gov)**: `AdminV1` split touches timelocked ops. Migration must preserve pending ops or refuse to upgrade while pendingOps non-empty (already the case via `pendingData` mapping).

**Revised consensus**: Phase 42B = peripheral extraction + dex AC swap, no module splits. Phase 42C = `AdminV1` split if EIP-170 budget breached after refactor; gated on pending-ops drain.
