# 05. Style Guide -dex parity targets vs alm

Concrete deltas to bring dex contracts to alm-style strict / opinionated / minimal / aggressively-optimized parity. Reference: `~/Work/btr/alm/evm/src/`.

## 1. Custom errors only -no string reverts, no `IErrors` inheritance

**alm**: `revert Err.ZeroAddr();` (lib import, 4-byte selector, zero ABI surface).
**dex (now)**: `revert IErrors.ZeroValue();` (interface inheritance leaks selectors into every contract's ABI).
**target**: `import {Err} from "@btr/peripheral/errors/Errors.sol"; revert Err.X();` everywhere.

Specific errors (not in shared `Err`) stay declared inside the contract body, named tightly:
```solidity
// alm pattern
error AssetMismatch();
error AdapterCap();
```

## 2. Reentrancy via Solady transient guard

**alm**: `import {ReentrancyGuardTransient} from "solady/utils/ReentrancyGuardTransient.sol";` then `nonReentrant` modifier inherited.
**dex (now)**: inline assembly `tload`/`tstore` in `BaseV1`.
**target**: replace `BaseV1.nonReentrant` w/ Solady inheritance. Saves bytecode + uniformity.

## 3. Named selector constants for staticcall hops

**alm pattern**:
```solidity
bytes4 internal constant SEL_DECIMALS  = 0x313ce567;
bytes4 internal constant SEL_SLOT0     = 0x3850c7bd;
```
declared at top of contract, used in `abi.encodeWithSelector(SEL_X, ...)`. Improves audit-trace.

**target**: dex contracts using `staticcall` (e.g. external oracle reads) hoist all selectors as `bytes4 internal constant SEL_*`.

## 4. EIP-170 budget per contract -track in CI

**alm**: every contract <24kB; `Vault.sol` ≈ 22kB w/ Solady + Initializable.
**dex**: `AdminV1` near limit (625 LOC + module-imported libs).

**target**:
- Add `forge build --sizes` to CI; fail if any contract >23.5kB (0.5kB headroom).
- If `AdminV1` overflows post-refactor → split into `AdminConfigV1` + `AdminTimelockV1` (Phase 42C).
- Prefer `internal` libraries (inlined) over `public/external` (DELEGATECALL) for hot-path math.

## 5. Audit-trace stripping doctrine

alm comments are **dense, audit-grade**, every non-trivial branch documented w/ rationale. Pattern:

```solidity
/// @dev Cluster-wide kill budget; keepers can collectively kill at most KEEPER_KILL_DAILY_CAP
///      adapters per UTC day. Without this, 1 keeper × 8 adapters @ 1/h would freeze a vault for 8h.
uint8  public keeperKillsToday;
```

**target dex**: every storage slot, every modifier, every non-trivial revert documented w/ **invariant** + **attack vector mitigated**. Drop redundant `// FIX: H-01` / `// CRITICAL FIX (CRITICAL-10)` comments -replace with `@dev` tags pointing to audit findings index. Single `audit-findings.md` lookup table in repo.

## 6. Ternary-first style

alm pattern: prefer ternary w/ aligned columns over branched assignment.
```solidity
return f.dec < TARGET ? raw * (10 ** (TARGET - f.dec)) : raw / (10 ** (f.dec - TARGET));
```
**target**: dex modules replace 3+ line if/else value-set blocks with ternary where readability holds.

## 7. Idiomatic Solady usage

alm: `SafeTransferLib as Safe`, `FixedPointMathLib as Math`, `SafeCastLib as Cast`, `LibClone`, `Initializable`, `Ownable`, `ReentrancyGuardTransient`, `ERC20`.
**target dex**: drop OZ-style patterns where Solady equivalent exists. Already mostly aligned (uses `SafeTransferLib`, `Ownable`); enforce comprehensively.

Aliases (consistency):
- `import {SafeTransferLib as Safe}`
- `import {FixedPointMathLib as Math}`
- `import {SafeCastLib as Cast}`
- `import {LibClone as Clone}` (where used)

## 8. Slot-packing discipline

alm `Feed { address addr; uint32 heartbeat; uint8 dec; }` = 200 bits / 1 slot. Comment-annotated.
**target**: every `struct` in dex audited for packing; comment slot-bit math inline:
```solidity
/// @dev 160 + 32 + 8 = 200 bits → 1 slot per token.
```

## 9. Initializer pattern uniformity

alm: `solady/Initializable` + `_disableInitializers()` in constructor + `external initializer` on `init`.
**target**: dex pool/module `init` flows mirror this; explicit init flag (e.g. alm's `hwmInit`) over numeric-state inference.

## 10. Comment density

alm style ratio: ~1 comment line / 3 code lines for non-trivial logic. dex modules vary; AdminV1 over-comments fix history, BaseV1 dense.
**target**: consistent ratio across modules; remove fix-history comments (move to git-blame + audit-findings table).

## 11. No NatSpec gold-plating

alm uses `@title`, `@notice`, `@dev` only. No `@param` / `@return` boilerplate when fn name + types are self-explaining. dex over-uses `@param` -strip when redundant.

## 12. Constant scoping

`internal constant` for compile-time inlined values not exposed to ABI. `public constant` only when ABI exposure is intentional (e.g. `MIN_EXIT_PIP` for off-chain consumers). dex frequently `public constant`; audit each for ABI necessity.

## 13. Pragma alignment

All shared peripheral + alm + dex post-refactor: `^0.8.29`. dex currently `^0.8.33` -downgrade enforced.

## 14. Test parity

alm: 263 tests, fork + unit + invariant. dex: similar coverage targets per-module post-refactor. Foundry `invariant` suite required for AC + PriceProvider on peripheral.

## 15. File header

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.29;

import {…} from "solady/…";
import {…} from "@btr/peripheral/…";
import {…} from "../local/…";
```
Import order: external (Solady) → peripheral → local. One blank line between groups.
