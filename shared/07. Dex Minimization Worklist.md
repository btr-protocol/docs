Audit-only pass on `dex/contracts/src/`. ALM-bar comparison. No file edits. This document is the authoritative input for stage-C per-module batch agents.

### 4.3 AdminV1 split (Phase 42A risk #4 directive)
Per Phase 42A doctrine and user's explicit "over-engineered" complaint:
- Split: `AdminConfigV1.sol` (~200 LOC; instant ops) + `AdminTimelockV1.sol` (~250 LOC; queued ops).
- Modules registry in PoolProxyV1 already supports per-selector dispatch -the split is transparent.
- Replace 8 timelock triplets with a single `_queue<T>(bytes32 id, uint48 delay, T data) → _execute<T>(bytes32 id)` template. Selector→ID map stored once.

### 4.4 Replace `LibTimelock` with ALM-style `Queued` struct
- ALM uses a 2-field `struct Queued { address addr; uint64 eta; }` (`AccessControl.sol:19`). LibTimelock packs `(executeAt:48, grace:48)` into uint96.
- BTR's grace-period adds complexity ALM doesn't need (cluster-wide, not per-vault).
- **Decision**: keep `LibTimelock` (grace is genuinely useful for module updates) but unify packed-uint96 → `struct Queued { uint48 eta; uint48 grace; address proposer; bytes32 dataHash; }` -adds proposer-binding (security improvement) and dataHash (replay protection).
- Net LOC neutral; security upgrade. **Defer to security audit phase.**
