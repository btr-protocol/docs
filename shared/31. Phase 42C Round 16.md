## 3. Four-auditor independent pass -targeted scope

### A1 -invariants / conservation

#### Staking.sol -stake/unstake invariants
- **`Pool.lpBalances` vs `Pool.lpStaked` decoupling**: `Staking.stakeLP` (line 102-108) increments `$.lpStaked[user][lpToken]` but does NOT decrement `$.lpBalances[user][t]`. The check at line 104 `available < alreadyStaked + amount` prevents stake-from-unbacked-balance, but Pool.withdraw (Pool.sol:124-126) only checks `$.lpBalances[msg.sender][fromTk] < lpAmount` -it does NOT consult `$.lpStaked`. **Invariant violation:** user with `lpBalances=100`, `lpStaked=100` (everything staked) can call `Pool.withdraw(100)` → succeeds, drains underlying reserves, leaves `lpBalances=0` while `lpStaked=100`. Subsequent `unstakeLP` would revert at SoladyERC20 `_burn` (sLP balance non-zero in token storage but no underlying backing) -and the user has already extracted real value.
  - **F-A1-R16-1 (HIGH, NEW)**: Pool.withdraw lacks `lpStaked` lock-check. Stake then withdraw drains the pool without burning sLP. Severity HIGH because: (a) bypasses staking lock entirely; (b) economic value extractable; (c) Staking module was the un-audited surface flagged for R16 -exactly the predicted defect class.
  - Fix: in `Pool.withdraw`, require `$.lpBalances[user][fromTk] - $.lpStaked[user][fromTk] >= lpAmount`. (Symmetric guard needed in `swapLiability` -Pool.sol:201.)

- **Staking module uninitialized `sGovToken`**: `IPool.PoolStorage.sGovToken` is declared (`IPool.sol:148`) but **never assigned anywhere in `src/`** (verified via `grep -rn "sGovToken\s*="`). No setter, no initializer, no `Admin` path, no `PoolProxyFactory` wiring. `Staking.stakeGov` (line 36) calls `IMintable($.sGovToken).mint(...)` where `$.sGovToken == address(0)` → low-level call to addr 0 with non-empty calldata → revert (no code).
  - **F-A1-R16-2 (CRITICAL, NEW)**: `stakeGov` and `unstakeGov` are permanently DOA. The gov-staking surface advertised by `IStaking.stakeGov` is uncallable on any deployed pool. Severity CRITICAL because: (a) advertised feature non-functional; (b) no recovery path (no setter exists); (c) governance-surface (delegate, voting power) all hangs off this; (d) zero tests exercise it (verified via `grep -rn "stakeGov" test/` returns ZERO).

- **Staked tokens lack public `mint`/`burn`**: `StakedToken.sol` (lines 1-45) is abstract; neither `StakedGov.sol` nor `StakedLP.sol` defines public `mint(address,uint256)` or `burn(address,uint256)` selectors. Solady ERC20 only exposes `_mint`/`_burn` as `internal`. The override at `StakedToken._update` (line 40) is dead code -solady's `_mint`/`_burn` operate via inline assembly on `_BALANCE_SLOT_SEED` and **do NOT call `_update`** (verified at `.deps/solady/src/tokens/ERC20.sol:496-552`).
  - Consequence: `IMintable(sLP).mint(msg.sender, amount)` (Staking.sol:106) hits a non-existent selector → no fallback → revert. **`stakeLP` is also DOA** even after `updateStakingConfig` deploys an sLP.
  - **F-A1-R16-3 (CRITICAL, NEW)**: `stakeLP`/`unstakeLP` are permanently DOA on all configured LP tokens -`StakedLP` lacks the `IMintable` surface required by `Staking`. Severity CRITICAL: same reasoning as F-A1-R16-2; entire LP staking subsystem uncallable.
  - Fix: add `function mint(address to, uint256 amount) external { if (msg.sender != STAKING) revert Ownable.Unauthorized(); _mint(to, amount); }` (and `burn` symmetric) to `StakedToken.sol`.
