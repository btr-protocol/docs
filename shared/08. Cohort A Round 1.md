## 2. 3-expert debate

### Round 1

**A1 Architect -proposal.** Rename `dex/contracts/` → `dex/evm/` for symmetry w/ `alm/evm/` and to reserve `dex/svm/`, `dex/move/` slots. Bootstrap `peripheral/evm/` w/ same convention. Top tree: `~/Work/btr/{alm,dex,peripheral,sdk,swap,back,front,prime}/` × per-VM subdir. One `foundry.toml` template per VM family lives in `peripheral/evm/foundry.toml`; alm + dex inherit via `forge config` `--profile peripheral` or symlink-include not supported -so use a `foundry.base.toml` checked into peripheral and a thin local override. Remappings shared via `peripheral/remappings.txt` consumed as `lib/peripheral/remappings.txt` after submodule add.

**A2 Security -proposal.** ONE `AccessControl` shared by alm + dex is acceptable iff (a) owner = same multisig (BTR governance), (b) treasury = same sink, (c) keeper sets are role-scoped and disjoint per consumer (alm-keeper ≠ dex-keeper). Single `PriceProvider` is a blast-radius concern: an oracle outage halts both alm vaults AND dex pools simultaneously. Mitigation: keep `PriceProvider` shared but with per-consumer fallback registry -alm Vault retains worst-of(pool, oracle); dex retains its `InternalOracleV1` as primary, peripheral `PriceProvider` as Chainlink secondary only. Key rotation must be atomic (single tx replaces owner across alm + dex via AC) -feasible iff AC is the SSoT and consumers read `ac.owner()` rather than caching.

**A3 Conventions -proposal.** Drop `V<N>` suffix entirely. Rationale: (1) alm is V-free and is the style anchor (per doc 05); (2) versioning belongs in deploy registry + git tags, not in identifier; (3) Phase 42's mandate is consolidation -V1 exists only because no V2 was ever conceived; (4) audit traces (Phase 16) reference contract by deployment address, not by symbol name. Apply uniformly: `PoolProxyV1` → `PoolProxy`, `AdminV1` → `Admin`, `IRouterV1` → `IRouter`, etc. Tokens (`GovToken`, `StakedLP`) already V-free. Errors lib is `Err` (alm) -keep, drop dex `IErrors` interface name entirely (consistent with doc 05 §1).

### Critiques (round 1)

- **A1 critiques A2**: shared PP reduces audit surface but creates a single oracle owner -if PP is compromised, both surfaces drain. Counter-proposal: PP is a peripheral *library + interface*, deployed twice (alm-PP, dex-PP) with shared bytecode but distinct admin. Loses key-rotation atomicity.
- **A1 critiques A3**: dex SDK + frontends already pin `PoolProxyV1` ABI hash; mass-rename forces SDK regeneration. Mitigation: schedule rename pre-mainnet only; after mainnet freeze, V1→V2 must follow for any breaking change. ⇒ V<N> drop is safe NOW but future Vs are mandatory on breaking.
- **A2 critiques A1**: `dex/evm/` rename is cosmetic but touches every relative path in scripts, CI, deploy registry, sdk ABI exports -risk of broken pipelines mid-audit. Sequence: do rename in its own commit, run full CI before any other 42B work resumes.
- **A2 critiques A3**: dropping V1 from `BridgeV1` loses LayerZero-protocol semantics (LZ has v1 vs v2 endpoints -`BridgeV1` reads as "uses LZ v1" to LZ-literate auditors, but actually targets v2). Rename `BridgeV1` → `Bridge` AND verify no auditor confusion vector.
- **A3 critiques A1**: foundry profile sharing via `foundry.base.toml` + local override is unsupported by Foundry directly (no `extends`); workaround = `forge` reads only one `foundry.toml`. Real solution: keep per-repo `foundry.toml`, factor common values into a doc'd canonical block + lint-check via CI grep.
- **A3 critiques A2**: per-consumer fallback registry doubles AC complexity and re-fragments the SSoT. Cleaner: PP shared, but each consumer owns a local "trust-staleness" threshold; on PP staleness, consumer halts only its own ops, not global.

### Round 2 -convergence

- A1 + A3 agree: `dex/contracts/` → `dex/evm/` rename done as isolated commit pre-1d completion; foundry profile alignment via doc-spec, not extends.
- A2 + A3 agree: drop `V<N>` everywhere pre-mainnet; reinstate V<N> only on breaking upgrades post-deploy. `BridgeV1` → `Bridge` w/ NatSpec clarifying "LayerZero v2 endpoint".
- A1 + A2 agree: AC singleton shared; PP singleton shared but consumers retain local staleness gating + secondary feed registry. Keeper sets disjoint per consumer via role-scoped AC roles (`KEEPER_ALM`, `KEEPER_DEX`).
- All 3 agree: peripheral repo must own `foundry.base.toml` + `remappings.txt` as canonical reference; consumers copy + diff in CI.

## 4. Consensus -singleton sharing

**Decision (3/3): shared AC; shared PP w/ guarded blast radius.**

Topology:

```
peripheral/evm/src/
  access/AccessControl.sol     (singleton -owner = BTR multisig, treasury = single)
  access/IAccessControl.sol
  oracle/PriceProvider.sol     (singleton -Chainlink + sequencer guard)
  oracle/IPriceProvider.sol
  errors/Errors.sol            (Err lib -union of alm + dex selectors)
  ...

alm/evm/src/Vault.sol           → reads ac.owner(), ac.isKeeper(KEEPER_ALM, x)
                                → reads pp.price(t) w/ local staleness gate + worst-of(pool,pp)
dex/contracts/src/PoolProxy.sol → reads ac.owner(), ac.isKeeper(KEEPER_DEX, x)
                                → InternalOracle primary, pp secondary fallback
```

Risks vs benefits:

| Risk | Mitigation |
|---|---|
| AC compromise drains both | Multi-sig owner; 2-step ownership rotation already in alm AC; 7d/14d timelocks |
| PP outage halts both | Per-consumer staleness threshold; consumers degrade-to-halt locally, not globally |
| Governance entanglement | Roles scoped: `KEEPER_ALM`, `KEEPER_DEX`, `PAUSER_DEX`, `BRIDGE_OPERATOR`. Owner is the only shared role |
| Key rotation atomicity | SSoT: consumers MUST read `ac.owner()` live, no caching. Single tx rotates both surfaces |
| Audit blast radius | Peripheral L1 audit highest-density (smallest, most-trust-loaded) -already in doc 06 |

Benefits: ~330 LOC duplication eliminated; one audit lot for AC + PP; one upgrade path; one treasury reconciliation.

## 5. Consensus -V<N> policy

**Decision (3/3): drop `V<N>` from all symbols pre-mainnet. Reinstate ONLY on breaking ABI upgrade post-deploy.**

Rule:

- Pre-mainnet (now): `*V1.sol` → `*.sol` for all 33 dex files. Interfaces: `IRouterV1` → `IRouter`. Internals (modules): `AdminV1` → `Admin`. Public: `PoolProxyV1` → `PoolProxy`, `BridgeV1` → `Bridge` (NatSpec: "LayerZero v2").
- Post-mainnet breaking change: introduce `*V2.sol` ONLY when ABI/storage layout breaks. Non-breaking patches stay on `*.sol`.
- Versioning surface = git tag + deploy registry (`alm/evm/registry.json`, `dex/evm/registry.json`), not symbol.
- alm is the anchor -it is V-free and stays V-free.

Apply order: rename in same commit as `dex/contracts/` → `dex/evm/` rename, OR immediately after, before any new module work. SDK regen + ABI re-export in same commit.
