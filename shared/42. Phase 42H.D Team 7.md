| Spline + AnchorTree degenerate inputs | `Spline.sol` is monotone-cubic-Hermite 1e18 fixed-point; `Pricing._traverseSplineByVolume` traverses by volume slices -degenerate (zero-liquidity) profile would yield zero-width segments, but `_buildSplinePoints` filters by dispersion. No public attack surface beyond admin-set risk config. |
| PriceProvider sequencer-down | No L2 sequencer-aware grace logic in `ExternalOracle.sol`. Staleness is per-feed `ttl` (`updatedAt`+`ttl >= now`). On L2 sequencer outage, all feeds become stale → quotes revert via stale-data check in `PoolOracle._readOracle`. Conservative fail-closed posture. Acceptable for v1 (no soft gap raised, only noted). |
| Pool slot-0 layout if `PoolStorage` extended | `PoolStorage` is `IPool` struct used via `storage $` reference in libraries (`PoolAdmin`, `Pricing`, `PoolOracle`, `AnchorTree`). All library functions take `IPool.PoolStorage storage $` as 1st arg → libraries are layout-agnostic. Slot-0 invariant lives in `Pool.sol` itself; struct extension at TAIL is safe (Solidity layout rule). Sound but no compile-time guard exists -SOFT NIT only. |
| feeOverride > 0 + delta < 0 conservation | `_processSwap` with feeOverride: recomputes fees from `raw = out + protoFee + lpFee` → `q.amountOut = raw - fee`. Then `delta < 0` (rebate): `out += uint256(-delta)`. `_reconcile` then adjusts `a.reserves` by `actual - expected` where `expected = q.amountOut`. Reserves absorb the rebate symmetrically with the user payout. Conservation holds. `Pool.sol:546-580, 812-818` |
| UUPS upgrade flow under reentrancy | newImpl has zero callback opportunity (empty data). Pending-state cleared pre-call → cancelUpgrade reverts mid-flow. Transient flag is single-bit, set/cleared inline. No reentrancy vector. |
| Bridge cancelConfigChange / cancelSetPeer race | Each `cancel*` function is `onlyOwner` + reverts on no-pending. Cross-family isolation: id derivation includes `OpType` → cancelSetPeer cannot delete a ConfigUpdate slot. Independent test `test_cancelSetPeer_doesNotAffectConfigChange` exercises this directly. |
