## 4. Consensus -SDK 0.2.0 release notes

**Decision (3/3).** Bump `@btr-protocol/sdk` `0.1.0` → `0.2.0`. Single coordinated release, post-contract-rename.

### Breaking changes summary

1. **ABI exports renamed.** Existing `AIMM_ABI`, `POOL_ABI`, `DARKPOOL_ABI` unchanged in name; their underlying contract symbols (Solidity ABI `name` field) shed `V1` suffix. Type-level: no break. Selector-level: no break (selectors are computed from function signatures, not contract names). **No frontend code break for existing 3 ABIs.**
2. **New ABIs added** (the actual breaking surface): `BRIDGE_ABI`, `ROUTER_ABI`, `TREASURY_ABI`, `POOL_PROXY_ABI`, `POOL_PROXY_FACTORY_ABI`. These were absent in 0.1.0. Doc samples in `docs/dex/5.*` reference them; samples become runnable in 0.2.0.
3. **TSDoc updates.** `sdk/src/eth/abi.ts:62` comment `// Pool functions (from IPoolV1.sol)` → `// Pool functions (from IPool.sol)`. All `@module` headers stay.
4. **Re-export surface.** `sdk/src/abis/index.ts` adds 5 new `export * from './*.js'` lines. Frontend wildcard imports get the new ABIs automatically.

### Migration guide (consumer-facing)


## 5. Consensus -terminology canonicalization

**Decision (3/3): canonical = "Dex adapter".**

| Form | Status | Rationale |
|---|---|---|
| **Dex adapter** | ✓ canonical | Matches `alm/evm/src/adapters/Dex.sol`. Capital D = proper noun (the contract type). Lowercase "adapter" = role. |
| DEX adapter | ✗ | All-caps "DEX" reads as asset-class abbreviation (the BTR DEX product), conflates layers. |
| dex adapter | ✗ | Loses contract-name correspondence. |
| ALM adapter | ✗ | Wrong direction: ALM is the vault layer, adapters bridge OUT to dex venues. "ALM adapter" would mean an adapter TO alm, not FROM alm. |
| vault adapter | ✗ | Inverts semantics: vault uses adapters; vault is not the adapter. |
| Adapter (bare) | acceptable | Use only when context unambiguous. |

**Glossary disambiguation entry (NEW).**
> **Dex adapter** -concrete contract at `alm/evm/src/adapters/Dex.sol`. A multi-holder shim that wraps a single third-party concentrated-liquidity pool (Uniswap v3/v4, PancakeSwap V3/Infinity, Algebra, Ramses, Aerodrome Slipstream). Holders (Vaults, EOAs, multisigs) deposit principal; adapter manages LP positions on the underlying pool. NOT to be confused with **Dex** (product, `dex/evm/`) which is the BTR-native AIMM.

**Files needing rewrite to canonical form:** `docs/legal/{Terms of Service, Privacy Policy, Risk Disclaimer, Acknowledgement}.md`, `docs/Overview.md`, `docs/supply/{01,02,03,07,08,09}.md`, `docs/Glossary.md` (entry exists), `docs/dex/2. Governance/2.4. Liquid Staking.md`. ~30 occurrences total. Mostly mechanical sed (case + spacing).

**SDK type-name alignment.** Solidity `Dex.sol` → TS class (if generated): `Dex`. Adapter wrapper helper class (if shipped): `DexAdapter` (single token, conventional TS class). Doc body uses "Dex adapter" (two-token noun phrase). Both are valid in their context.
