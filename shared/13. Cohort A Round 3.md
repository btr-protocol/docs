## 2. Path α audit

### 2.1 Correctness

`dex/evm/src/modules/Base.sol:44-47`:

```solidity
function _owner() internal view returns (address) {
    address ac = _s().ac;
    return ac == address(0) ? _s().owner : IAccessControl(ac).owner();
}
```

`onlyOwner` modifier (`Base.sol:37-40`) routes through `_owner()`. `Admin.sol:20-23`
overrides identically. `Distributor.sol:144` re-routed. `Admin.setAc` (`Admin.sol:141-146`)
is `onlyOwner`-gated, idempotent, emits `IAdminConfig.AcUpdated`.

PathAlphaACTest (`dex/evm/test/PathAlphaACTest.t.sol`) covers:
- legacy path (ac=0) -`test_legacyPath_ownerEnforced` ✓
- singleton path with **atomic rotation across N=2 pools in 1 tx** -`test_singletonPath_atomicRotation` ✓
- gate + reversibility -`test_setAc_gatedAndReversible` ✓
- storage append-only invariant -`test_storageLayout_acAppended` ✓ (probes slot offsets, asserts `ac` lives strictly after `modules` mapping @ offset 13)

### 2.2 Storage layout integrity

`IPool.PoolStorage` (`dex/evm/src/interfaces/IPool.sol:128-157`) -`ac` appended
as **last field** at line 156. Pre-existing `modules` mapping @ offset 13 untouched
(verified live by the test suite which still dispatches through it). ERC-7201
namespaced; append-only invariant ✓.

### 2.3 Gas

`onlyOwner` cost when `ac != 0`:
- 1× cold SLOAD `_s().ac` (~2,100 gas first call, 100 warm)
- 1× cold STATICCALL to `IAccessControl(ac).owner()` (~2,600 cold + 100 warm SLOAD on AC side)
- Total: ~4,800 cold-path overhead vs legacy single SLOAD.
Legacy path (ac=0) short-circuits at the ternary -zero overhead beyond 1 SLOAD.
Acceptable for governance-tier fns (none on hot swap path).

### 2.4 Regressions found (⚠ 2 residual `_s().owner` reads)

`grep -n "_s().owner\|_owner()" dex/evm/src/modules/*.sol`:

| File:Line | Site | Severity | Note |
|---|---|---|---|
| `InternalOracle.sol:77` | `updateFeed` auth: `if (msg.sender != _s().owner && msg.sender != address(this))` | **MEDIUM** | Direct owner-driven `updateFeed` ignores singleton AC rotation. `executeAddAsset` self-call short-circuits via `address(this)` branch so test suite stays green, but external owner-pushed price updates remain bound to legacy `$.owner`. |

### 5.1 Concrete blockers

**Blocker count: 2** (one root cause).

1. **`InternalOracle.sol:77` still reads `_s().owner` directly.** Fix: replace
   `if (msg.sender != _s().owner && msg.sender != address(this))` with
   `if (msg.sender != _owner() && msg.sender != address(this))`. 1-line change.
   Lifts Logic 9.4 → 9.6, Governance 9.4 → 9.5+.

2. **`Pool.sol:402` `owner()` view returns `_s().owner`.** Fix: return
   `_owner()` instead so the view reflects live authoritative owner when AC is
   set. 1-line change. Lifts Governance another +0.1 (observability parity).

Both fixes are mechanical, ≤2 LOC total, no storage layout change, no new tests
required (existing PathAlphaACTest covers the rotation path; would benefit from
1 added assertion in `test_singletonPath_atomicRotation` that calls
`InternalOracle.updateFeed` directly under the rotated owner -recommended but
not blocking).
