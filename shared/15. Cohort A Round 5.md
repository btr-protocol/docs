## 2. `d287d26` audit

### 2.1 Correctness -`Treasury.salvage`

`dex/evm/src/Treasury.sol:271-280`:

```solidity
function salvage(address token, address to, uint256 amount) external onlyOwner {
    if (to == address(0) || amount == 0) revert Err.ZeroValue();
    if (token == C.NATIVE) {
        SafeTransferLib.safeTransferETH(to, amount);
    } else {
        if (token == address(0)) revert Err.InvalidInput();
        SafeTransferLib.safeTransfer(token, to, amount);
    }
    emit Salvaged(token, to, amount);
}
```

- Auth: Solady `Ownable.onlyOwner` modifier (Treasury inherits Solady `Ownable` at
  `Treasury.sol:19`). ✓
- Custom errors only (`Err.ZeroValue`, `Err.InvalidInput`) -no require-strings. ✓
- Solady `SafeTransferLib` for both ERC-20 (`safeTransfer`) and native
  (`safeTransferETH`). ✓
- NATIVE sentinel `C.NATIVE = 0xEeee...EEeE` (`LibConstants.sol:20`). ✓
- Zero-receiver and zero-amount both revert `Err.ZeroValue`. ✓
- `address(0)` ERC-20 token → `Err.InvalidInput` -explicit guard prevents accidental
  fall-through to a `safeTransfer(address(0), …)` call. ✓
- `Salvaged(token, to, amount)` event declared on `ITreasury.sol:29` and emitted. ✓
- `receive() external payable {}` (`Treasury.sol:342`) so ETH can land for sweep. ✓

### 2.3 Parity vs alm `Vault.salvage`

`alm/evm/src/Vault.sol:539-550`:

```solidity
function salvage(address token, uint256 amount, uint256 minOut, bytes calldata routerData)
    external onlyOwner nonReentrant
{
    if (token == _asset || token == address(this)) revert NotSalvageable();
    if (amount == 0) revert NothingToSalvage();
    address swp = ac.swapper();
    if (swp == address(0)) revert SwapperUnset();
    Safe.safeApprove(token, swp, amount);
    uint256 got = ISwapper(swp).swap(token, _asset, amount, minOut, routerData);
    Safe.safeApprove(token, swp, 0);
    emit Salvaged(token, amount, got);
}
```

**Shape ≈ but semantically distinct -by design:**

- alm Vault is asset-bearing for LPs ⇒ salvage MUST swap stuck token back into the
  underlying `_asset` (LP-protective) and forbid touching `_asset` itself.
- dex Treasury / Bridge hold heterogeneous protocol-owned funds (gov tokens, fees,
  bridged inventory) ⇒ salvage is a direct ERC-20 / native sweep with no swap step.
  Swap-route would be wrong here (no canonical underlying).

Common contract: **owner-only · custom errors · `Salvaged` event · Solady SafeTransferLib ·
single-step (timelock comes from owner = multisig)**. This matches the commit message
and the user's parity intent. ✓
