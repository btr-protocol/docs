## 5. Top gaps preventing 9.5+

### G8 (NEW) -PoolDecay + PoolAdmin libs have **zero direct unit tests**
`grep -rn "PoolDecay\|PoolAdmin" test/` → 0 hits. Both libs were extracted in 7e06d30 alongside PoolOracle, but **only PoolOracle got a dedicated test suite**. Coverage is exclusively transitive through Pool integration tests.
- `PoolDecay.calculateDecay` (`PoolDecay.sol:43-53`) has 4 boundary cases (dt=0, slope=0, coverage≥threshold, raw>maxDecay cap) -none tested directly. The `liabilities-reserves` cap branch (line 51) is silent under transitive coverage if no integration test crosses the deficit threshold.
- `PoolAdmin.validateProfileMemory` (`PoolAdmin.sol:16-39`) has 4 revert paths (weights[0]=0, sum≠200, knot non-monotonic, span≠100) -only one indirectly hit.
- `PoolAdmin.validateOracleConfig` try/catch reachability not unit-tested.
**Fix:** `test/unit/PoolDecay.t.sol` + `test/unit/PoolAdmin.t.sol` mirroring the harness pattern from `PoolOracle.t.sol:11`. Drops Test rigor from 9.28 → 9.55.

### G9 (NEW) -sdk ABI export missing GovToken + BridgeableERC20
`sdk/src/abis/index.ts:15-22` exports Pool/PoolFactory/Admin/Bridge/Distributor/Flash/Router/Staking/StakedGov/StakedLP/Treasury. **GovToken** (`dex/evm/src/tokens/GovToken.sol:13`) and **BridgeableERC20** (`dex/evm/src/tokens/BridgeableERC20.sol:9`) are NOT exported. Treasury is the GovToken owner (mint/burn gateway with max-supply enforcement) and front/back consumers cannot ABI-encode `mint(address,uint256)` / `crosschainMint` without these. Cross-repo alignment scoring is capped at 9.3 by this drift.
**Fix:** regen ABIs + add `export * from './GovToken.js'` and `./BridgeableERC20.js` (or document as intentional internal-only).

### G10 (NEW) -Pool ABI mirror constants now masking lib names in sdk
`Pool.sol:88-94` re-exports 5 PoolOracle constants for ABI stability. Necessary, but the sdk consumer reads them via `Pool` ABI; a downstream rename of `PoolOracle.FAST_WINDOW` would silently desync. No test asserts mirror equality at compile time (would need a `static_assert`-style invariant test or a `forge inspect Pool` constant-snapshot in CI).
**Fix:** add a `test_constants_mirror_PoolOracle` test asserting `Pool.FAST_WINDOW() == PoolOracle.FAST_WINDOW`.

### G6-residual -Timelock 4-shape divergence still doc-only
`shared/Timelock.sol:13-24` documents AC `Queued` / Swapper `Pending` / PoolFactory raw `uint256` divergence. Round-2 did not touch this. Cross-repo alignment reviewer (C, D) cannot lift past 9.3.

### G11 (NEW) -Deploy.s.sol still stub
`script/Deploy.s.sol:32-39` emits only `AccessControl` and bears a `TODO 42B.9b: pool proxy + router + module wiring`. No e2e deploy path is exercised by any test or CI. Governance criterion penalised: deploy provenance not audit-grade.

### G12 (minor) -`BridgeableERC20.supportsInterface` is `external pure`
`BridgeableERC20.sol:30` declares `supportsInterface` as `external` instead of `public`. Inheriting tokens (GovToken) cannot `super.supportsInterface` if extended later. Solady `ERC20` does not implement ERC165 itself, so no immediate clash; but composability with downstream ERC721/4626 mixins requiring `public override` will need refactor.
