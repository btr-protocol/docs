### A2 -auth / access -NEW critical surface

#### Previously un-audited -GovToken bridge wiring
- `GovToken._getBridge()` (tokens/GovToken.sol:30-32): calls `IBridgeProvider(TREASURY).getBridge()`.
- **Treasury exposes selector `bridge()` (line 35 `address public bridge;` auto-getter, selector `0xe78cea92`), NOT `getBridge()` (selector `0xe78cea92`-different).** The interface declares `function getBridge() external view returns (address);` -selector is `keccak256("getBridge()")[:4] = 0xe78cea92` ≠ `bridge()` selector `0x...`.
- Verification: `grep -rn "function getBridge" src/` returns ZERO hits in Treasury/ITreasury. The interface `IBridgeProvider` at GovToken.sol:8 defines a function selector that does NOT exist on the Treasury contract.
- Concrete consequence: `IBridgeProvider(TREASURY).getBridge()` reverts (selector mismatch = no fallback handler on Treasury → revert). The `try/catch` block at GovToken.sol:31 swallows the revert and **returns `address(0)`**.
- Downstream: `BridgeableERC20.onlyBridge` modifier (line 14-17) requires `msg.sender == _getBridge()`, i.e. `msg.sender == address(0)`, which is impossible.
- ⇒ **`crosschainMint` and `crosschainBurn` on the gov token always revert with `UnauthorizedBridge()`.**
- Impact path: Bridge contract calls `IERC7802(token).crosschainBurn(msg.sender, amount, ...)` (Bridge.sol:89) inside `bridgeViaLayerZero`. If `token == govToken`, the burn reverts ⇒ users CANNOT bridge gov tokens out. On the inbound side, `lzReceive` (Bridge.sol:150) calls `crosschainMint` which reverts ⇒ inbound message goes to `failedMessages` queue forever (admin recovery via `recoverFailedMessage` ALSO calls `crosschainMint` at Bridge.sol:242 and would revert too).
- **F-A2-R15-1 (HIGH, NEW)**: GovToken's bridge resolver calls a non-existent selector on Treasury, silently swallowed, breaking ALL crosschain mint/burn for gov tokens. Cross-chain gov token movement is fully bricked. Inbound messages permanently stuck in `failedMessages`; even owner recovery cannot resolve (would still revert at `crosschainMint`).
- Fix options:
  1. Rename `IBridgeProvider.getBridge()` → `IBridgeProvider.bridge()` in GovToken.sol:8 (matches Treasury's auto-getter selector).
  2. Add `function getBridge() external view returns (address) { return bridge; }` to Treasury.sol.
  3. Make `GovToken._getBridge` call `Treasury(TREASURY).bridge()` directly using the typed contract instead of a one-off interface.
- Why not caught earlier: no test calls `crosschainMint`/`crosschainBurn` on GovToken end-to-end -bridging gov tokens is a multi-chain integration scenario absent from the unit suite. Pattern matches every prior round (un-tested surface harbors bugs).
