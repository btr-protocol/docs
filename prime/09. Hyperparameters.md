# Hyperparameters

> Complete reference for every tunable parameter in BTR Prime. Source: `prime/config.yml` (runtime) + `prime/crates/{ml,config}/src/` (compile-time constants).

---

## 1. Bar Generation (NXR upstream - informational)

Bars are generated by NX Rates and consumed by Prime as `nxr_sdk::Bar`. The brick-size rule is **not** tuned by Prime; it is set once at the bar-generator layer.

| Symbol | Value | File:line | Role |
|---|---|---|---|
| $m$ | `0.075` | `nx-rates/series-factory/src/bar_construction/renko.rs:36` | Adaptive Renko brick multiplier ($b_t = m \cdot \bar\sigma_P \cdot P_t$, clipped). |
| $b_{\min}$ | `0.001` (10 bps) | same | Floor on brick size as fraction of price. |
| $b_{\max}$ | `0.10` (10 %) | same | Ceiling on brick size. |
| MTF lookbacks | `[14, 60, 180]` days | `nx-rates/sdk/rust/src/vol.rs` | Inverse-variance-weighted Žā blend windows. |
| Re-compute cadence | `30 min` (`ts / 1_800_000`) | `renko.rs` | Brick rule recomputation interval (12h recalibration cooldown). |

> **NXR upstream change (2026-06).** The NXR renko sigma basis moved from Parkinson-over-idx-HLC to **Rogers-Satchell over gapless s10 OHLC** (`nxr_sdk::vol_estimator::rs_sigma_from_ohlc`; `offline == live`). The $\bar\sigma_P$ in the brick multiplier formula above is now an RS-over-s10 statistic, aligning with prime's own `rs` default. This is **distinct** from prime's internal `parkinson_lookback = 50` (┬¦2, unchanged). The consumed `nxr_sdk::Bar` contract is unchanged.

See [┬¦01 Information Bars](./01.%20Information%20Bars.md) for derivation.

## 4. Perpetual Gradient Boosting

| Parameter | Default | File:line | Role | Sensitivity |
|---|---|---|---|---|
| `budget` | `0.3` | `prime/config.yml` | Complexity ceiling for `PerpetualBooster::fit` (`perpetual` crate v2.1). Library-recommended for noisy signals. | High - bias / variance tradeoff. Not optimised in prod; sweep TODO. |
| `train_bars` | `5000` | `prime/config.yml` | Bars per training window. | Medium - data sufficiency. |
| `retrain_bars` / `step_bars` | `500` | `prime/config.yml` | Bars between retrains. | Low - concept-drift handling. |
| `horizon_bars` $H$ | `20` | `prime/config.yml` | Prediction horizon in bars. | **High** - defines label window; all Žā-scaled thresholds use $\sqrt{H}$. |
| `eval_interval` $\Delta_{\text{eval}}$ | `20` | `prime/config.yml` | Signal-generation cadence (bars). | Low. |
| `min_bars_per_fold` | impl-defined | `crates/ml/src/trainer.rs::WalkForwardConfig` | Minimum bars before a fold begins. | - |
| `mode` | `"lo"` or `"so"` | `prime/config.yml` | Long-only vs short-only (never mixed). | **Hard invariant**. |
| `ensemble_count` $E$ | `4` | `crates/ml/src/trainer.rs::WalkForwardConfig` | Inter-fold ensemble size (decay-weighted aggregation). Intra-fold seed-diversity reserved but inactive. | Medium - variance reduction. |
| `ensemble_decay` $\delta$ | `0.5` | same | Recency decay for inter-fold weighting: $w_i = \delta^{\text{age}_i}$. | Medium. |
| `meta_model` | `None` (inactive) | `crates/ml/src/trainer.rs:204-208`, `:587`, `:888` | Stage-2 meta-classifier (L├│pez de Prado). **Plumbed but not activated** in master. | - see [┬¦10 Open Questions](./10.%20Open%20Questions.md). |

### Fitness sub-score weights (production)

From `crates/backtest/src/fitness.rs:152` (default) and `crates/ml/src/cl.rs:542` (CL-specific override):

| Sub-score | $w$ (default) | $w$ (CL override) | Description |
|---|---:|---:|---|
| `trajectory` (log-equity $t$-stat ├Ś $R^2$) | 0.30 | 0.40 | Path quality. |
| `drawdown` | 0.10 | 0.10 | Max DD penalty (geometric monthly). |
| `drawdown_duration` (`ddur`) | 0.10 | 0.10 | Time-under-water penalty. |
| `tail` (CVaR-style) | 0.10 | 0.10 | Tail-loss penalty. |
| `profit_factor` (`pf`) | 0.10 | 0.10 | Gross win / gross loss. |
| `coverage` (`cov`) | 0.15 | 0.15 | Time-in-market. |
| `win_rate` (`wr`) | 0.15 | **0.05** | Directional hit rate. |
| **Sum** | **1.00** | **1.00** | (Compensating reduction in `wr` when `trajectory` is up-weighted.) |

Final fitness = (weighted geometric mean of sub-scores) ├Ś (trade-count confidence discount) ├Ś (1 if all hard filters pass else 0).
