# Hyperparameters

> Complete reference for every tunable parameter in BTR Prime. Source: `prime/config.yml` (runtime) + `prime/crates/{ml,config}/src/` (compile-time constants).

---

## 1. Bar Generation (NXR upstream - informational)

Bars are generated by NX Rates and consumed by Prime as `nxr_sdk::Bar`. The brick-size rule is **not** tuned by Prime; it is set once at the bar-generator layer.

| Symbol | Value | File:line | Role |
|---|---|---|---|
| $m$ | `0.075` | `nx-rates/series-factory/src/bar_construction/renko.rs:36` | Adaptive Renko brick multiplier ($b_t = m \cdot \bar\sigma_P \cdot P_t$, clipped). |
| $b_{\min}$ | `0.001` (10 bps) | same | Floor on brick size as fraction of price. |
| $b_{\max}$ | `0.10` (10 %) | same | Ceiling on brick size. |
| MTF lookbacks | `[30, 60, 180]` days | `parkinson.rs:51-130` | Inverse-variance-weighted Parkinson blend windows. |
| Re-compute cadence | `30 min` (`ts / 1_800_000`) | `renko.rs:118-135` | Brick rule recomputation interval. |

See [§01 Information Bars](./01.%20Information%20Bars.md) for derivation.

## 4. Perpetual Gradient Boosting

| Parameter | Default | File:line | Role | Sensitivity |
|---|---|---|---|---|
| `budget` | `0.3` | `prime/config.yml` | Complexity ceiling for `PerpetualBooster::fit` (`perpetual` crate v2.1). Library-recommended for noisy signals. | High - bias / variance tradeoff. Not optimised in prod; sweep TODO. |
| `train_bars` | `5000` | `prime/config.yml` | Bars per training window. | Medium - data sufficiency. |
| `retrain_bars` / `step_bars` | `500` | `prime/config.yml` | Bars between retrains. | Low - concept-drift handling. |
| `horizon_bars` $H$ | `20` | `prime/config.yml` | Prediction horizon in bars. | **High** - defines label window; all σ-scaled thresholds use $\sqrt{H}$. |
| `eval_interval` $\Delta_{\text{eval}}$ | `20` | `prime/config.yml` | Signal-generation cadence (bars). | Low. |
| `min_bars_per_fold` | impl-defined | `crates/ml/src/trainer.rs::WalkForwardConfig` | Minimum bars before a fold begins. | - |
| `mode` | `"lo"` or `"so"` | `prime/config.yml` | Long-only vs short-only (never mixed). | **Hard invariant**. |
| `ensemble_count` $E$ | `4` | `crates/ml/src/trainer.rs::WalkForwardConfig` | Inter-fold ensemble size (decay-weighted aggregation). Intra-fold seed-diversity reserved but inactive. | Medium - variance reduction. |
| `ensemble_decay` $\delta$ | `0.5` | same | Recency decay for inter-fold weighting: $w_i = \delta^{\text{age}_i}$. | Medium. |
| `meta_model` | `None` (inactive) | `crates/ml/src/trainer.rs:204-208`, `:587`, `:888` | Stage-2 meta-classifier (López de Prado). **Plumbed but not activated** in master. | - see [§10 Open Questions](./10.%20Open%20Questions.md). |

See [§05 Perpetual Boosting](./05.%20Perpetual%20Boosting.md).

## 5. Walk-Forward / Fitness

| Parameter | Default | File:line | Role |
|---|---|---|---|
| $E_{\text{purge}} = H$ | `20` bars | `crates/ml/src/trainer.rs` | Label-leakage purge (intra-fold). |
| $E_{\text{disj}} = \Delta_{\text{eval}}$ | `20` bars | same | Inter-fold disjointness embargo. |
| **Total embargo** $E$ | `40` bars | derived | Combined no-look-ahead gap. |
| Fee per side | `3` bps | `crates/backtest/src/fitness.rs` | Round-trip transaction cost model. |
| Slippage | `0` | same | **TODO**: no slippage modelled. |
| Funding / borrow | `0` | same | **TODO**: no funding cost for shorts. |
| `min_trades` (floor) | `20` | `fitness.rs:124` | Absolute floor for fitness to score. |
| `min_trades_per_week` | impl-defined | `fitness.rs` | Load-bearing gate against low-frequency overfit. |

### Fitness sub-score weights (production)

From `crates/backtest/src/fitness.rs:152` (default) and `crates/ml/src/cl.rs:542` (CL-specific override):

| Sub-score | $w$ (default) | $w$ (CL override) | Description |
|---|---:|---:|---|
| `trajectory` (log-equity $t$-stat × $R^2$) | 0.30 | 0.40 | Path quality. |
| `drawdown` | 0.10 | 0.10 | Max DD penalty (geometric monthly). |
| `drawdown_duration` (`ddur`) | 0.10 | 0.10 | Time-under-water penalty. |
| `tail` (CVaR-style) | 0.10 | 0.10 | Tail-loss penalty. |
| `profit_factor` (`pf`) | 0.10 | 0.10 | Gross win / gross loss. |
| `coverage` (`cov`) | 0.15 | 0.15 | Time-in-market. |
| `win_rate` (`wr`) | 0.15 | **0.05** | Directional hit rate. |
| **Sum** | **1.00** | **1.00** | (Compensating reduction in `wr` when `trajectory` is up-weighted.) |

Final fitness = (weighted geometric mean of sub-scores) × (trade-count confidence discount) × (1 if all hard filters pass else 0).

See [§06 Walk-Forward Validation](./06.%20Walk-Forward%20Validation.md).

## 6. Directional Trading

All in $\sigma_P \cdot \sqrt{H}$ units unless stated.

| Parameter | Default | File:line | Role |
|---|---|---|---|
| `entry_threshold_sigma` | `0.20` | `prime/config.yml` | Signal-magnitude floor for entry. |
| `max_loss_per_trade_pct` $\rho$ | `0.02` (2%) | `prime/config.yml` | Risk-budget cap per trade. |
| `sl_min_sigma` | `0.5` | `prime/config.yml` | σ-floor for stop loss. |
| `sl_max_sigma` | `4.0` | `prime/config.yml` | σ-ceiling for stop loss. |
| `use_take_profit` | `true` | `prime/config.yml` | Toggle for TP exit. |
| `tp_sigma_mult` | `2.0` | `prime/config.yml` | TP = $\|\hat y\| + 2\hat\sigma_P\sqrt{H}$ (Bayesian-margin guard). |
| `position_size_pct` $s$ | `0.15` (15%) | `prime/config.yml` | Capital fraction per trade. |
| `leverage` $\ell$ | `3.0×` | `prime/config.yml` | Default leverage. |
| `leverage_min` | `0.1×` | `prime/config.yml` | Grid-search floor. |
| `leverage_max` | `10×` | `prime/config.yml` | Grid-search ceiling. |
| DD-scaled sizing multiplier $\beta(\text{DD})$ | $1.0 \to 0.3$ floor at 12% DD | `trainer.rs:1219-1222` | Drawdown-dependent position de-risking. |
