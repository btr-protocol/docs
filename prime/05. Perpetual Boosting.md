# Perpetual Boosting


This section formalizes the learning core of BTR Prime: a walk-forward, ensemble, two-stage gradient boosting pipeline built on the `perpetual` crate (v2.1). We derive each component from first principles, map the algorithm to the concrete source in `crates/ml/{model.rs, trainer.rs}`, and document every hyperparameter visible in `WalkForwardConfig::default()`.

### §5.1 Background - Gradient Boosting (Friedman 2001 [F1])

> [F1] J. H. Friedman, "Greedy Function Approximation: A Gradient Boosting Machine", *Annals of Statistics*, vol. 29, no. 5, pp. 1189–1232, 2001.


Gradient boosting is the additive, function-space realization of empirical risk minimization. Given a training sample $\lbrace (x_i, y_i)\rbrace_{i=1}^N$ with $x_i \in \mathbb{R}^d$, $y_i \in \mathcal{Y}$, and a differentiable loss $L : \mathcal{Y} \times \mathbb{R} \to \mathbb{R}_{\ge 0}$, we seek

$$F^\star = \arg\min_{F \in \mathcal{F}} \; \mathbb{E}_{(X,Y)}\left[ L(Y, F(X)) \right].$$

Friedman (2001) constructs $F^\star$ as a greedy, stage-wise sum of weak learners $h_m \in \mathcal{H}$ (typically depth-bounded regression trees):

$$F_M(x) = F_0(x) + \sum_{m=1}^M \eta \cdot h_m(x), \qquad h_m \in \arg\min_{h \in \mathcal{H}} \sum_{i=1}^N L\left(y_i, F_{m-1}(x_i) + h(x_i)\right),$$

where $F_0 = \arg\min_c \sum_i L(y_i, c)$ is the constant prior and $\eta \in (0,1]$ is the shrinkage rate. The inner minimization is approximated by fitting $h_m$ to the negative gradient (pseudo-residual)

$$r_{i,m} = -\left.\frac{\partial L(y_i, F(x_i))}{\partial F(x_i)}\right|_{F = F_{m-1}}.$$

For squared loss $L(y,F) = \tfrac12(y-F)^2$, $r_{i,m} = y_i - F_{m-1}(x_i)$; for logistic loss $L(y,F) = \log(1 + e^{-(2y-1)F})$ on $y \in \lbrace 0,1\rbrace$, $r_{i,m} = y_i - \sigma(F_{m-1}(x_i))$. Modern variants (XGBoost, LightGBM, CatBoost) refine this with second-order Newton steps: the optimal leaf weight under a tree partition $\lbrace R_j\rbrace$ is

$$w_j^\star = -\frac{\sum_{i \in R_j} g_i}{\sum_{i \in R_j} h_i + \lambda}, \qquad g_i = \partial_F L,\; h_i = \partial_F^2 L,$$

with $\lambda$ an L2 regularizer. The split gain when partitioning a node $R \to (R_L, R_R)$ is

$$\Delta\mathcal{L} = \tfrac12\left[\frac{(\sum_{i \in R_L} g_i)^2}{\sum_{i \in R_L} h_i + \lambda} + \frac{(\sum_{i \in R_R} g_i)^2}{\sum_{i \in R_R} h_i + \lambda} - \frac{(\sum_{i \in R} g_i)^2}{\sum_{i \in R} h_i + \lambda}\right] - \gamma,$$

with $\gamma$ a split-cost penalty. Histogram-based learners (LightGBM, CatBoost) discretize features into $B$ bins and accumulate $(g_i, h_i)$ sufficient statistics in $O(NB)$ per split, sidestepping per-feature sort.

The standard recipe exposes a high-dimensional hyperparameter surface - $(M, \eta, \text{depth}, \lambda, \gamma, B, \text{subsample}, \text{colsample}, \ldots)$ - and is typically tuned by Bayesian optimization or random search at significant compute cost. The pipeline below removes this surface entirely.

### §5.2 Perpetual Variant - Self-Generalizing Boosting Under a Budget

Library: `perpetual = "2.1"` (`Cargo.toml:15`), maintained at <https://github.com/perpetual-ml/perpetual>. Author: M. Simsek; algorithmic paper in preparation as of 2026-05. The core idea is to replace the discrete iteration count $M$ and the joint hyperparameter tuning step with a single continuous *generalization budget* $b \in \mathbb{R}_{\ge 0}$. The model is constructed iteratively, but each candidate split is admitted only if it satisfies a budget-controlled generalization criterion estimated from an internal validation signal. Algorithm growth halts when no further admissible split exists.

Informally - and consistent with the public documentation, since the formal paper is unpublished - the booster maintains an implicit relation of the form

$$\widehat{\text{GenGap}}(t_k) \le \phi(b),$$

where $\widehat{\text{GenGap}}$ is an estimator of the generalization gap induced by candidate split $t_k$ and $\phi$ is a monotone increasing function of the budget. Lower $b$ admits only splits with small generalization gap (high-confidence signal, simpler model); higher $b$ admits weaker splits (more complex model, lower training loss). The user is therefore exposed to a single scalar that traces a one-dimensional path in the bias–variance plane - analogous in spirit to early-stopping the LightGBM iteration count, but applied per-split and per-tree rather than per-round.

The Rust API (see `model.rs:62–77`) is the minimal surface:

```rust
PerpetualBooster::default()
    .set_objective(objective)            // LogLoss | SquaredLoss
    .set_budget(cfg.budget as f32)       // self-generalizing knob, in (0, ∞)
    .set_max_bin(64)                     // histogram discretization
    .set_num_threads(Some(1))            // avoid rayon pool thrash under outer parallelism
    .set_allow_missing_splits(false)     // we pre-filter non-finite rows
    .set_seed(42)
    .set_stopping_rounds(Some(10))       // safety net: 10 rounds without improvement
    .set_iteration_limit(Some(200));     // hard cap
```

Salient implementation facts inferred from the crate's public docs and the call site:

- **Histogram trees.** `set_max_bin(64)` indicates feature pre-binning, in the LightGBM / CatBoost style. Sufficient statistics $(\sum g, \sum h)$ are accumulated per bin in $O(NB)$.
- **Second-order steps.** With `Objective::LogLoss` the derivation requires per-row $h_i = \sigma(F)(1-\sigma(F))$, so the underlying step is Newton, not bare first-order. *(TODO confirm via library source - surface API is opaque.)*
- **Single-run training.** No external CV loop, no learning-rate sweep. The user calls `booster.fit(&matrix, labels, None, None)` once.
- **Online extension.** Per upstream docs, `fit_predict` semantics allow continual learning: an already-trained model can be extended on a new $(X, y)$ shard, retaining prior trees while admitting new ones under the same budget rule. Prime does **not** currently use online extension - each fold trains a fresh booster (`model.rs:55, 132`) - but the capacity is preserved for future incremental retraining.

The practical consequence is that the *outer* pipeline (this section) carries zero GBM hyperparameter search; tuning effort is redirected to the meta-pipeline (window sizes, ensemble depth, σ-budgets), which is where most of the financial signal/noise trade-off lives.

### §5.3 The Complexity Budget

Configured value: `budget = 0.3` (`WalkForwardConfig::default()`, `trainer.rs:113`; mirrored in `ModelConfig::default()`, `model.rs:19`).

Algorithmically the budget is the only knob controlling the model's capacity. Define a model class $\mathcal{F}_b \subset \mathcal{F}$ parameterized by budget $b$. We may informally relate $b$ to the achieved excess risk by

$$\mathbb{E}[L(Y, F_b(X))] - \mathbb{E}[L(Y, F^\star(X))] \;=\; \underbrace{\mathbb{E}[L(Y, F_b^\star(X))] - \mathbb{E}[L(Y, F^\star(X))]}_{\text{bias}^2(b),\; \downarrow b} \;+\; \underbrace{\mathbb{E}[L(Y, F_b(X)) - L(Y, F_b^\star(X))]}_{\text{variance}(b),\; \uparrow b}.$$

The mapping $b \mapsto (N_{\text{trees}}, N_{\text{leaves/tree}})$ is implicit: increasing $b$ relaxes the per-split admissibility test, so both depth and round count grow. At $b = 0.3$ Prime sits near the conservative end of the typical operating range $[0.1, 2.0]$ recommended by the crate, which is consistent with the typical signal-to-noise ratio of forward-return labels on high-frequency bars: aggressive growth on $|\rho_{x,y}| \sim 10^{-2}$ labels is variance-dominated and degrades out-of-sample Sharpe.

Sensitivity: high in principle. $b$ is the single highest-leverage knob in the trainer; small changes propagate through every fold and dominate the bias–variance allocation.

**Defense of $b = 0.3$.** The configured value follows the upstream
`perpetual` crate's recommended default for noisy regression / classification
signals (v2.1 README and crate docs at
<https://github.com/perpetual-ml/perpetual>; the library's example
notebooks routinely use $b \in [0.1, 0.5]$ for low-SNR financial / tabular
data). It is therefore best characterised as a **library-default
calibration**, *not* a Prime-internal derivation. We have **not** run a
formal $b$-sweep with Sharpe / hit-rate curves on BTR's Renko/DIB bars
inside the production loop; sensitivity in OOS performance is **unknown**
in the strict sense, and $b$ is **not** optimised by the outer pipeline.
Tracked in [Open Questions §3.5](./10.%20Open%20Questions.md). The experiment to run: insert a $b$-grid walk-forward Sharpe sweep over
$b \in \lbrace 0.1, 0.2, 0.3, 0.5, 1.0\rbrace$ to either confirm the library default
or move the production setting. Until that experiment lands, treat $b = 0.3$
as a justified library default rather than an empirically optimised choice.

Safety rails (`model.rs:75–76`): `stopping_rounds = 10` and `iteration_limit = 200`. These bound pathological growth even if the budget criterion fails to halt; in practice the perpetual halt condition fires well before either cap.

### §5.4 Two-Stage Meta-Labeling (López de Prado 2018, ch. 3)

Prime implements a two-stage classifier architecture. Each stage is an independent perpetual booster.

**Stage 1 - Primary classifier.** Binary target $y^{(1)} \in \lbrace 0, 1\rbrace$ encoding whether the forward return exceeds a threshold in the configured trading direction. From `trainer.rs:457–476`:

$$r_i \;=\; \frac{C_{i+H}}{C_i} - 1, \qquad \tilde r_i = \begin{cases} +r_i & \text{mode} = \text{lo} \\ -r_i & \text{mode} = \text{so} \end{cases},$$

$$y^{(1)}_i = \mathbb{1}\left[\tilde r_i > \tau\right], \qquad \tau = \text{label-threshold} = 10^{-3}.$$

Here $C_i$ is the close price of bar $i$ and $H = \text{horizon-bars} = 20$. The pipeline is strictly directional: long-only ("lo") or short-only ("so"), never simultaneously. This is an explicit design constraint - a single binary classifier per directional regime is unbiased relative to a two-class signed classifier whose decision boundary would otherwise distort the loss surface under class imbalance.

In regression mode (`trainer.rs:461–468`) the label is replaced by the Parkinson-normalized return $\tilde r_i / (\sigma_P^{(i)} \sqrt{H})$, clamped to $[-5, 5]$. This effectively predicts moves in units of forward-horizon Parkinson volatility, decoupling the model from the absolute price scale.

**Stage 2 - Meta classifier.** Conditional on Stage-1 positives, predict whether the trade would actually be profitable after costs. Target $y^{(2)} \in \lbrace 0,1\rbrace$ indicates a profitable realization of the trade prescribed by Stage 1.

The López de Prado argument for this decomposition rests on three pillars:

1. **Conditional probability factorization.** $P(\text{profitable trade} \mid x) = P(\text{direction} \mid x) \cdot P(\text{profitable} \mid \text{direction}, x)$. Forcing a single model to learn the product wastes capacity on the joint distribution; the two-stage factorization isolates each conditional.
2. **Direction and sizing decoupled.** Stage 1 provides a *signal*; Stage 2 provides a *conviction*. Position size $f \cdot \hat p^{(2)}$ uses the meta-probability as a Kelly-like sizing factor, independent of the directional decision.
3. **False-positive filtering.** Stage 2 is trained only on Stage-1 positives, so its base rate reflects the *quality* of Stage-1 alerts rather than the unconditional return distribution. This is the classical recall-trade-precision lever.

**Status in master - plumbed, not active.** This subsection describes the
*theoretical* two-stage architecture. The current production loop does
**not** train a Stage-2 meta classifier:

- `FoldModels.meta_model: Option<GBMModel>` is the architectural slot
  (`trainer.rs:204–208`).
- Every active construction site instantiates `meta_model: None`
  (`trainer.rs:587`, `trainer.rs:888`).
- There is no Stage-2 fit anywhere in `walk_forward_ml_sliced` or
  `walk_forward_signals_labeled`.

In place of a meta-probability, the bet-size / confidence signal in
`backtest_signals` (`trainer.rs:1222–1227`) is the **Stage-1 prediction
magnitude itself**:

- In **regression mode** (`use_regression: true`), `confidence = |ŷ|` where
  $\hat y$ is the Parkinson-normalised Stage-1 regression output. This is
  literally the absolute value of the prediction, *not* a calibrated
  probability of trade profitability.
- In **classification mode**, `confidence = 2·|ŷ − 0.5|` (the
  "conviction" - distance of the predicted class probability from the
  decision boundary, mapped to $[0,1]$).

Neither quantity is a true Stage-2 meta-label. The López de Prado
factorisation $P(\text{profit} \mid x) = P(\text{direction} \mid x) \cdot
P(\text{profit} \mid \text{direction}, x)$ is **not currently realised**
in Prime; the second factor is approximated by the magnitude of the first
factor's prediction. Turning Stage 2 on requires (i) implementing nested
cross-fitting so the meta classifier does not train on Stage-1 in-sample
predictions (López de Prado [1, ch. 7]; see [§06 Walk-Forward Validation](./06.%20Walk-Forward%20Validation.md) for the
architectural slot but does not enforce it), and (ii) wiring a Stage-2 fit
+ inference call into the walk-forward loop. Both are explicit
*architectural intent*, not active behaviour.

### §5.5 Fold-Decay Aggregation (and Architectural Seed-Diversity Slot)

This section distinguishes the **inter-fold decay aggregation** that runs in
master from the **intra-fold seed-diverse ensemble** whose framework is
plumbed in the type system but **not** activated in the production loop.

**Status (master, as of this writing).** The single per-fold model is the
operative reality:

- `FoldModels.primary_models: Vec<GBMModel>` is the seed-replicate slot
  (`trainer.rs:204–208`).
- At every fit site the loop pushes a **one-element vector**:
  `primary_models: vec![model]` (`trainer.rs:587`, mirrored at
  `trainer.rs:888`).
- No seed sweep is performed; `ModelConfig::default()` sets a fixed
  `seed = 42` (`model.rs:75`).

Concretely the master pipeline is therefore:

> **One Stage-1 booster per fold; up to $E$ folds retained in a
> decay-weighted moving ensemble across walk-forward steps.**

The "seed-diverse intra-fold ensemble" is **architecturally supported but
inactive**. We describe both layers below, separated by what runs vs what
is reserved capacity.

#### §5.5.1 Inter-fold decay aggregation (ACTIVE)

Configured values (`trainer.rs:108–109`):

$$E = \text{ensemble-count} = 4, \qquad \delta = \text{ensemble-decay} = 0.5.$$

Effective ensemble size is $N_{\text{eff}} = (\sum w_f)^2 / \sum w_f^2 \approx 2.65$ at $(E, \delta) = (4, 0.5)$ - meaningfully lower than the nominal $E = 4$ because the geometric decay concentrates weight on the most recent fold. Explicit arithmetic in §5.5.1 below.

`fold_history: VecDeque<FoldModels>` has capacity $E$
(`trainer.rs:504, 590`). At prediction time, fold $f$ with age
$a_f = (|\text{history}| - 1 - f)$ contributes weight $w_f = \delta^{a_f}$:

$$\hat y(x) \;=\; \frac{\sum_{f} w_f \hat p^{(f)}(x)}{\sum_{f} w_f}
         \;=\; \frac{\sum_{f} \delta^{a_f} \hat p^{(f)}(x)}{\sum_{f} \delta^{a_f}}.$$

Here $\hat p^{(f)}(x)$ is the **single** fold-$f$ booster's prediction (not
an intra-fold mean), reflecting the `primary_models: vec![model]` reality.
The inner averaging loop at `trainer.rs:633–642` still iterates over
`primary_models`, but degenerates to identity for a length-1 vector.

This is the discrete-time analogue of exponential smoothing of a
non-stationary regressor / classifier. With $\delta = 0.5$ and $E = 4$,
the effective sample size in folds is

$$N_{\text{eff}}^{\text{fold}} \;=\; \frac{(\sum_f w_f)^2}{\sum_f w_f^2}
\;=\; \frac{(1+0.5+0.25+0.125)^2}{1+0.25+0.0625+0.015625}
\;\approx\; \frac{3.516}{1.328} \;\approx\; 2.65,$$

so the ensemble effectively averages $\approx 2.65$ folds, heavily weighted
to the most recent fit while retaining a stabilising tail.
In `walk_forward_signals_labeled` (`trainer.rs:931–938`) ensemble members
with $w_f < 0.01$ are skipped as a compute optimisation - at $\delta = 0.5$
this prunes folds with age $> 6$.

**Justification - robustness to data shift.** Under regime change between
fold $f$ and prediction-time, the fold's bias contribution grows roughly
linearly with the distance in feature-distribution space. Geometric decay
$\delta^{a_f}$ ensures the bias contribution from a fold $k$ steps stale
shrinks faster than typical drift accumulates, while $E \ge 2$ ensures we
never rely solely on the most recent (smallest sample, highest variance)
fit.

#### §5.5.2 Intra-fold seed diversity (RESERVED, not active)

**FUTURE - when $|M_f| > 1$:** boosters trained on the same data with
different random seeds differ via (i) histogram tie-breaking, (ii) feature
subsampling order, and (iii) any stochastic component of the perpetual
budget test. An intra-fold mean
$\bar p^{(f)}(x) = \frac{1}{|M_f|}\sum_{m \in M_f} \hat p_m(x)$
would replace the single-model $\hat p^{(f)}(x)$ above. The bias–variance
decomposition under squared loss is

$$\mathbb{E}\left[(\bar p^{(f)} - y)^2\right]
  = \mathrm{bias}_f^2
  + \frac{1}{|M_f|}\mathrm{Var}_m
  + \left(1 - \frac{1}{|M_f|}\right)\overline{\mathrm{Cov}}_{m \ne m'},$$

where $\mathrm{Var}_m$ is per-seed prediction variance and
$\overline{\mathrm{Cov}}_{m \ne m'}$ is the average pairwise covariance
across seeds. Because seed-replicates fit on identical $X$ with identical
$b$, $\overline{\mathrm{Cov}} < \mathrm{Var}_m$, and the variance term
strictly decreases in $|M_f|$. Note this is a squared-loss identity; under
log-loss (classification mode) the bias–variance identity does not have the
same closed form and the argument above is a **structural analogy**, not a
derivation. Pinsker / Bregman decompositions of log-loss
(see e.g. James 2003) recover similar qualitative behaviour but differ in
the coefficients.

Combined intra-fold + inter-fold variance reduction (when both layers are
active) would give

$$\hat y(x) = \frac{\sum_f w_f \bar p^{(f)}(x)}{\sum_f w_f},$$

with full bias–variance–covariance decomposition

$$\mathbb{E}\left[(\hat y - y)^2\right]
  = \Bigl(\sum_f \tilde w_f \mathrm{bias}_f\Bigr)^2
  + \sum_f \tilde w_f^2 \mathrm{Var}_f
  + 2 \sum_{f<g} \tilde w_f \tilde w_g \mathrm{Cov}(f,g),
  \qquad \tilde w_f = w_f / \textstyle\sum_g w_g.$$

**Candidate activation parameters.** A plausible future configuration is
$|M_f| = E_{\text{intra}} = 4$ seeds per fold combined with the existing
$\delta = 0.5$ inter-fold decay (so a folded $N_{\text{eff}}$ on the order
of $|M_f| \cdot N_{\text{eff}}^{\text{fold}} \approx 10.6$ effective
boosters at prediction time). Enabling this requires:

1. Replacing `primary_models: vec![model]` with a seed sweep at
   `trainer.rs:587, 888` - see the existing per-seed builder at
   `model.rs:98–116`.
2. Auditing the parallel-mean code at `trainer.rs:633–642` for correctness
   when $|M_f| > 1$ (currently exercised only as the identity).
3. Computing predictive uncertainty (per-seed std-dev) for downstream
   confidence calibration - a capability currently absent.

Until this work lands, the practical bias–variance budget is carried by
inter-fold decay alone (§5.5.1).

### §5.6 Algorithm - Walk-Forward Training Loop

The full pipeline is implemented in `walk_forward_ml_sliced` (`trainer.rs:435–730`) and the signals-only variant `walk_forward_signals_labeled` (`trainer.rs:787–1016`). The structure below is a unified pseudocode mapped to source line ranges.

```text
Input:  bars[0..n], features[i, :] ∈ ℝ^d, cfg
Output: signals[t_k] = (signal ∈ {−1,0,+1}, confidence, predicted_return)

# ── Labeling (trainer.rs:444–476) ───────────────────────────────────────────
σ_P[i] ← rolling_parkinson(bars, lookback)                  # eq. §5.7
for i in 0..n:
    if i + H < n and C_i > 0:
        r_i ← C_{i+H}/C_i − 1
        r̃_i ← (mode = "so") ? −r_i : r_i
        if regression:
            y_i ← clamp(r̃_i / (σ_P[i] · √H), −5, 5)
        else:
            y_i ← 𝟙[r̃_i > τ]                                 # τ = label_threshold

# ── Walk-forward loop (trainer.rs:504–703) ──────────────────────────────────
cursor ← train_bars
embargo ← H + eval_interval                                  # purge leakage
fold_history ← empty deque (capacity = E)
stride ← max(1, train_bars / 7000)                           # cap fold to 7k rows

while cursor + embargo < n:
    train_end   ← cursor − H
    train_start ← train_end − train_bars

    # Stage-1 fit (trainer.rs:524–570, 875)
    (X_tr, y_tr) ← gather valid rows in [train_start, train_end] step stride
    if |X_tr| < min_bars_per_fold: cursor += retrain_bars; continue
    booster_f ← PerpetualBooster(b = budget, obj, seed).fit(X_tr, y_tr)

    fold_history.push(booster_f)
    if |fold_history| > E: fold_history.pop_front()

    # Prediction window (trainer.rs:594–677)
    pred_idx ← [cursor + embargo, cursor + retrain_bars + embargo) step eval_interval
    for each f in fold_history:                              # rayon parallel
        # In master: |primary_models| = 1, so this mean degenerates to a
        # single-model prediction. The averaging stays for future seed
        # sweep activation (see §5.5.2).
        p̄_f ← mean over seed-replicates of booster_f.predict(features[pred_idx])
    for j, i in enumerate(pred_idx):
        ŷ_ij ← Σ_f δ^{age_f} · p̄_f[j]   /   Σ_f δ^{age_f}

        if regression:
            signal ← sign(ŷ) · 𝟙[|ŷ| > entry_threshold_σ]
            confidence ← |ŷ|
            predicted_return ← ŷ
        else:
            conviction ← 2·|ŷ − 0.5|
            signal ← sign(ŷ − 0.5) · 𝟙[conviction > min_conviction]
            confidence ← conviction
            predicted_return ← ŷ − 0.5

        emit MLSignal(ts[i], predicted_return, signal, confidence)

    cursor += retrain_bars
```

**Embargo.** The gap $H + \text{eval-interval}$ between training-end and prediction-start (`trainer.rs:495, 594`) is the *purge / embargo* required to defeat label leakage: a sample at index $i$ has label depending on $C_{i+H}$, so training on $i \le \text{train-end}$ and predicting at $i' \ge \text{train-end} + H$ ensures the forward window of any training row does not overlap the prediction set.

**Stride.** `train_stride = ceil(train_bars / 7000)` (`trainer.rs:501–502`) caps the per-fold training matrix at ~7k rows. Histogram boosters exhibit diminishing returns past ~10k samples (variance reduction $\propto N^{-1/2}$, training cost $\propto N$); stride keeps walk-forward latency $O(1)$ in `train_bars`.

**Cache-friendly transpose.** Features are stored row-major globally (one row per bar) but `perpetual::Matrix::new` consumes column-major. The blocked transpose at `trainer.rs:548–560, 859–873` uses $64 \times 64$ blocks chosen to fit in L1 cache, avoiding the $O(N d)$ random-write pattern of a naive transpose.

**Per-fold feature importance.** Each fold records `feature_importance()` via `ImportanceMethod::Gain` (`model.rs:121`), which sums the Stage-1 split-gain $\Delta\mathcal{L}$ contributions per feature. Aggregated importance averages across the final ensemble (`trainer.rs:706–719`).

**Meta-stage hook.** `FoldModels.meta_model: Option<GBMModel>` is the architectural slot for Stage 2 (currently `None` in the active loop). The meta-labels would be generated by replaying Stage-1 predictions on the training window, simulating the trade under the same SL/TP rules as `backtest_signals`, and assigning $y^{(2)}_i = \mathbb{1}[\text{net pnl}_i > 0]$.

### §5.7 Hyperparameter Surface

All defaults below are taken from `WalkForwardConfig::default()` (`trainer.rs:99–135`) and `ModelConfig::default()` (`model.rs:18–20`). Values not yet wired to the production `config.yml` carry the source default.

| Param | Value | Algorithmic role | Sensitivity |
|---|---:|---|---|
| `budget` | 0.3 | Perpetual booster complexity ceiling; sole GBM knob. Controls per-split admissibility test $\widehat{\text{GenGap}} \le \phi(b)$. | **high** - direct bias/variance allocation |
| `train_bars` | 5 000 | Length of the Stage-1 training window in information units (bars). | medium - data sufficiency vs. stationarity |
| `retrain_bars` | 500 | Bars between consecutive fold fits (a.k.a. `step_bars`). | low - sets drift-tracking rate |
| `horizon_bars` $H$ | 20 | Forward-return horizon defining the label $r_i = C_{i+H}/C_i - 1$. | **high** - defines the prediction target |
| `eval_interval` | 20 | Bars between successive signal emissions inside a fold. | low - signal cadence |
| `min_bars_per_fold` | 500 | Reject folds with insufficient valid rows. | low - safety floor |
| `ensemble_count` $E$ | 4 | Folds retained in decay-weighted **inter-fold** ensemble. Intra-fold seed count is fixed at 1 in master (§5.5). | **high** - variance reduction vs. memory |
| `ensemble_decay` $\delta$ | 0.5 | Geometric decay of older folds. Effective fold sample size $N_{\text{eff}}^{\text{fold}} \approx 2.65$ at $E{=}4$. | medium - recency bias under drift |
| `label_threshold` $\tau$ | 0.001 | Stage-1 positive cutoff (classification mode). Decoupled from conviction per user note. | low (fixed) |
| `use_regression` | false | Toggle: classification (default) vs. Parkinson-normalized regression. | medium - changes objective + signal interpretation |
| `with_time_features` | false | Append `(hour_sin, hour_cos)` features - only valid for time-based bars. | low |
| `parkinson_lookback` | 50 | Rolling window for $\sigma_P$ used in labels (regression), SL, TP, entry. | medium |
| `entry_threshold_sigma` | 0.20 | Regression entry: trade only if $|\hat y| > k \cdot \sigma_P \sqrt{H}$ with $k = 0.20$. | high - selectivity/turnover |
| `max_loss_per_trade_pct` | 0.02 | Risk-budget SL: $SL = L_{\max}/(\text{lev} \cdot \text{pos-pct})$ bounded in $\sigma_P$ units. | high |
| `sl_min_sigma`, `sl_max_sigma` | 0.5, 4.0 | $\sigma_P\sqrt{H}$-bounded SL clamp; trades skipped if too tight. | high |
| `tp_sigma_mult` | 2.0 | Adaptive TP: $\text{TP} = |\hat r| + k_{tp} \cdot \sigma_P \sqrt{H}$. | high |
| `leverage`, `position_size_pct` | 3.0, 0.15 | Notional sizing pre-confidence-scaling. | high |
| `fee_rate_bps` | 3.0 | Per-side fee (0.03%) applied at entry & exit; deducted from PnL. | medium |
| `compound` | false | Position base = initial capital (fixed-fraction), not running equity. | medium - affects compounding dynamics |

The Parkinson volatility estimator used throughout (`trainer.rs:238–272`) is

$$\hat\sigma_P^{(i)} = \sqrt{\frac{1}{4\ln 2}\cdot \frac{1}{L}\sum_{j=i-L+1}^{i}\left[\ln\frac{H_j}{L_j}\right]^2}, \qquad L = \text{parkinson-lookback} = 50,$$

scaled by $\sqrt H$ to produce horizon-aligned volatility $\sigma_P\sqrt H$ used as the unit of measurement for entry thresholds, stop-loss bounds, take-profit offsets, and regression label normalization.
