# Features


### §3.1 Overview

BTR Prime's supervised ML stage operates on a stream of `nxr_sdk::Bar` records (96 B = 64 B OHLCV + 32 B TICK-derived microstructure). Each bar carries not only the classical 5-tuple $B_t = (o_t, h_t, l_t, c_t, v_t)$ but also nine sub-bar TICK statistics computed upstream by NX Rates from the raw tick stream: `realized_var`, `bipower_var`, `drift`, `vol_imbalance`, `avg_spread_bps`, `max_abs_return`, `tick_count`, `avg_ci_ubp`, `reject_rate`, plus `open_ts/close_ts` (yielding `duration_ms`). Bars are **information units**, not time units - the renko construction ensures each $B_t$ encodes approximately the same quantum of price information (a fixed $\Delta_{px}$ in renko, a fixed dollar-imbalance threshold in DIB). All features defined below are therefore implicitly conditioned on equal-information sampling, which is what restores the near-Gaussian moment structure exploited downstream by the GBM walk-forward optimizer.

**Block decomposition.** The matrix concatenates four blocks in order (computed in `crates/bin/src/ml.rs:400-593` and `crates/ml/src/features/mtf.rs`):

| Block | Count | Source file | Always on? |
|---|---:|---|---|
| MTF core (per-bar, S/M/L) | **29** | `features/mtf.rs::compute_all_features` | yes |
| MTF time features (optional) | 2 | `features/mtf.rs:240-241` | **disabled** in prod (`ml.rs:604` `with_time_features: false`) |
| Renko-tracker (3 scales x 4) | **12** | `renko_tracker.rs::compute_renko_features` | yes (off only via `--skip-renko`; 11 if `--renko-ablation`) |
| Cross-asset (per alt) | **8** | `cross_features.rs::compute_cross_features` | only when at least one `--alt-data` is passed |
| Microstructure (TICK-derived) | **12** | `crates/bin/src/ml.rs:406-502` | yes (appended unconditionally at `ml.rs:585-593`) |

Configuration counts:
- **Baseline** (no alt): $29 + 12 + 0 + 12 = \textbf{53}$ features assembled.
- **Single alt (canonical audited config)**: $29 + 12 + 8 + 12 = \textbf{61}$ features assembled, **32 active in production** after Phase 42Z compression. The 61-baseline path is retained for ablation studies (see §3.7).
- **Maximum** (2 alts, all blocks on): $29 + 12 + 16 + 12 = \textbf{69}$ assembled.

Time features (`time_sin`, `time_cos`) are intentionally **excluded** from the production matrix (`with_time_features: false` at `crates/bin/src/ml.rs:604`); see §3.3.15. The previous documentation reported "$29 + 8 + 12 = 49$", which silently dropped the 12 microstructure features appended in `ml.rs:585-593`; this page is the corrected reference.

Two design invariants hold across the entire feature module ([`crates/ml/src/features/mod.rs`](file:///Users/derpa/Work/btr/prime/crates/ml/src/features/mod.rs), [`features/mtf.rs`](file:///Users/derpa/Work/btr/prime/crates/ml/src/features/mtf.rs)):

1. **Causality.** Every feature at index $i$ is a measurable function of $\lbrace B_j : j \leq i\rbrace$ only. Look-ahead bias is structurally impossible by construction - all windows are written as $[i+1-w \, .. \, i]$ or $[i-w \, .. \, i]$.
2. **Single-pass O(n) precomputation.** Return series $r_t$, on-balance volume $\text{OBV}_t$, and the two EMAs $\text{EMA}^S_t, \text{EMA}^M_t$ are computed once over the full series (`mtf.rs:125-145`); per-bar feature evaluation then runs over precomputed slices.

Time-of-day features (`time_sin`, `time_cos`) are **optional** and gated by `with_time_features`. The default configuration excludes them: information-bar sampling already absorbs intraday seasonality (slow-trade periods produce fewer bars), so injecting calendar time as a feature would be redundant at best and a leakage vector at worst (regimes correlate with wall-clock time across the train/test boundary).

The number of valid rows is $N - L$, where $L$ is the long-scale lookback. Earlier rows are filled with `f64::NAN` (`mtf.rs:148-150`) and the GBM trainer skips them. The constant `MIN_BARS = 500` (`mtf.rs:39`) enforces the minimum series length before any feature row is considered valid, providing margin above the longest default lookback $L = 200$.

### §3.2 Multi-Timeframe Scheme

The MTF scheme uses three lookback scales tied to the prediction horizon $H$ (`horizon_bars`):

$$
(S, M, L) = (1 \cdot H,\ 3 \cdot H,\ 10 \cdot H), \quad H \geq 5
$$

implemented in `compute_lookbacks` (`mtf.rs:48-51`) with the safety floor `let h = horizon_bars.max(5)`. When the caller passes `horizon_bars = 0`, the defaults `(DEFAULT_S, DEFAULT_M, DEFAULT_L) = (20, 60, 200)` (`mtf.rs:27-31`) apply.

The 1:3:10 ratio is literature-aligned (López de Prado, Kakushadze) and serves a specific role: the GBM acts as a **feature selector** rather than a hyperparameter optimizer. By computing each underlying indicator at three orthogonal scales and exposing all three to the tree learner, the system avoids the classic pitfall of optimizing window length, which overfits to the lookback distribution of the training regime. The S scale matches the prediction window (direct momentum proxy); M captures intraday trend context; L is sized for regime detection - in particular Hurst R/S analysis requires $\gtrsim 100$ samples for a stable exponent estimate, satisfied by $L \geq 10 \cdot 5 = 50$ in the worst case and $L = 200$ by default.

Seven indicators are computed at all three scales, yielding $7 \times 3 = 21$ MTF features:

$$
\mathcal{F}_{\text{MTF}}(i) = \bigotimes_{w \in \lbrace S, M, L\rbrace } \big\lbrace r^{(w)}_{i},\ \text{RSI}^{(w)}_{i},\ {\sigma_P}^{(w)}_{i},\ \text{ER}^{(w)}_{i},\ \text{ADX}^{(w)}_{i},\ {\rho_1}^{(w)}_{i},\ z^{(w)}_{i} \big\rbrace
$$

These cover four indicator families: momentum (returns, RSI), volatility (Parkinson $\sigma_P$), trend (efficiency ratio, ADX, lag-1 autocorrelation $\rho_1$), and mean reversion (z-score). The remaining $29 - 21 = 8$ core features are: 2 cross-scale regime ratios, 5 derived single-scale features, and 1 vol-of-vol regime feature. With optional time features, $N_{\text{features+time}} = 31$.

### §3.3 Per-Bar Features

Let $\lbrace c_i, h_i, l_i, o_i, v_i\rbrace$ denote the close, high, low, open, volume of bar $i$. Simple returns are precomputed as

$$
r_i = \frac{c_i}{c_{i-1}} - 1, \quad r_0 = 0
$$

(`mtf.rs:138`). On-balance volume is

$$
\text{OBV}_i = \text{OBV}_{i-1} + \text{sgn}(r_i) \cdot v_i
$$

(`mtf.rs:139-142`) with $\text{sgn}(0) = 0$. The two EMAs use the standard recursive form

$$
\text{EMA}^{(w)}_i = \alpha_w c_i + (1 - \alpha_w) \text{EMA}^{(w)}_{i-1}, \quad \alpha_w = \frac{2}{w + 1}
$$

with $w \in \lbrace S, M\rbrace$ (`mtf.rs:130-145`).

**§3.3.1 Returns (MTF, 3 features).** The MTF returns are log-style ratios over each scale:

$$
\text{ret}^{(w)}_i = \frac{c_i}{c_{i-w}} - 1, \quad w \in \lbrace S, M, L\rbrace
$$

(`mtf.rs:159-161`). Note these are simple (arithmetic) returns over a $w$-bar lookback, not log returns - preserved as such for direct comparability with the threshold-based labels in `labels.rs:22-27`.

**§3.3.2 RSI (MTF, 3 features).** Normalized RSI (`indicators.rs:97-113`) is computed from precomputed returns rather than from prices. Over $[i+1-w \, .. \, i]$, define $n = w$ and

$$
G_w = \frac{1}{n} \sum_{j=i+1-w}^{i} \max(r_j, 0), \quad L_w = \frac{1}{n} \sum_{j=i+1-w}^{i} \max(-r_j, 0)
$$

The relative strength is $\text{RS}_w = G_w / L_w$. The classical Wilder RSI is $100 - 100/(1 + \text{RS})$; Prime's variant is normalized to $[0, 1]$:

$$
\text{RSI}^{(w)}_i = 1 - \frac{1}{1 + \text{RS}_w}
$$

with the limit cases $L_w < 10^{-15} \Rightarrow \text{RSI} = 1$ and $n = 0 \Rightarrow \text{RSI} = 0.5$. Two deviations from textbook Wilder: (i) simple mean rather than Wilder smoothing (a $1/n$ FIR instead of an EMA-equivalent), keeping the indicator strictly windowed; (ii) the $[0, 1]$ range matches the scale of the other bounded MTF features (ER, autocorr, ADX), aiding GBM split-gain comparability.

**§3.3.3 Parkinson Volatility (MTF, 3 features).** Wraps `nxr_sdk::stats::parkinson_vol` (`indicators.rs:33-36`). The Parkinson estimator (full derivation in §A.1 of A1) is

$$
\hat{\sigma}_P^{(w)}(i) = \sqrt{\frac{1}{4 w \ln 2} \sum_{j=i-w}^{i} \left(\ln \frac{h_j}{l_j}\right)^2}
$$

over `[i-window..=i]` (note this window is $w + 1$ samples wide, off-by-one relative to other indicators - see `indicators.rs:34`). $\hat{\sigma}_P$ has variance approximately $5\times$ lower than close-to-close realized vol on the same data, exploiting the bar's intra-period range.

**§3.3.4 Efficiency Ratio (MTF, 3 features).** Kaufman's efficiency ratio (`indicators.rs:41-49`) over the inclusive window $[i-w \, .. \, i]$:

$$
\text{ER}^{(w)}_i = \frac{|c_i - c_{i-w}|}{\sum_{j=i-w+1}^{i} |c_j - c_{j-1}|}
$$

Range $[0, 1]$. The numerator is net signed displacement; the denominator is total path length. $\text{ER} = 1$ iff the path is monotone (trend); $\text{ER} \to 0$ as the path becomes increasingly choppy. The implementation guards division with a $10^{-12}$ floor, returning $0$ on collapse.

**§3.3.5 ADX (MTF, 3 features).** A simplified-aggregation Average Directional Index (`indicators.rs:53-76`). For each $j$ in the window, define directional movement

$$
\text{up}_j = h_j - h_{j-1}, \quad \text{dn}_j = l_{j-1} - l_j
$$

$$
\text{DM}^+_j = \begin{cases} \text{up}_j & \text{up}_j > \text{dn}_j \text{ and } \text{up}_j > 0 \\ 0 & \text{otherwise} \end{cases}
$$

$$
\text{DM}^-_j = \begin{cases} \text{dn}_j & \text{dn}_j > \text{up}_j \text{ and } \text{dn}_j > 0 \\ 0 & \text{otherwise} \end{cases}
$$

and true range

$$
\text{TR}_j = \max\left(h_j - l_j,\ |h_j - c_{j-1}|,\ |l_j - c_{j-1}|\right)
$$

Aggregating $\Sigma_w \text{DM}^+, \Sigma_w \text{DM}^-, \Sigma_w \text{TR}$ over $[i+1-w \, .. \, i]$:

$$
\text{DI}^+_w = \frac{\Sigma_w \text{DM}^+}{\Sigma_w \text{TR}}, \quad \text{DI}^-_w = \frac{\Sigma_w \text{DM}^-}{\Sigma_w \text{TR}}
$$

$$
\text{ADX}^{(w)}_i = \text{clamp}\left(\frac{|\text{DI}^+_w - \text{DI}^-_w|}{\text{DI}^+_w + \text{DI}^-_w},\ 0,\ 1\right)
$$

Deviation from classical Wilder ADX: no Wilder smoothing of TR or DI series, and no second-stage DX smoothing. This is the **directional movement index** (DX in $[0, 1]$) computed on raw window sums, not the smoothed ADX. The simplification is intentional - Wilder smoothing introduces an effective EMA-equivalent lookback of $\sim 2w$, which would scramble the MTF scale separation.

**§3.3.6 Lag-1 Autocorrelation (MTF, 3 features).** Lag-1 ACF of returns (`indicators.rs:81-86`) over the window $[i+1-w \, .. \, i]$:

$$
\rho_1^{(w)}(i) = \frac{\sum_{j} (r_j - \bar{r})(r_{j-1} - \bar{r})}{\sum_{j} (r_j - \bar{r})^2}
$$

clamped to $[-1, 1]$, with a min-length gate of 4 (returns $0$ otherwise). Positive $\rho_1$ indicates short-term momentum (returns auto-reinforce); negative indicates short-term reversal (mean-reverting microstructure). In near-Gaussian DIB return space, $\rho_1$ should be close to zero in expectation; sustained deviations indicate non-stationarity.

**§3.3.7 Z-Score (MTF, 3 features).** Standardized price deviation from rolling SMA (`indicators.rs:91-94`):

$$
z^{(w)}_i = \text{clamp}\left(\frac{c_i - \bar{c}_w}{s_{c,w}},\ -4,\ +4\right), \quad \bar{c}_w = \frac{1}{w} \sum_{j=i+1-w}^{i} c_j
$$

with $s_{c,w}$ the sample standard deviation. Clamping at $\pm 4$ prevents catastrophic outliers (e.g., flash crashes) from dominating GBM split gain on a single sample. $z = 0$ when $s_{c,w} < 10^{-12}$.

**§3.3.8 Cross-Scale Regime Ratios (2 features).** Computed inline (`mtf.rs:195-197`):

$$
\text{vol-regime}_i = \begin{cases} \hat{\sigma}_P^{(S)} / \hat{\sigma}_P^{(L)} & \hat{\sigma}_P^{(L)} > 10^{-12} \\ 1 & \text{otherwise} \end{cases}
$$

$$
\text{er-regime}_i = \begin{cases} \text{ER}^{(S)} / \text{ER}^{(L)} & \text{ER}^{(L)} > 10^{-6} \\ 1 & \text{otherwise} \end{cases}
$$

Both are scale-invariant ratios: $\text{vol-regime} > 1$ indicates short-term volatility expansion relative to the long-term baseline (Δσ regime shift), while $\text{er-regime} > 1$ indicates short-term trendiness exceeding the long-term mean. Together they encode a regime quadrant - high/low vol × trend/chop - without exposing absolute levels (which the underlying MTF features already provide).

**§3.3.9 Hurst Exponent (1 feature).** Rescaled-range estimator over the long window (`indicators.rs:117-120`, `mtf.rs:200`):

$$
H_i = H_{\text{R/S}}(r_{i+1-L} \, .. \, r_i)
$$

Hurst $H \in [0, 1]$ classifies the return process: $H = 0.5$ for Brownian (no memory), $H > 0.5$ for persistent (trending), $H < 0.5$ for anti-persistent (mean-reverting). Implementation delegates to `nxr_sdk::stats::hurst_rs`. The L-scale lookback is required for stability; using S would yield estimator variance that swamps the signal.

**§3.3.10 EMA Cross (1 feature).** Normalized short/medium EMA divergence (`mtf.rs:201`):

$$
\text{ema-cross}_i = \begin{cases} \dfrac{\text{EMA}^S_i}{\text{EMA}^M_i} - 1 & \text{EMA}^M_i > 10^{-10} \\ 0 & \text{otherwise} \end{cases}
$$

This is the scale-invariant analog of the classical MACD-line "short minus long EMA," expressed as a fractional deviation rather than a price-difference. Scale invariance matters because the GBM consumes features across instruments with vastly different price levels (e.g., BTC at $10^4$, EUR/USD at $10^0$).

**§3.3.11 Price Acceleration (1 feature).** Discrete second-difference of S-scale returns (`mtf.rs:202-204`):

$$
\text{price-accel}_i = \underbrace{\left(\frac{c_i}{c_{i-S}} - 1\right)}_{r^{(S)}_i \text{ (current)}} - \underbrace{\left(\frac{c_{i-S}}{c_{i-2S}} - 1\right)}_{r^{(S)}_{i-S} \text{ (previous)}}
$$

Positive values indicate momentum accelerating; negative values indicate decelerating or reversing. This is the **discrete second derivative of log-price** in disguise: $\Delta r^{(S)} \approx \Delta^2 \ln c$ for small returns, providing a signal orthogonal to the level $r^{(S)}_i$ itself.

**§3.3.12 OBV Slope (1 feature).** Volume-derived trend indicator (`mtf.rs:205-207`):

$$
\text{obv-slope}_i = \begin{cases} \dfrac{\beta_{\text{OBV},S}(i)}{\bar{v}_{S}(i)} & \bar{v}_S(i) > 10^{-10} \\ 0 & \text{otherwise} \end{cases}
$$

where $\beta_{\text{OBV},S}(i)$ is the OLS slope (`lin_reg_slope`, `indicators.rs:125-128`) of $\text{OBV}_j$ regressed on time index $j$ over $[i+1-S \, .. \, i]$, and $\bar{v}_S(i) = \frac{1}{S}\sum_{j=i+1-S}^{i} v_j$ is the average volume. The normalization by mean volume yields a dimensionless quantity invariant to instrument-level volume scale - analogous to the EMA-cross normalization. Positive slope means cumulative signed volume is rising (accumulation); negative means distribution.

**§3.3.13 Close Location Value (1 feature).** Williams' CLV (`mtf.rs:208-213`):

$$
\text{CLV}_i = \begin{cases} \dfrac{(c_i - l_i) - (h_i - c_i)}{h_i - l_i} = \dfrac{2 c_i - h_i - l_i}{h_i - l_i} & h_i - l_i > 10^{-10} \\ 0 & \text{otherwise} \end{cases}
$$

Range $[-1, +1]$. $\text{CLV} = +1$ iff $c_i = h_i$ (close at high of bar, bullish closing pressure); $\text{CLV} = -1$ iff $c_i = l_i$. This is a single-bar microstructure feature - it does not require a window - and is the basic building block of the Accumulation/Distribution Line and Chaikin Money Flow.

**§3.3.14 Vol-of-Vol (1 feature).** Coefficient of variation of per-bar Parkinson sigma estimates over the medium window (`mtf.rs:215-234`):

$$
\eta_j = \frac{|\ln(h_j / l_j)|}{2 \sqrt{\ln 2}} = \frac{|\ln(h_j / l_j)|}{1.6651}
$$

is the **single-bar Parkinson sigma estimator** (Parkinson 1980, full derivation in A1). Over the M-window:

$$
\mu_\eta = \frac{1}{M} \sum_{j=i+1-M}^{i} \eta_j, \quad \sigma_\eta = \sqrt{\max\left(\frac{1}{M}\sum_j \eta_j^2 - \mu_\eta^2,\ 0\right)}
$$

$$
\text{vol-of-vol}_i = \begin{cases} \text{clamp}(\sigma_\eta / \mu_\eta,\ 0,\ 4) & \mu_\eta > 10^{-12} \\ 0 & \text{otherwise} \end{cases}
$$

This is dispersion of volatility-of-the-vol-process - high values indicate volatility regime instability (volatility clustering breakdown or regime transition), low values indicate a stable vol regime. The clamp at 4 prevents extreme outliers from poisoning splits.

**§3.3.15 Time Features (optional, 2 features).** When `with_time_features = true` (`mtf.rs:237-242`), Fourier encoding of time-of-day:

$$
\phi_i = \frac{(\lfloor t_i / 1000 \rfloor \mod 86400)}{86400}, \quad \text{time-sin}_i = \sin(2\pi \phi_i), \quad \text{time-cos}_i = \cos(2\pi \phi_i)
$$

where $t_i$ is the bar's close timestamp in milliseconds. The sin/cos pair preserves cyclic topology - midnight is a continuous point, not a discontinuity. **Disabled by default** since information bars already absorb intraday seasonality; enabling adds risk of fitting calendar artifacts that fail to generalize across regime shifts.

### §3.4 Cross-Asset Features

[`crates/ml/src/cross_features.rs`](file:///Users/derpa/Work/btr/prime/crates/ml/src/cross_features.rs) defines 8 features per alt asset (`FEATURES_PER_ALT = 8`, line 13) computed from a primary–alt bar pair. Alignment is LOCF (Last Observation Carried Forward, lines 226-237): for each primary bar at timestamp $t^P_i$, the latest alt index $\text{ai}(i)$ satisfies $t^A_{\text{ai}(i)} \leq t^P_i$. Log-prices and log-returns are computed on the aligned series:

$$
p^P_i = \ln c^P_i, \quad p^A_i = \ln c^A_{\text{ai}(i)}, \quad r^P_i = p^P_i - p^P_{i-1}, \quad r^A_i = p^A_i - p^A_{i-1}
$$

The 8 features use windows $(S, M, L) = (20, 60, 200)$ (fixed at default scales, line 15-17).

**§3.4.1 Cointegration Z-Score (`coint_z`).** Engle–Granger style rolling cointegration (lines 96-118). Rolling OLS on the L-window:

$$
p^P_j = \alpha + \beta p^A_j + \varepsilon_j, \quad j \in [i+1-L, i]
$$

Spread series and z-score:

$$
s_j = p^P_j - \beta p^A_j - \alpha, \quad \mu_s = \frac{1}{L}\sum_j s_j, \quad \sigma_s = \sqrt{\overline{s^2} - \mu_s^2}
$$

$$
\text{coint-z}_i = \frac{s_i - \mu_s}{\sigma_s} \mathbb{1}\lbrace \sigma_s > 10^{-12}\rbrace
$$

Rationale: signals statistical mispricing relative to long-run equilibrium; |z| > 2 is a standard threshold for pairs trading entry.

**§3.4.2 Spread Half-Life (`spread_hl`).** AR(1) on the spread (lines 120-137):

$$
\Delta s_j = \rho s_{j-1} + \eta_j, \quad \hat{\rho} = \frac{\sum_j \Delta s_j \cdot s_{j-1}}{\sum_j s_{j-1}^2}
$$

Half-life from the AR(1) decay:

$$
\text{spread-hl}_i = \begin{cases} \text{clamp}\left(\dfrac{-\ln 2}{\ln(1 + \hat{\rho})},\ 1,\ 200\right) & \hat{\rho} < -10^{-6} \\ 200 & \text{otherwise} \end{cases}
$$

Short half-life ⇒ fast mean reversion (tradeable); long half-life ⇒ structural break or no reversion.

**§3.4.3 Rolling Beta (`beta`).** OLS of primary returns on alt returns over M-window (lines 140-143):

$$
\hat{\beta}_i = \frac{\sum_j r^P_j r^A_j - M \bar{r}^P \bar{r}^A}{\sum_j (r^A_j)^2 - M (\bar{r}^A)^2}
$$

Tracks time-varying hedge ratio.

**§3.4.4 Residual Momentum (`res_mom`).** Beta-adjusted residual returns, momentum over S, normalized by full M-window std (lines 145-151):

$$
\varepsilon_j = r^P_j - \hat{\beta}_i r^A_j, \quad \text{res-mom}_i = \frac{\sum_{j = i+1-S}^{i} \varepsilon_j}{\text{std}(\lbrace \varepsilon_j\rbrace_{j=i+1-M}^{i})}
$$

Isolates idiosyncratic primary momentum after hedging out alt exposure.

**§3.4.5 DCC Correlation (`dcc`).** Simplified DCC-GARCH (Engle 2002) with EWMA variance and exponential quasi-correlation update (lines 154-163). With $\alpha = 2/(S+1)$ and decay $\lambda = 0.94$ (RiskMetrics standard):

$$
\sigma^2_{a,i} = (1-\alpha) \sigma^2_{a,i-1} + \alpha (r^P_i)^2, \quad \sigma^2_{b,i} = (1-\alpha) \sigma^2_{b,i-1} + \alpha (r^A_i)^2
$$

Standardized residuals $u_a = r^P_i / \sigma_{a,i}$, $u_b = r^A_i / \sigma_{b,i}$. Quasi-correlation:

$$
q_{ab,i} = (1-\lambda) u_a u_b + \lambda q_{ab,i-1}
$$

$$
q_{aa,i} = (1-\lambda) u_a^2 + \lambda q_{aa,i-1}, \quad q_{bb,i} = (1-\lambda) u_b^2 + \lambda q_{bb,i-1}
$$

$$
\text{dcc}_i = \text{clamp}\left(\frac{q_{ab,i}}{\sqrt{q_{aa,i} q_{bb,i}}},\ -1,\ +1\right)
$$

Adaptive correlation that responds to regime shifts faster than rolling-window Pearson; state is warmed up during the L-bar lookback before features begin emitting (lines 80-93).

**§3.4.6 Volume Sync (`vol_sync`).** Z-score of log volume ratio over M-window (lines 165-174):

$$
\ell_j = \ln\left(\frac{v^P_j}{v^A_j + 10^{-12}}\right), \quad \text{vol-sync}_i = \frac{\ell_i - \bar{\ell}}{\text{std}(\ell)}
$$

Detects cross-asset volume divergence; primary surging on volume while alt is quiet (or vice versa) ⇒ idiosyncratic flow.

**§3.4.7 Granger Causality Proxy (`granger`).** Incremental $R^2$ from adding alt-lag to the primary-lag-only AR model (lines 176-190). Restricted model:

$$
r^P_t = a + b \, r^P_{t-1} + u_t, \quad \text{SSR}_R
$$

Unrestricted model:

$$
r^P_t = a + b_1 r^P_{t-1} + b_2 r^A_{t-1} + u_t, \quad \text{SSR}_U
$$

$$
\text{granger}_i = \text{clamp}\left(1 - \frac{\text{SSR}_U}{\text{SSR}_R},\ 0,\ 1\right)
$$

This is the population $\Delta R^2$ from including the alt lag; a true Granger test would require an F-statistic on $(\text{SSR}_R - \text{SSR}_U)/\text{SSR}_U$, but as a GBM feature the unnormalized $\Delta R^2$ is preferable - it is monotone-equivalent under split selection and avoids needless degrees-of-freedom bookkeeping.

**§3.4.8 Vol-of-Vol Correlation (`vol_corr`).** Pearson correlation of vol-return series (lines 192-217). Compute per-bar S-window realized vol $\sigma^P_j, \sigma^A_j$, then their log-returns:

$$
\tilde{r}^P_j = \ln(\sigma^P_j / \sigma^P_{j-1}), \quad \tilde{r}^A_j = \ln(\sigma^A_j / \sigma^A_{j-1})
$$

$$
\text{vol-corr}_i = \text{corr}(\lbrace \tilde{r}^P_j\rbrace , \lbrace \tilde{r}^A_j\rbrace )
$$

where the Pearson coefficient is computed in the helper `pearson` (lines 255-266) with the standard numerically-stable form, requiring $n \geq 3$.

These 8 cross-asset features are concatenated **per alt asset** to the 29 MTF + 12 renko + 12 microstructure single-asset features (53 baseline) when the trainer is configured for multi-asset context. Rationale: capture lead/lag, regime synchronization, and statistical arbitrage signals that are invisible to single-asset features.

### §3.5 Indicator Math

All indicator helpers in `indicators.rs` are pure functions over precomputed arrays. Key implementation details beyond the formulas already given in §3.3:

- **Mean and std (`mean_slice`, `sma_std`, lines 14-27).** Window $[i+1-w \, .. \, i]$, falls back to $(c_i, 0)$ for $n < 2$.
- **Parkinson (`parkinson_vol`, lines 33-36).** Window $[i-w \, .. \, i]$ (note: $w+1$ samples - distinct from the standard $w$-sample windowing of other indicators; this is intentional, since Parkinson is a sum-of-squares quantity where one extra sample materially changes the estimator's bias).
- **ER, ADX, ACF, z-score, RSI, Hurst, OLS slope.** All as defined in §3.3.4–§3.3.9, with explicit numerical guards (`1e-12` for division floors, $[-4, 4]$ or $[-1, 1]$ clamps where indicated).

The OLS slope (`lin_reg_slope`, `indicators.rs:125-128`) delegates to `nxr_sdk::stats::ols_slope`; the closed-form for time-indexed regression $y_j = \beta j + \alpha + \epsilon_j$ over a window of length $w$ is

$$
\hat{\beta} = \frac{\sum_j (j - \bar{j})(y_j - \bar{y})}{\sum_j (j - \bar{j})^2}
$$

which is what `obv_slope` consumes.

### §3.6 Microstructure Features

Block of **12** TICK-derived features appended unconditionally to the matrix at `crates/bin/src/ml.rs:585-593`, computed in `crates/bin/src/ml.rs:411-502`. Z-scores use rolling window $L_\mu = 200$ for slow series (`rv`, `bipower`, `ofi`, `vol`) and $S_\mu = 20$ for fast series (`drift`, `maxr`, `spread`, `tick_eff`, `bar_dur`); regime ratios use EMA short = 20 vs long = 200 (the EMA helpers are `rolling_zscore` at `ml.rs:429-459` and `rolling_ema_ratio` at `ml.rs:461-475`). All TICK inputs are NXR-upstream fields populated by the bar generator (see [§01 Information Bars](./01.%20Information%20Bars.md) and NXR `mitch/impl/rust/src/bar.rs`).

**§3.6.1 RV / bipower / jump (3 features).** `realized_var` and `bipower_var` are NXR-precomputed sub-bar TICK aggregates; `jump_var` is the derived $\max(\text{realized}-\text{bipower}, 0)$ jump component.

$$
\text{micro-rv-z}_i = z_{L_\mu}(\text{realized\_var}_i), \quad \text{micro-bipower-z}_i = z_{L_\mu}(\text{bipower\_var}_i)
$$

$$
\text{micro-jump-regime}_i = \frac{\text{EMA}_{20}(\text{jump\_var})}{\text{EMA}_{200}(\text{jump\_var})} - 1
$$

(`ml.rs:411-413,477-478,488`). Rationale: RV is the standard high-frequency variance proxy; bipower variation is the jump-robust counterpart; their difference isolates jump-component variance. The regime ratio fires when the jump fraction inflates relative to the recent baseline - a leading indicator for tail events.

**§3.6.2 Order-flow imbalance (2 features).** `vol_imbalance` = NXR sub-bar signed-volume imbalance (buy-initiated minus sell-initiated, in `vbid`/`vask` units).

$$
\text{micro-ofi-z}_i = z_{L_\mu}(\text{vol\_imbalance}_i), \quad \text{micro-ofi-regime}_i = \frac{\text{EMA}_{20}(\text{ofi})}{\text{EMA}_{200}(\text{ofi})} - 1
$$

(`ml.rs:414,479,487`). The z-score captures instantaneous order-flow extremes; the regime ratio captures sustained directional flow.

**§3.6.3 Volume (1 feature).** Bar total volume (`Bar::total_volume()` = `vbid + vask`), log-compressed:

$$
\text{micro-vol-z}_i = z_{L_\mu}\big(\ln(1 + \text{total\_volume}_i)\big)
$$

(`ml.rs:415,480`). This is the only microstructure feature whose data source is **BAR-VOL** rather than TICK - the prefix `micro_` is retained for column-order stability but is mildly misleading; the same flow information is partly captured in `obv_slope` (§3.3.12).

**§3.6.4 Drift / max-abs-return / spread (3 features, fast window).** All TICK-derived NXR fields:

$$
\text{micro-drift-z}_i = z_{S_\mu}(\text{drift}_i), \quad \text{micro-maxr-z}_i = z_{S_\mu}(\text{max\_abs\_return}_i), \quad \text{micro-spread-z}_i = z_{S_\mu}(\text{avg\_spread\_bps}_i)
$$

(`ml.rs:416-418,481-483`). `drift` is NXR's local-drift estimate from the sub-bar tick stream (orthogonal to bar `close-open`); `max_abs_return` is the largest single-tick log-return within the bar (intra-bar tail signal); `avg_spread_bps` is the time-weighted quoted spread (liquidity proxy).

**§3.6.5 Tick efficiency (1 feature, hybrid TICK + BAR-OHLC).** Mixes the bar's $(o_i, c_i)$ with the TICK metadata `tick_count`:

$$
\text{tick\_eff}_i = \frac{|c_i - o_i|}{c_i \cdot \text{tick\_count}_i}, \quad \text{micro-tick-eff-z}_i = z_{S_\mu}(\text{tick\_eff}_i)
$$

with the guard $|c_i| > 10^{-10}$ and $\text{tick\_count}_i > 0$ (`ml.rs:419-426,484`). Interpretation: small values mean many ticks moved little net price (slow / mean-reverting microstructure); large values mean few ticks moved a lot of price (informed-flow / momentum microstructure).

**§3.6.6 Bar duration (1 feature, BAR-meta).** Derived from `open_ts/close_ts` via `Bar::duration_ms()`:

$$
\text{micro-bar-dur-z}_i = z_{S_\mu}\big(\ln(\max(\text{duration\_ms}_i, 1))\big)
$$

(`ml.rs:427,485`). Inverse information-bar speed: short duration = fast trade arrival = high information rate. Inversely correlated with `tick_eff` at $\rho \approx -0.7$ - see §3.7 redundancy notes.

### §3.7 Feature Economy & Redundancy

Phase 42Y identified redundancy at 61; Phase 42Z (live 2026-05-17) implemented the 4-expert panel-consensus 32-feature compression and promoted it to production. The 8 hard gates (§3.7.2) define the empirical-validation checklist that production CI must run before each model deploy. The 61-baseline path is retained as ablation baseline via `--baseline-features`.

The baseline (research / ablation) matrix is **61 features** at the 1-alt canonical configuration. The training window is $N_{\text{train}} = 5000$ bars (`prime/config.yml`) with horizon $H = 20$ bars. The original Phase 42Y audit cited $N_{\text{eff}} \approx N_{\text{train}} / H = 250$ (Politis non-overlap) and recommended a **28-feature** compressed shortlist on the basis of Harrell's "$\geq 10$ samples per feature" rule. A subsequent **4-expert panel review** (Phase 42Y; statistics, microstructure, GBM-ML engineering, risk-adversarial) returned a **unanimous APPROVE-WITH-MODS** verdict with a raised floor: the consensus shortlist is **$\sim 32$ features (range 30-35)**, achieved via *residualisation* of higher-$\rho$ partners rather than strict drop. See §3.7.1 for verdicts and §3.7.2 for hard pre-deploy gates.

**Sample-size correction.** Expert 1 re-derived $N_{\text{eff}}$ under Politis-Romano block-bootstrap with block length $\ell^\star \sim H^{1/3} N^{1/3} \approx 46$, yielding $N_{\text{block}} \approx 5000/46 \approx 109$ independent blocks. Combined with a $40$-bar inter-fold embargo, the realistic working range is $N_{\text{eff}} \in [110, 200]$, anchored near **150** for budgeting decisions (not 250). Expert 3 further notes that perpetual_booster v2.1 exposes none of the standard regularisers (no $\lambda$, no $\alpha$, no `colsample_bytree`, no explicit shrinkage); effective degrees of freedom $\text{df} \approx 30 - 80$ leaves (Lin-Jeon 2006; ESL §10.11) are bounded by the budget gate, *largely independent of input dimension* once $d \gg 30$. At $N_{\text{eff}} = 150$ and $\text{df} \approx 50$, the samples-per-DoF ratio is $\approx 3$ — **already underpowered at any feature count**. The 32 vs 61 question is downstream of the more serious power problem; the real fix is to raise `train_bars` to 8000-10000 (data permitting) OR lower `budget` to $\sim 0.15$ to bring $\text{df}$ down to $\sim 30$.

**Residualisation principle.** Trees do not suffer from multicollinearity the way OLS does (split-gain dilution, not variance inflation; Strobl 2007). Dropping the higher-$\rho$ partner outright destroys orthogonal residual signal that lives only in the *difference series*. For each suspected pair, the panel recommends constructing a residualised feature
$$x_{\text{resid}} = x - \hat{\beta}\,y, \qquad \hat{\beta} = \frac{\widehat{\mathrm{Cov}}(x,y)}{\widehat{\mathrm{Var}}(y)},$$
fit on a rolling training window (per-fold, not global). For the bipower / RV pair, the BNS jump-variance decomposition is preferred to a linear residual:
$$\text{jump\_var}_t = \max(\text{rv}_t - \text{bipower}_t,\ 0), \qquad \text{jump\_var\_z}_t = z_{200}(\text{jump\_var}_t).$$

**Consensus drops, rescues, and new derived features.**

| Audit verdict | Panel verdict | Rationale |
|---|---|---|
| DROP `micro_bipower_z` | RESCUE as `jump_var_z = z_{200}(\max(\text{rv}-\text{bipower},\,0))` | BNS decomposition; preserves bar-level jump signal (Experts 1, 2, 4) |
| DROP `micro_drift_z` | RESCUE as `drift_resid_z` (residualise against `ret_s`) | Sub-bar OLS drift orthogonal to bar-close return (Experts 1, 2) |
| DROP `micro_maxr_z` | RESCUE raw | Aït-Sahalia–Jacod jump diagnostic distinct from Parkinson range (Experts 2, 4) |

- **MTF per-bar (22):** drop `ret_m`, `rsi_s`, `pvol_m`, `er_m`, `autocorr_m`, `adx_s`, `adx_m` — keep `adx_l` (panel rescue). Net $29 - 7 = 22$.
- **Renko-tracker (2):** keep `renko_wide_rate` and `renko_wide_ema_dir`; cut the other 10.
- **Cross-asset (3 per alt):** keep `res_mom`, `dcc`, plus one of $\lbrace$`coint_z`, `beta`$\rbrace$ as regime indicator (Expert 4 — floor-railing is information); drop `granger` (Expert 1), drop `spread_hl`, `vol_sync`, `vol_corr`.
- **Microstructure (5, not 2):** `micro_ofi_z`, `micro_ofi_regime`, `drift_resid_z` (new derived), `micro_maxr_z`, `jump_var_z` (new derived).
- **Optional 1-2 rescues** subject to per-fold permutation gate: `micro_spread_z` (tail canary), `renko_wide_compress` (price-anchored coiled-spring; Expert 4), `rsi_s` (SO-mode short-squeeze sensitivity; Expert 4).

Total: $21 + 2 + 3 + 5 = \mathbf{31}$ baseline, $+1$ to $+3$ conditional rescues = **30 to 35**. Median target $\sim 32$.

**New derived features required (not in current `crates/ml/src/features/`):**

1. `jump_var_z` = $z_{200}\big(\max(\text{rv}_t - \text{bipower}_t,\,0)\big)$ — BNS jump-variance z-score.
2. `drift_resid_z` = $z_{200}\big(\text{drift}_t - \hat{\beta}_{\text{drift}|\text{ret\_s}}\cdot \text{ret\_s}_t\big)$ — sub-bar drift residualised against bar-close return, $\hat{\beta}$ refit per walk-forward fold.
3. `tick_speed` = $\max(\text{tick\_eff\_z},\ \text{bar\_dur\_z})$ — consolidation of the duration/efficiency duo.

**Important caveats.**

1. **Per-fold permutation importance and SHAP** are MANDATORY in CI pre-deploy (Experts 1, 2, 3, 4 unanimous). Theoretical $\rho$ priors are *necessary but not sufficient*. See §3.7.2.
2. **§6.3 of [§10 Open Questions](./10.%20Open%20Questions.md) (per-fold feature selection plumbing) MUST CLOSE FIRST** as part of the gate-3.7.2 §8 pre-deploy checklist. Until selection is inside the walk-forward loop, importance rankings are leaky.
3. **Production is now 32 features (Phase 42Z).** Compression deployed 2026-05-17 per 4-expert panel sign-off. The 8 hard gates in §3.7.2 remain mandatory CI promotion gates for each model deploy. The 61-baseline is retained for ablation studies (`--baseline-features`).
4. **Sample-power bottleneck is independent of feature count.** At $N_{\text{eff}} \approx 150$ and $\text{df} \approx 50$, samples-per-DoF $\approx 3$ even at the compressed shortlist. Cutting features is necessary regularisation but not sufficient power; the structural fix is `train_bars` $\uparrow$ or `budget` $\downarrow$.

### §3.7.2 Hard Gates Required Before Production Deploy

The compressed 32-feature set is the production target as of Phase 42Z (2026-05-17). The following eight gates remain non-negotiable preconditions — they are the **CI promotion gates** run before each model deploy (each retrain, each parameter change). Aggregate Sharpe-within-10% is necessary but NOT sufficient.

1. **Per-fold permutation importance, LO and SO labels separated.** Reject any drop whose removal costs **SO Sharpe $> 5\%$** even if aggregate (LO+SO) is flat. Asymmetric jump-detection cost falls disproportionately on SO (Expert 4).
2. **Tail-event subset CV.** Restrict evaluation to bars in the top-5% of $|\text{ret\_l}|$. Compressed-set Sharpe must be within **$5\%$** of the 61-baseline on the tail subset (tighter than the 10% aggregate gate).
3. **Max-loss-month delta.** Worst calendar-month return on the compressed set must not be worse than worst-month on 61 by more than **25 bps**.
4. **CL range stability.** $\Theta(w;\,H)$ time-in-range predictions on a held-out fold must be within **$\pm 15\%$** of 61-baseline predictions, evaluated separately on **low-vol** and **tail-vol** subsets.
5. **Drawdown-distribution KS test.** Per-fold drawdown distributions (compressed vs 61) must be KS-indistinguishable at $p > 0.05$.
6. **Sensor-fault simulation.** Inject sign-flip on the `vol_imbalance` field. The 61-baseline must degrade $< 30\%$ Sharpe; the compressed set must degrade $< 50\%$. If $> 50\%$, the redundancy floor is too thin and `ofi_z` becomes a single point of failure.
7. **Stale-feed simulation.** Freeze the Parkinson estimator at last value for 100 bars. SL/CL outputs must flag uncertainty (widen, suppress, or surface a $\sigma$-stale signal); silent narrowing of CL range half-width is rejection-grade.
8. **Per-fold feature selection plumbing shipped (close [§10 §6.3](./10.%20Open%20Questions.md)) BEFORE any pruning lands in production.** Global selection is leaky; compressed-set evaluation against a leaky baseline is invalid.

**No-go conditions (Expert 3, verbatim):** ship without permutation-importance corroboration $\rightarrow$ reject. Ship without per-fold selection refactor $\rightarrow$ reject. Ship without stress-window validation $\rightarrow$ reject.

### §3.10 Derived Features (Phase 42Z)

Three new derived features ship with the Phase 42Z compression. Each replaces a higher-correlation parent group via residualisation or BNS-style decomposition (see §3.7 rationale). Code references will be filled after impl lands; placeholders below.

**§3.10.1 `micro_jump_var_z`.** BNS jump-variance z-score replacing `micro_bipower_z` and `micro_jump_regime`.

$$
\text{jump\_var}_t = \max(\text{realized\_var}_t - \text{bipower\_var}_t,\ 0), \qquad \text{micro\_jump\_var\_z}_t = z_L(\text{jump\_var}_t)
$$

with window $L = 200$. Preserves jump signal without retaining both correlated parents. Code ref: `crates/bin/src/ml.rs:LINE` (TBD).

**§3.10.2 `micro_drift_resid_z`.** Sub-bar drift residualised against the bar-close return.

$$
\text{drift\_resid}_t = \text{drift}_t - \hat{\beta}\cdot \text{ret\_s}_t, \qquad \hat{\beta} = \frac{\widehat{\mathrm{Cov}}(\text{drift},\,\text{ret\_s})}{\widehat{\mathrm{Var}}(\text{ret\_s})}
$$

$$
\text{micro\_drift\_resid\_z}_t = z_S(\text{drift\_resid}_t)
$$

with $S = 20$. $\hat{\beta}$ is estimated series-wide as $\mathrm{Cov}(\text{drift},\,\text{ret\_s}) / \mathrm{Var}(\text{ret\_s})$; rolling $L=200$ refit is roadmapped in [§10 §6.7](./10.%20Open%20Questions.md). Recovers the sub-bar drift component not explained by bar-close returns. Code ref: `crates/bin/src/ml.rs:LINE` (TBD).

**§3.10.3 `tick_speed_z`.** Consolidated tick-speed metric replacing the redundant `tick_eff_z` / `bar_dur_z` pair.

$$
\text{tick\_speed}_t = \frac{|c_t - o_t|}{c_t \cdot \text{tick\_count}_t \cdot \ln(\max(\text{bar\_dur\_ms}_t,\ 1))}, \qquad \text{tick\_speed\_z}_t = z_S(\text{tick\_speed}_t)
$$

with $S = 20$, guard $|c_t| > 10^{-10}$ and $\text{tick\_count}_t > 0$. Combines information-arrival rate ($\text{tick\_count}/\text{bar\_dur}$) with per-tick price impact ($|c - o| / \text{tick\_count}$) into a single dimensionless quantity. Code ref: `crates/bin/src/ml.rs:LINE` (TBD).
