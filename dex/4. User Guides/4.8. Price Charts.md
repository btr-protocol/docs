# Price Charts

> Advanced charting with technical indicators and synthetic pairs

The BTR interface includes professional-grade price charts with real-time data, technical analysis indicators, and support for synthetic trading pairs.

---

## Chart Types

Choose from three visualization modes:

- **Candlesticks** - Traditional OHLC candles showing open, high, low, close
- **Bars** - OHLC bars for a more compact view
- **Line** - Simple closing price line for trend clarity

---

## Timeframes

Available intervals: **1m**, **5m**, **15m**, **1h**, **4h**, **1D**

Charts store only 1-minute candles on the backend and reconstruct higher timeframes on-demand for efficient storage.

## Synthetic Pairs

BTR supports synthetic trading pairs (e.g., BTC/ETH, SOL/AVAX) that don't exist as direct market pairs.

### How Synthetic Pairs Work

For a pair like **BTC/ETH**:
1. Fetch both **BTC/USDC** and **ETH/USDC** price feeds
2. Construct synthetic candles using **correlation-dampened extremes**
3. Display the computed ratio in real-time

### Synthetic OHLC Construction

The chart computes synthetic candles using a correlation-based approach:

```typescript
// Exact open/close from observed prices
open = btc.open / eth.open
close = btc.close / eth.close

// Compute candidate extremes from all possible combinations
candidateHigh = max(
  btc.high / eth.low,    // Max-range upper bound
  btc.high / eth.open,
  btc.high / eth.close,
  btc.open / eth.low,
  btc.close / eth.low
)

candidateLow = min(
  btc.low / eth.high,    // Max-range lower bound
  btc.low / eth.open,
  btc.low / eth.close,
  btc.open / eth.high,
  btc.close / eth.high
)

// Dampen wicks based on correlation (ρ)
// When BTC and ETH move together (high ρ), extreme divergences are unlikely
center = (open + close) / 2
dampFactor = 1 - 0.5 * ρ  // At ρ=0.85, dampFactor ≈ 0.575

high = center + (candidateHigh - center) * dampFactor
low = center + (candidateLow - center) * dampFactor
```

**Key insight:** When correlation (ρ) is high (e.g., 0.85 for BTC/ETH), the algorithm reduces wick size because simultaneous opposite extremes (BTC hitting high while ETH hits low in the same candle) are statistically unlikely.

**Default correlation:** 0.85 (typical for major crypto pairs)
**Rolling window:** 20 bars for dynamic correlation estimation

This produces realistic wicks without requiring M1 data aggregation on the client.

## Technical Indicators

BTR charts support a comprehensive suite of technical analysis tools using a **Multi-Timeframe (MTF)** approach for smoother signals.

### MTF Methodology

All indicators combine fast (10-period) and slow (20-period) calculations:

```
MTF_Value = (Fast_10 + Slow_20) / 2
```

This dual-timeframe average provides:
- **Smoother signals** - Less noise than single-period indicators
- **Better memory** - Captures both recent and longer-term trends
- **Reduced whipsaws** - More stable entry/exit signals

### Available Indicators

#### **EMA Trend** (Overlay)
- Dual exponential moving averages overlaid on price chart
- Default: EMA(10) and EMA(20)
- **Use case:** Trend direction and crossover signals

#### **EMACD** (Sub-pane)
- EMA divergence: `MTF_EMA - MA(MTF_EMA, 14)`
- Histogram showing momentum strength
- **Use case:** Identify momentum shifts and trend exhaustion

#### **RSIMA** (Sub-pane)
- MTF Relative Strength Index vs its moving average
- Shows RSI(10,20) and MA(14) of MTF_RSI
- **Use case:** Overbought/oversold conditions with confirmation

#### **RSIMACD** (Sub-pane)
- RSI divergence: `MTF_RSI - MA(MTF_RSI, 14)`
- Histogram centered at zero
- **Use case:** RSI momentum and reversal signals

#### **ADXMA** (Sub-pane)
- MTF Average Directional Index vs its moving average
- Shows ADX(10,20) and MA(14) of MTF_ADX
- **Use case:** Trend strength measurement (>25 = strong trend)

#### **ADXMACD** (Sub-pane)
- ADX divergence: `MTF_ADX - MA(MTF_ADX, 14)`
- Histogram showing trend strength momentum
- **Use case:** Identify strengthening or weakening trends

#### **SDEVMA** (Sub-pane)
- MTF Standard Deviation vs its moving average
- Shows SDEV(10,20) and MA(14) of MTF_SDEV
- **Use case:** Volatility analysis and breakout prediction

#### **SDEVMACD** (Sub-pane)
- Standard deviation divergence: `MTF_SDEV - MA(MTF_SDEV, 14)`
- Histogram showing volatility momentum
- **Use case:** Detect volatility expansion/contraction

### Customizing Indicators

All indicators accept three configurable parameters:

- **Fast period** (default: 10) - Short-term sensitivity
- **Slow period** (default: 20) - Long-term smoothing
- **Signal period** (default: 14) - MA baseline for divergence calculations

Click the **pencil icon** next to any indicator to adjust parameters.

### Divergence Indicators

Indicators ending in **"CD"** (EMACD, RSIMACD, ADXMACD, SDEVMACD) are **divergence-type** histograms:

- **Positive values** - Indicator above its baseline (bullish momentum)
- **Negative values** - Indicator below its baseline (bearish momentum)
- **Zero line** - Equilibrium reference

The chart supports professional drawing tools for technical analysis:

- **Horizontal/Vertical Lines** - Mark key price levels or time events

## Data Architecture

- **Storage:** Only M1 (1-minute) candles stored on backend
- **Reconstruction:** Higher timeframes built on-demand from M1 data
- **Gap filling:** Missing data automatically fetched from exchanges after downtime
- **Live updates:** WebSocket price streams with rate-limited updates
- **Synthetic feeds:** Real-time triangulation for non-native pairs

2. **Limit active indicators** - 2-3 indicators per chart for best performance

## Technical Notes

### Correlation Estimation

For synthetic pairs, the chart estimates correlation (ρ) using a rolling 20-bar window of log returns:

```
ρ = cov(log(base_t / base_{t-1}), log(quote_t / quote_{t-1})) /
    sqrt(var(base) * var(quote))
```

**Fallback:** Default ρ = 0.85 if insufficient data (<5 bars)

### Wick Dampening

High correlation reduces synthetic wick size by pulling extremes toward the candle center:

- **ρ = 0** (uncorrelated): Full range (100% of candidate wicks)
- **ρ = 0.85** (typical): ~58% of candidate range
- **ρ = 1** (perfect correlation): 50% of candidate range

This prevents unrealistic wicks from theoretical price combinations that rarely occur in practice.

## Troubleshooting

**Chart shows gaps after reconnection:**
- Gaps <1 minute ignored (noise)
- Larger gaps auto-filled from exchange history
- If persists, refresh page to force full data fetch

**Synthetic pair shows wrong data:**
- Verify both component feeds are available (check /health endpoint)
- Check if correlation calculation has sufficient data (≥5 bars)
- Try switching to a higher timeframe for more stable data

**Indicators not updating:**
- Ensure minimum data requirements met (indicators need ~20+ candles)
- Check if live price stream is connected (watch chart header)
- Try toggling indicator off/on to force recalculation
