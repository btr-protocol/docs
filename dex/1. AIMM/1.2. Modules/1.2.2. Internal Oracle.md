# Internal Oracle

> Self-updating TWAP oracle with dual-window price tracking and volatility EMAs

---

## 1. Overview

The Internal Oracle provides price feeds that update automatically on every swap. This eliminates external oracle dependencies for active pools while maintaining manipulation resistance through dual-window TWAPs.

> **Implementation note (Phase 42H)**: the oracle is **inherited directly by `Pool`** (no separate module / contract). The functions documented below are exposed on each `Pool` clone. The price-update accounting lives in `PoolOracle` library code, invoked from `Pool` swap paths.

**Key Features:**
- **Zero external calls**, Prices derived from swap activity
- **Dual TWAPs**, 5-minute (fast) and 1-hour (slow) windows
- **Volatility tracking**, Fast and slow EMAs for spread calculation
- **B64 encoding**, Gas-efficient storage
- **Same-block rejection**, Prevents flash loan manipulation

## 4. B64 Float Encoding

Compact 64-bit price representation:

```
[mantissa:52 | decimals:5 | exponent+bias:7]
```

- **Mantissa**: 52 bits (~15 decimal digits precision)
- **Decimals**: 5 bits (0-31 token decimals)
- **Exponent**: 7 bits with bias 64 (range: 10^-64 to 10^63)

**Decoding:**

$$p_{18} = M * 10^{e - 64 + d_t - d_s}$$

where:
- $M$ = mantissa
- $e$ = exponent
- $d_t$ = target decimals
- $d_s$ = stored decimals

See: `contracts/src/libraries/LibMaths.sol:62-140`

---

## 5. Accumulator Decimals

The `accDecimals` parameter controls accumulator precision vs. range tradeoff:

| Asset Class | accDecimals | Use Case |
|-------------|-------------|----------|
| Stablecoins | 6 | ~$1 prices, maximum range |
| ETH-like | 12 | ~$3k prices, balanced |
| BTC-like | 18 | ~$100k prices, full precision |

Lower `accDecimals` = more headroom for long-running accumulators.

## 6. Price Update Flow

### 6.1. On Every Swap

```solidity
function pushFeedInternal(
    address tokenA,
    address tokenB,
    uint64 priceA,    // Price of tokenA in base (B64)
    uint64 priceB     // Price of tokenB in base (B64)
) external
```

**Called internally by Core after swap execution.**

### 6.2. Update Algorithm

1. Skip same-block updates (manipulation resistance)
2. Update price accumulator with time-weighted contribution
3. Check if fast window elapsed → update fast TWAP offset
4. Check if slow window elapsed → update slow TWAP offset
5. Update volatility EMAs with price change
6. Store new spot price and timestamp

## 7. TWAP Calculation

### 7.1. Offset Encoding

TWAPs are stored as offsets from spot price (0.0001% units):

$$o = obrace{(p_T - p_s)}/{p_s}^{"relative price difference"}$$

where:
- $o$ = TWAP offset from spot price
- $p_T$ = TWAP
- $p_s$ = spot

[^1]: **Solidity implementation**: Returns `((twap - spot) * BPS_PRECISION) / spot` where `BPS_PRECISION = 1,000,000` (0.0001% precision). Positive offset means TWAP above spot. See `LibOracle.updateFastSlowSnapshots()`.

### 7.2. Offset Interpretation

| $o$ | Meaning |
|-----|---------|
| 0 | TWAP = Spot |
| +10000 | TWAP 1% above spot |
| -5000 | TWAP 0.5% below spot |

### 7.3. TWAP Reconstruction

$$p_T = p_s * (1 + o)$$

[^2]: **Solidity implementation**: `price = spot * (BPS_PRECISION + offset) / BPS_PRECISION` where `BPS_PRECISION = 1,000,000`. The offset is in 0.0001% units.

## 8. Volatility Tracking

> This section covers the **EMA mechanism** for tracking volatility. For how volatility is used as a pricing risk signal, see [Spread & Fees §3](/docs/1.1.4-spread-fees#3-spread-calculation).

### 8.1. EMA Update Formula

$$sigma' = sigma + alpha * (|Delta p| - sigma)$$

where:
- $sigma$ = current EMA
- $sigma'$ = updated EMA
- $alpha$ = smoothing factor
- $|Delta p|$ = absolute price change

[^3]: **Solidity implementation**: `newEMA = oldEMA + (alpha * (abs(priceDelta) - oldEMA)) / VOL_BASE` where `VOL_BASE = 1,000,000` (0.0001% precision). The smoothing factors are `FAST_VOL_ALPHA = 200` (fast) and `SLOW_VOL_ALPHA = 1800` (slow). See `LibOracle.updateVolatilityEMAs()`.

### 8.2. Volatility Interpretation

| Value (1e6) | Percentage | Description |
|-------------|------------|-------------|
| 10,000 | 0.01% | Very stable |
| 100,000 | 0.1% | Low volatility |
| 1,000,000 | 1% | Moderate |
| 10,000,000 | 10% | High |
| 100,000,000 | 100% | Maximum |

## 9. Security Properties

### 9.1. Flash Loan Resistance

**Same-block rejection:**
```solidity
if (dt == 0) return;  // Skip if same block
```

Attackers cannot manipulate price and use it in the same transaction.

### 9.2. Multi-Block Manipulation

**Dual-window protection:**
- Fast TWAP (5 min): Requires sustained manipulation over 300 seconds
- Slow TWAP (1 hr): Requires sustained manipulation over 3600 seconds
- Both must agree for narrow spreads (otherwise deviation surcharge applies)

### 9.3. Staleness Protection

Configurable TTL per feed. Low-volume pools should:
1. Use external oracle fallback
2. Set shorter TTL
3. Increase spread when stale

## 11. Integration with Pricing

### 11.1. Price Resolution Order

1. Check internal oracle → Use if `MODE_USE_INTERNAL` set
2. Check external primary → Use if `MODE_USE_EXTERNAL` set
3. Fallback to secondary → Use if `MODE_ALLOW_FALLBACK` set
4. Use transient cache if available

### 11.2. Deviation Calculation

$$Delta = max(|o_f - o_s|, |o_f|)$$

where:
- $o_f$ = fast offset
- $o_s$ = slow offset

High deviation triggers wider spreads on toxic flow (deviation surcharge $U$).

See [Spread & Fees §3](/docs/1.1.4-spread-fees#3-spread-calculation) for how deviation is used as a pricing risk signal.

## 12. Transient Caching (EIP-1153)

### 12.1. Purpose

Oracle reads within the same transaction are cached using EIP-1153 transient storage. This avoids redundant storage reads for multi-hop swaps.

### 12.2. Gas Savings

| Operation | Without Cache | With Cache |
|-----------|---------------|------------|
| First oracle read | ~2,100 gas | ~2,100 gas |
| Subsequent reads | ~2,100 gas | ~100 gas |
| 3-hop swap (6 reads) | ~12,600 gas | ~2,500 gas |

### 12.3. Cache Lifetime

Transient storage is automatically cleared at transaction end, no explicit cleanup needed.

See: `contracts/src/libraries/LibTransientCache.sol`
