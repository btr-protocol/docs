# Staking Contract

> Governance token and LP token staking with voting power and reward boosting

---

## 1. Overview

`Staking` (`dex/evm/src/Staking.sol`) is a standalone singleton staking contract serving every pool. Each public function takes `address pool` as its first arg; state is keyed by `(pool, ...)`. Owner authority routes through the shared singleton `AccessControl`. There is no `delegatecall` -Staking calls `Pool`'s restricted setters via standard external calls.

Staking enables users to:
- **Stake BTR**  Receive sBTR for voting and reward boosting
- **Stake LP tokens**  Receive sLP for reward eligibility
- **Delegate voting power**  Transfer voting rights without transferring tokens
- **Lock positions**  Time-based lock for commitment signaling

---

## 2. Token Types

### 2.1. BTR (Governance Token)
- **Purpose:** Protocol governance and reward multiplier
- **Supply:** Fixed maximum (enforced by Treasury)
- **Staking:** Lock BTR  Receive sBTR

### 2.2. sBTR (Staked BTR)
- **Purpose:** Receipt token for staked BTR
- **Utility:** Voting power + reward boosting
- **Non-transferable after cooldown:** Prevents vote/reward renting

### 2.3. sLP (Staked LP)
- **Purpose:** Receipt token for staked LP positions
- **Deployment:** CREATE3 for deterministic addresses
- **Utility:** Reward eligibility

## 3. Governance Token Staking

### 3.1. Stake BTR

```solidity
function stakeGov(uint256 amount) external whenNotPaused
```

**Flow:**
1. Transfer BTR from user to pool
2. Mint sBTR 1:1 to user
3. Update staking balances
4. Set/extend unlock time

**Effects:**
- `govStaked[user] += amount`
- `totalGovStaked += amount`
- `govUnlockTime[user] = max(current, now + stakeLockDuration)`

### 3.2. Unstake BTR

```solidity
function unstakeGov(uint256 amount) external whenNotPaused
```

**Requirements:**
- `block.timestamp >= govUnlockTime[user]`
- `amount <= govStaked[user]`

**Flow:**
1. Burn sBTR from user
2. Transfer BTR to user
3. Update staking balances

## 4. LP Token Staking

### 4.1. Configure Staking for LP Token

```solidity
function updateStakingConfig(
    address lpToken,
    bytes32 salt
) external onlyOwner
```

**Requirements:**
- LP token must be a pool asset
- `STAKEABLE_BIT` flag must be set
- Not already configured

**Actions:**
1. Generate deterministic salt
2. Deploy sLP token via CREATE3
3. Register sLP address

### 4.2. Stake LP

```solidity
function stakeLP(
    address lpToken,
    uint256 amount
) external whenNotPaused
```

**Requirements:**
- sLP token deployed for this LP
- `STAKEABLE_BIT` flag set
- User has sufficient LP balance

**Flow:**
1. Verify user LP balance
2. Mint sLP to user
3. Update staking balances
4. Set/extend unlock time

**Note:** LP tokens remain in pool (not transferred). Staking is accounting-only.

### 4.3. Unstake LP

```solidity
function unstakeLP(
    address lpToken,
    uint256 amount
) external whenNotPaused
```

**Requirements:**
- `block.timestamp >= lpUnlockTime[user][lpToken]`
- `amount <= lpStaked[user][lpToken]`

**Flow:**
1. Burn sLP from user
2. Update staking balances

## 5. Voting Power

### 5.1. Formula

$$V = \frac{L \cdot \text{lpBase}}{10000} + \frac{\min(\text{boosted},\; \text{boostCap} \cdot G) \cdot \text{lpBoosted}}{10000} + \frac{G \cdot \text{govMultiplier}}{10000}$$

where:
- $L$ = total LP stake value
- $G$ = governance token stake
- $\text{boosted}$ = LP that qualifies for boost
- $\text{boostCap}$ = Max LP boost per \$1 of gov stake

### 5.2. Configuration Parameters

```solidity
struct StakingConfig {
    uint48 stakeLockDuration;   // Default 14 days
    uint48 transferCooldown;    // Cooldown after claim
    uint16 minClaimPower;       // Minimum claim multiplier (5000 = 50%)
    uint16 matchingRatio;       // BTR:LP matching ratio (500 = 5:1)

    uint16 lpBaseMultiplier;    // Base LP voting power (5000 = 0.5x)
    uint16 lpBoostedMultiplier; // Boosted LP voting power (5000 = 0.5x)
    uint16 govMultiplier;       // Gov token voting power (10000 = 1x)
    uint16 boostCap;            // Max LP boost per $1 BTR (50000 = 5x)
}
```

### 5.3. Example Calculation

**User has:**
- 1000 LP tokens staked ($1000 value)
- 100 BTR staked ($100 value)

**With default config:**
```
lpBase = 5000 (0.5x)
lpBoosted = 5000 (0.5x)
govMultiplier = 10000 (1x)
boostCap = 50000 (5x)

maxBoostedLP = 100 × 5 = $500
actualBoostedLP = min($1000, $500) = $500
unboostedLP = $1000 - $500 = $500

V = ($500 × 0.5) + ($500 × 0.5) + ($100 × 1.0)
V = $250 + $250 + $100 = $600 voting power
```

## 6. Delegation

### 6.1. Delegate Voting Power

```solidity
function delegateVoting(address to) external
```

**Effects:**
- `delegateOf[msg.sender] = to`
- Delegating to self clears delegation

**Note:** Off-chain snapshot systems read delegation mapping for vote counting.

### 6.2. Get Delegate

```solidity
function getDelegateOf(address owner) external view returns (address)
```

Returns current delegate or address(0) if none.

## 7. Lock Mechanics

### 7.1. Stake Lock Duration

- **Default:** 14 days (`stakeLockDuration`)
- **Effect:** Cannot unstake until lock expires
- **Extending:** New stakes extend lock to `now + lockDuration`

### 7.2. Transfer Cooldown

- **Purpose:** Prevent sToken transfer immediately after claiming rewards
- **Duration:** Configurable (`transferCooldown`)
- **Scope:** Applies to sBTR and sLP transfers

## 10. Storage Layout

State lives at the `Staking` singleton (default storage; no ERC-7201 since Phase 42H). Cross-pool keying via `mapping(address pool => ...)` at the storage root.

```solidity
struct PoolStakingConfig { uint48 stakeLockDuration; bool paused; }

mapping(address pool => PoolStakingConfig) public poolConfig;
mapping(address pool => address) public govTokenOf;
mapping(address pool => address) public sGovTokenOf;

// Governance staking
mapping(address pool => mapping(address user => uint256)) public govStaked;
mapping(address pool => uint256)                          public totalGovStaked;
mapping(address pool => mapping(address user => uint48))  public govUnlockTime;

// LP staking (sLP deployed via CREATE3)
mapping(address pool => mapping(address lpToken => address sLP))
                                                          public sLPTokens;
mapping(address pool => mapping(address user => mapping(address lpToken => uint256)))
                                                          public lpStaked;
```

LP-staked totals consumed by `Distributor` are re-derived from `Staking`'s public getters (no shared `PoolStorage` field for cross-singleton coupling).

## 12. Security Considerations

### 12.1. Sybil Resistance
- Lock durations prevent quick stake/vote/unstake
- Transfer cooldown prevents sToken farming

### 12.2. Economic Security
- Boosting requires BTR commitment
- LP staking is accounting-only (no transfer risk)

### 12.3. Emergency Controls
- Pause functionality for incidents
- Timelock for config changes

## 13. Integration Notes

### 13.1. For Reward Distributors
- Read `lpStaked` and `totalLPStaked` from PoolStorage
- Use `govStaked` for boost calculations
- Check `delegateOf` for voting snapshots

### 13.2. For Governance Systems
- Query voting power via view functions
- Respect delegation mappings
- Consider lock status for proposals
