# Liquidity Shaping

> Fritsch-Carlson monotone cubic Hermite spline-based liquidity profiles for customizable price impact curves

---

## 1. Overview

Traditional AMMs use fixed mathematical invariants (`x * y = k`, or more elaborate functions in the case of stable swaps and elliptical curves) that determine price impact. AIMM uses **Fritsch-Carlson monotone cubic Hermite spline interpolation** (Fritsch & Carlson 1980, *SIAM J. Numer. Anal.* 17(2)) with user-defined control points, allowing precise control over depth at any price point. Note: this is **not** Catmull-Rom - Catmull-Rom uses centered differences without monotonicity guarantee; Fritsch-Carlson applies sign-preservation on adjacent secants plus the `α² + β² ≤ 9` magnitude clamp, which strictly enforces monotonicity between knots (a hard requirement for AMM depth - a non-monotone segment would imply negative marginal liquidity).

**Key Benefits:**
- Concentrate liquidity around expected price ranges
- Replicate and improve all existing CFMM curves with zero overhead
- Fit any asset volatility profile: stable, volatile, inflating, deflating, multi-modal
- Create asymmetric depth relevant to directional markets
- Update profiles overnight without redeploying contracts

## 2. Profile Structure

### 2.1. Storage Layout

```solidity
struct LiquidityProfile {
    uint8[16] weights;      // Segment weights (0.5% resolution)
    int8[17] knots;         // Price offset knots (weights.length + 1 points)
}
```

**Knot Semantics**: Knots represent price offsets as **percentage of dispersion range around TWAP**:
- **Dispersion** defines the liquidity breadth around TWAP (e.g., 1% = 10000 bps means ±1% range)
- **Knots** are offsets within this range as percentage of dispersion
- **Formula**: `price = TWAP × (1 + offset)` where `offset = dispersion × knot / 100`
- **Example**: `TWAP = $1.00`, `dispersion = 1%` (10000 bps), `knot = -50`:
  - Offset = `-50% of 1% dispersion = -0.5% from TWAP`
  - Price = `$1.00 × (1 - 0.005) = $0.995`
- **Range**: `knots` span from `-100` to `+100` (full dispersion range coverage)
- **Constraint**: `max(knots) - min(knots) = 100` ensures complete dispersion range

[^2]: **Solidity implementation**: Computes price as `TWAP * (BPS_PRECISION + dispersion * knot / 100) / BPS_PRECISION` where `BPS_PRECISION = 1,000,000`. The dispersion parameter is in 0.0001% units, knots are in percentage (1-100) of dispersion. See `LibPricing._buildSplinePoints()`.

### 2.2. Constraints

| Property | Requirement |
|----------|-------------|
| Segments | 1-16 active (first zero weight marks end) |
| Weight sum | Must equal 200 (represents 100% depth) |
| Knots | Must be monotonically non-decreasing (N weights → N+1 knots) |
| Knot range | max(knots) - min(knots) = 100 (100% of dispersion coverage) |
| Weight resolution | 0.5% per weight unit (1 unit = 0.5% of domain) |
| Knot resolution | 1% per knot unit (1 unit = 1% of dispersion) |

### 2.3. Domain Mapping

Profile weights define control points on a cumulative depth axis:

$$x = w_c / W_{\text{sum}}$$

where:
- $x$ = control point position (depth axis)
- $w_c$ = cumulative weight
- $W_{\text{sum}}$ = weight sum (200)

[^1]: **Solidity implementation**: Maps `[0, WEIGHT_SUM]` to `[0, 10000]` via `x = (w_c * 10000) / WEIGHT_SUM` where `WEIGHT_SUM = 200`. Each weight unit represents 0.5% of depth. See `LibSpline.buildPoints()`.

### 2.4. Example: Concentrated Stablecoin Profile

| Segment | Weight | Cumulative | X-position | End Offset |
|---------|---------|------------|-------------|-------------|
| 0       | 80      | 80         | 4000        | 0           |
| 1       | 60      | 140        | 7000        | +5          |
| 2       | 30      | 170        | 8500        | +20         |
| 3       | 20      | 190        | 9500        | +50         |
| 4       | 10      | 200        | 10000       | +100        |
| **Total** | **200** | | | |

**Interpretation:** 40% of liquidity depth available at ≤0 offset, 70% within +5 offset.

## 3. Fritsch-Carlson Monotone Cubic Hermite Spline Mathematics

### 3.1. Why Fritsch-Carlson?

AIMM uses **Fritsch-Carlson monotone cubic Hermite interpolation** (Fritsch & Carlson 1980, *SIAM J. Numer. Anal.* 17(2), "Monotonic Piecewise Cubic Interpolation") rather than Catmull-Rom. These are **distinct** families:

- **Catmull-Rom**: centered secant tangents, C¹ continuous, but **not monotone** in general - can overshoot between control points.
- **Fritsch-Carlson**: cubic Hermite with **sign-preservation** on adjacent secants and the `α² + β² ≤ 9` tangent-magnitude clamp, which **guarantees** monotonicity between knots.

AIMM's reasons:

1. **Monotonicity**: Strict - non-monotone segments would imply negative marginal liquidity (breaks pricing). Fritsch-Carlson enforces; Catmull-Rom does not.
2. **C¹ continuity**: Smooth price curve without discontinuities.
3. **Local control**: Each segment depends only on adjacent points.
4. **Gas efficient**: Simple tangent calculation vs full cubic solve.

### 3.2. Knot Point Construction

From profile storage, we build spline control points:

$$\begin{aligned} p_0 &= (x: 0,\; y: -\kappa) \\ p_i &= (x: x_i,\; y: \delta_i \cdot \kappa / 100) \\ p_n &= (x: 10000,\; y: +\kappa) \end{aligned}$$

where:
- $p_0$ = starting spline point
- $p_i$ = intermediate point $i$
- $p_n$ = ending point
- $x_i$ = x-coordinate of point $i$
- $\delta_i$ = knot offset at position $i$
- $\kappa$ = dispersion parameter (in 0.0001% units)

### 3.3. Tangent Calculation

At interior knot $i$, the tangent is the average of adjacent secants:

$$\begin{aligned} m_i &= \frac{s_{i-1} + s_i}{2} \\ s_i &= \frac{y_{i+1} - y_i}{x_{i+1} - x_i} \end{aligned}$$

where:
- $m_i$ = tangent at knot $i$
- $s_i$ = secant slope between knots
- $y_i$, $x_i$ = price offset and depth position coordinates

If adjacent secants have opposite signs, $m_i = 0$ (preserves monotonicity).

**Magnitude clamp** (Fritsch-Carlson 1980, §3) - applied when adjacent secants share sign but tangent magnitudes are extreme:

$$ \alpha_i = \frac{m_i}{s_i}, \quad \beta_i = \frac{m_{i+1}}{s_i}, \quad \alpha_i^2 + \beta_i^2 \le 9 $$

If the inequality is violated, scale both tangents: $m_i \leftarrow \tau\alpha_i s_i,\; m_{i+1} \leftarrow \tau\beta_i s_i$ with $\tau = 3/\sqrt{\alpha_i^2 + \beta_i^2}$. This guarantees monotonicity even when adjacent secants share sign but tangent magnitudes are extreme.

### 3.4. Cubic Hermite Interpolation

For $t \in [0, 1]$ within segment $[p_i, p_{i+1}]$:

$$y(t) = \overbrace{(2t^3 - 3t^2 + 1)}^{H_{00}(t)} \cdot y_i + \overbrace{(t^3 - 2t^2 + t)}^{H_{10}(t)} \cdot m_i \cdot h + \overbrace{(-2t^3 + 3t^2)}^{H_{01}(t)} \cdot y_{i+1} + \overbrace{(t^3 - t^2)}^{H_{11}(t)} \cdot m_{i+1} \cdot h$$

where:
- $t$ = parameter in $[0, 1]$ within segment $[p_i, p_{i+1}]$
- $h$ = $x_{i+1} - x_i$ segment width
- $m_i$, $m_{i+1}$ = tangents at endpoints
- $y_i$, $y_{i+1}$ = values at endpoints

## 4. Spline Integration (Area)

### 4.1. Purpose

For VWAP (volume-weighted average price) calculations, we need the **integral** of the price curve over a traversed range.

### 4.2. Exact Analytical Integration

The integral of a cubic Hermite polynomial has closed form:

$$\int_0^1 y(t) \, dt = \overbrace{\frac{y_i + y_{i+1}}{2}}^{\text{average value}} + \overbrace{\frac{(m_i - m_{i+1}) \cdot h}{12}}^{\text{curvature correction}}$$

where:
- $h$ = $x_{i+1} - x_i$ segment width
- $m_i$, $m_{i+1}$ = tangents at endpoints
- $y_i$, $y_{i+1}$ = values at endpoints

For arbitrary bounds $[a, b]$, we sum contributions across segments.

### 4.3. Average Execution Price

Given traversal from $x_0$ to $x_1$:

$$\bar{p} = p_T \cdot \left(1 + \frac{A(x_0, x_1)}{x_1 - x_0}\right)$$

where:
- $\bar{p}$ = average execution price
- $p_T$ = TWAP (oracle price)
- $A$ = integrated area over depth range $[x_0, x_1]$

[^5]: **Solidity implementation**: Returns `TWAP * (BPS_PRECISION + area / (x1 - x0)) / BPS_PRECISION` where `BPS_PRECISION = 1,000,000`. The area from spline integration is divided by the depth range and added to the TWAP as an offset. See `LibPricing._traverseSplineByVolume()`.

### 4.4. Implementation


## 5. Inventory Skew Mapping (Skew -> Depth)

> For how skew is computed from coverage ratio, see [Inventory Management §3](1.1.1.%20Inventory%20Management.md#3-inventory-skew).

### 5.1. Skew to Depth Position

Once computed, inventory skew $\psi \in [-100, +100]$ maps to cumulative depth $d \in [0, 10000]$:

$$d = 5000 + \psi \cdot 50$$

where:
- $d$ = cumulative depth position (in $[0, 10000]$)
- $\psi$ = inventory skew (in $[-100, +100]$)

[^3]: **Solidity implementation**: Linear mapping where `skew = -100` maps to `depth = 0`, `skew = 0` maps to `depth = 5000`, and `skew = +100` maps to `depth = 10000`. Formula: `depth = 5000 + skew * 50`. See `LibPricing._skewToDepth()`.

| $\psi$ | $d$ | Meaning |
|-------|-----|---------|
| -100 | 0 | Maximum discount (full sell pressure) |
| 0 | 5000 | Mid-price (equilibrium) |
| +100 | 10000 | Maximum premium (full buy pressure) |

See: `LibPricing._skewToDepth()`

## 6. Dispersion Dynamics

### 6.1. Purpose

**Dispersion** $\kappa$ scales the profile's price range based on volatility. Higher dispersion = wider price curve = steeper impact.

### 6.2. Formula

$$\kappa = \operatorname{clamp}(\kappa_0 + \sigma \cdot \nu,\; \kappa_{\min},\; \kappa_{\max})$$

where:
- $\kappa$ = dispersion (basis points, clamped)
- $\kappa_0$ = base dispersion (1000 = 0.1%)
- $\sigma$ = volatility
- $\nu$ = vega sensitivity
- $\kappa_{\min}$, $\kappa_{\max}$ = dispersion bounds

[^4]: **Solidity implementation**: Computes `scaledVol = (volatility * vega) / (1000 * MULT_BASE)` where `MULT_BASE = 10000`, then `dispersion = 1000 + scaledVol` clamped to `[minDispersion, maxDispersion]`. All dispersion values use 0.0001% precision (`BPS_PRECISION = 1,000,000`). See `LibPricing._calculateDispersion()`.

### 6.3. Parameters

| Parameter | Type | Unit | Purpose |
|-----------|------|------|---------|
| `vega` | uint16 | basis 10000 | Volatility sensitivity |
| `minDispersion` | uint32 | 0.0001% | Floor (prevents free options) |
| `maxDispersion` | uint32 | 0.0001% | Ceiling (prevents illiquidity) |

### 6.4. Volatility Effect

| $\sigma$ (1e6) | $\nu$ | $\Delta \kappa$ |
|---------------|------|---------------|
| 1,000,000 (1%) | 10000 | +100 |
| 5,000,000 (5%) | 10000 | +500 |
| 1,000,000 (1%) | 20000 | +200 |

### 6.5. Implementation

## 8. Profile Design Patterns

### 8.1. Concentrated (Stablecoins)

Maximize depth near peg, wide wings for black swan events:

```
weights:  [80, 60, 30, 20, 10, 0, ...]
offsets:  [0,  5,  20, 50, 100, ...]
```

**Use case:** USDC/USDT pools where 99% of trades occur within 10 bps.

### 8.2. Uniform (Volatile Assets)

Even distribution across price range:

```
weights:  [25, 25, 25, 25, 25, 25, 25, 25, 0, ...]
offsets:  [0,  25, 50, 75, 100, 125, 150, 200, ...]
```

**Use case:** ETH/BTC pairs with unpredictable price movements.

### 8.3. Asymmetric (Directional)

More depth on expected direction:

```
weights:  [30, 40, 50, 40, 30, 10, 0, ...]
offsets:  [-20, 0, 20, 60, 120, 200, ...]
```

**Use case:** Pools expecting persistent buy/sell pressure.

### 8.4. Deep Core (Institutional)

Very high concentration for large trades:

```text
weights:  [120, 40, 20, 10, 10, 0, ...]
offsets:  [0,   2,  5,  10, 20, ...]
```

**Use case:** Institutional stablecoin pools with tight spread requirements.

## 9. Traversal Flow

### 9.1. Complete Pricing Pipeline

```text
1. Load oracle data → get TWAP, σ
2. Compute dispersion: κ = f(σ, vega, min, max)
3. Compute inventory skew: s = f(coverage, gamma)
4. Compute effective depth: D = f(reserves, liabilities, amplifier)
5. Map skew to start position: x₀ = 5000 + s × 50
6. Compute volume fraction: Δx = amountIn × 10000 / D
7. Compute end position: x₁ = x₀ ± Δx (buy/sell)
8. Build spline points from profile + dispersion
9. Integrate: Area(x₀, x₁)
10. Average price: TWAP × (1 + Area / Δx / BPS_PRECISION)
```

### 9.2. Implementation

```solidity

    uint256 twap,
    uint32 dispersion,
    IPool.LiquidityProfile storage profile,
    int8 inventorySkew,
    uint256 amountIn,
    uint256 depth,
    bool selling
) internal view returns (uint256 avgPrice)
```

## 11. Gas Optimization

### 11.1. Efficient Storage

- Profiles fit in 2 storage slots (32 bytes each)
- Weights as uint8 (0-255 range)
- Offsets as int8 (-128 to +127 range)

### 11.2. Traversal Cost

| Operation | Gas Cost |
|-----------|----------|
| Load profile | ~2,100 (cold) / 100 (warm) |
| Binary search | O(log n) comparisons |
| Cubic interpolation | ~500 gas |
| Total (typical) | ~3,000-5,000 gas |

Comparable to Uniswap v3 tick traversal for small swaps.

## 13. Best Practices

### 13.1. For Stablecoins
1. Heavy concentration (60%+) within 5 bps offset
2. Moderate wings to 50-100 bps
3. Low dispersion bounds (1000-5000 in 0.0001% units)
4. High depth amplifier (500,000+)

### 13.2. For Volatile Assets
1. More uniform distribution
2. Wider max offsets (200+ bps)
3. Higher dispersion bounds
4. Lower depth amplifier

### 13.3. For New Assets
1. Start with conservative (uniform) profile
2. Monitor trade distribution
3. Adjust based on observed depth utilization
4. Consider volatility regime changes
