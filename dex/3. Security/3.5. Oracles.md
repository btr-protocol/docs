# Oracle System

> Multi-timeframe TWAP and volatility tracking for informed pricing

---

## 1. Overview

AIMM uses a **two-oracle pattern**:

1. **Internal Oracle** (InternalOracle) -Auto-updated on every swap, tracks TWAPs and volatility from execution prices
2. **External Oracle** (ExternalOracle) -Push-based feeds from Chainlink, Pyth, or custom keepers

The system decouples price discovery from reserves, enabling informed market making instead of mechanical invariant pricing.

---

## 2. Internal Oracle (InternalOracle)

### 2.1. Purpose

Tracks pool-observed prices with dual-timeframe smoothing:

- **Fast TWAP**: 5-minute window (responsive to recent trades)
- **Slow TWAP**: 1-hour window (resistant to manipulation)

Both TWAPs update automatically after each swap -no external keeper required.

### 2.2. Feed Structure

Storage per token (spans multiple slots):

```solidity
struct FeedAccumulator {
    // Accumulator state (B64 with configurable accDecimals)
    uint64 priceAccB64;       // Time-weighted price accumulator
    uint64 fastSnapB64;       // Fast window snapshot
    uint64 slowSnapB64;       // Slow window snapshot
    uint32 fastSnapshotTime;  // Fast snapshot timestamp
    uint32 slowSnapshotTime;  // Slow snapshot timestamp

    // Price & offsets
    uint64 lastPriceB64;      // Current spot price (B64, token decimals)
    int32 fastOffset;         // Fast TWAP offset from spot (0.0001% units)
    int32 slowOffset;         // Slow TWAP offset from spot (0.0001% units)

    // Volatility & metadata
    uint32 lastUpdate;        // Timestamp of last update
    uint32 fastVolEMA;        // Fast volatility EMA (1e6 base)
    uint32 slowVolEMA;        // Slow volatility EMA (1e6 base)
    uint16 ttl;               // Time-to-live (default 1 hour)
    uint8 accDecimals;        // Accumulator encoding decimals (set at init)
    uint8 confidence;         // 0-100 confidence score
}
```

### 2.3. Accumulator Decimals (accDecimals)

The `accDecimals` parameter controls how TWAP accumulators are encoded in B64 format. Using **lower decimals** provides **more headroom** for time-weighted accumulation:

| Asset Class | Example | accDecimals | Max Accumulator | Practical Limit |
|-------------|---------|-------------|-----------------|-----------------|
| **Stablecoins** | USDC, USDT | 6 | ~4.5×10⁷² | >10,000 years |
| **Mid-cap** | ETH, SOL | 12 | ~4.5×10⁶⁶ | >10,000 years |
| **High-value** | BTC, WBTC | 18 | ~4.5×10⁶⁰ | >1,000 years |

**Why this matters:**

B64 float format has 52-bit mantissa with exponent range ±63. The accumulator stores:
```
accumulator += price_in_accDecimals_base × dt_seconds
```

For stablecoins (~$1), using `accDecimals=6`:
- Price in accDecimals base: `decodeB64(priceB64, 6)` ≈ 1×10⁶
- After 1 year: 1×10⁶ × 31.5M seconds ≈ 3×10¹³
- B64 mantissa max: ~4.5×10¹⁵ (plenty of room before exponent kicks in)
- With exponent: can handle up to ~4.5×10⁷²

For BTC (~$100k), using `accDecimals=18`:
- Price in accDecimals base: `decodeB64(priceB64, 18)` ≈ 1×10²³
- After 1 year: 1×10²³ × 31.5M ≈ 3×10³⁰
- Still within B64 range (~4.5×10⁶⁰)

**Configuration:**

Set via `OracleConfig.accDecimals` when adding assets:

```solidity
OracleConfig memory cfg = OracleConfig({
    primary: address(pool),  // Internal oracle
    secondary: address(0),
    feedId: bytes32(0),
    modeFlags: 0,
    accDecimals: 6,          // For stablecoins
    _pad: padArray
});
```

Default is `6` if not specified (stablecoin-optimized).

### 2.4. Constants

| Constant | Value | Description |
|----------|-------|-------------|
| `FAST_WINDOW` | 300s | 5-minute TWAP window |
| `SLOW_WINDOW` | 3600s | 1-hour TWAP window |
| `FAST_VOL_ALPHA` | 200 | 0.02% volatility smoothing |
| `SLOW_VOL_ALPHA` | 1800 | 0.18% volatility smoothing |
| `MAX_VOLATILITY` | 100,000,000 | 10,000% max vol (1e6 base) |
| `DEFAULT_TTL` | 3600 | 1 hour staleness threshold |

### 2.5. Update Flow

On every swap, `pushFeedInternal()` is called:

```
swap() executes
    ↓
Calculate execution price in B64 (token decimals)
    ↓
pushFeedInternal(tokenA, tokenB, priceA, priceB)
    ↓
For each non-base token with price:
    _updateFeeds(token, newPrice)
```

#### 2.5.1. `_updateFeeds()` Logic

1. **Same-block protection**: Skip if `dt == 0` (prevents manipulation)

2. **Accumulator update** (using accDecimals, not token decimals):
   ```
   accDec = acc.accDecimals              // Set at init (6 for stables, 12 for ETH, etc.)
   lastPriceInAccDec = decodeB64(lastPriceB64, accDec)
   increment = lastPriceInAccDec × dt
   currentAcc = decodeB64(priceAccB64, accDec) + increment
   priceAccB64 = encodeB64(currentAcc, accDec)   // Store with accDecimals
   ```

3. **Fast TWAP** (every 5 minutes):
   ```
   if dtFast >= FAST_WINDOW:
       fastSnapAcc = decodeB64(fastSnapB64, accDec)
       twapInAccDec = (currentAcc - fastSnapAcc) / dtFast
       fastTWAP = twapInAccDec × 10^(18 - accDec)   // Scale to 1e18
       fastOffset = encodeOffset1e18(spot1e18, fastTWAP)
       fastSnapB64 = priceAccB64  // Store snapshot
   ```

4. **Slow TWAP** (every hour):
   ```
   if dtSlow >= SLOW_WINDOW:
       slowSnapAcc = decodeB64(slowSnapB64, accDec)
       twapInAccDec = (currentAcc - slowSnapAcc) / dtSlow
       slowTWAP = twapInAccDec × 10^(18 - accDec)   // Scale to 1e18
       slowOffset = encodeOffset1e18(spot1e18, slowTWAP)
       slowSnapB64 = priceAccB64  // Store snapshot
   ```

5. **Volatility EMA**:
   ```
   priceChange = |lastPrice - newPrice| (1e6 base)
   fastVolEMA = EMA(fastVolEMA, priceChange, FAST_VOL_ALPHA)
   slowVolEMA = EMA(slowVolEMA, priceChange, SLOW_VOL_ALPHA)
   ```

### 2.6. Offset Encoding

Offsets represent TWAP deviation from spot price:

```
offset = ((TWAP / spot) - 1) × OFFSET_PRECISION

where OFFSET_PRECISION = 10,000,000 (0.0001% = 1 unit)
```

**Decoding** (in LibOracle):
```
twap = spot × (1 + offset / OFFSET_PRECISION)
```

**Benefits**:
- Single B64 decode per price read
- Offsets applied in 1e18 space (no redundant encoding)
- int32 range covers ±214,748% deviation

## 3. External Oracle (ExternalOracle)

### 3.1. Purpose

Push-based oracle for:
- Chainlink/Pyth price feeds
- Off-chain keeper updates
- Cross-chain price aggregation

### 3.2. Feed Structure

Same `FeedData` struct as internal oracle:

```solidity
struct FeedData {
    uint64 lastPriceB64;      // Fast EMA (reference price)
    int32 fastOffset;         // Always 0 (fast = reference)
    int32 slowOffset;         // Slow EMA offset from fast
    uint32 fastVolEMA;        // Fast volatility
    uint32 slowVolEMA;        // Slow volatility
    uint32 updatedAt;         // Last push timestamp
    uint16 ttl;               // Staleness threshold
    uint16 confidence;        // 0-100
}
```

### 3.3. Feed Management

**Adding a feed**:
```solidity
addFeed(
    base,           // Token being priced
    quote,          // Quote currency (e.g., USD)
    fastEMA,        // Initial fast price (B64)
    slowEMA,        // Initial slow price (B64)
    fastVolEMA,     // Initial fast vol (1e6)
    slowVolEMA,     // Initial slow vol (1e6)
    maxDeviation,   // Max allowed deviation (0.0001% units)
    ttl             // Staleness threshold (seconds)
)
```

Feed ID: `keccak256(abi.encodePacked(base, quote))`

**Pushing updates**:
```solidity
// Single feed
pushFeed(feedId, newFastEMA, newSlowEMA, newFastVolEMA, newSlowVolEMA)

// Batch (gas efficient)
batchPush(feedIds[], fastEMAs[], slowEMAs[], fastVolEMAs[], slowVolEMAs[])
```

### 3.4. Access Control

| Role | Capabilities |
|------|--------------|
| **Owner** | Add feeds, update config, grant/revoke oracles |
| **Oracle** | Push price updates |

## 4. B64 Float Encoding

### 4.1. Format (52/5/7)

```
[mantissa:52 | decimals:5 | exponent+bias:7]

Total: 64 bits
```

| Field | Bits | Range | Description |
|-------|------|-------|-------------|
| Mantissa | 52 | 0 - 4.5×10^15 | Normalized value |
| Decimals | 5 | 0-31 | Token decimal places |
| Exponent | 7 | -64 to +63 (biased by 64) | Scale factor |

### 4.2. Encoding (LibMaths)

```solidity
function encodeB64(uint256 value, uint8 decimals) internal pure returns (uint64)
```

1. Normalize mantissa to 52 bits
2. Compute exponent from normalization shift
3. Pack: `(mantissa << 12) | (decimals << 7) | (exponent + 64)`

### 4.3. Decoding (LibMaths)

```solidity
function b64To1e18(uint64 b64) internal pure returns (uint256)
```

1. Extract mantissa, decimals, exponent
2. Compute total shift = exponent - bias + decimals
3. Return: `mantissa × 10^totalShift` (normalized to 1e18)

### 4.4. Example

Price: 2500.00 USDC (6 decimals)
- Value: 2,500,000,000 (2500 × 10^6)
- Mantissa: Normalized to 52 bits
- Decimals: 6
- Exponent: Based on normalization

## 5. LibOracle Functions

### 5.1. Price Decoding

```solidity
/// Decode both TWAPs efficiently (single B64 decode)
function decodeB64s(FeedData memory feed)
    returns (uint256 priceFast, uint256 priceSlow)

/// Fast path for hot operations
function getFastEMA(FeedData memory feed) returns (uint256 priceFast)

/// Slow EMA only
function getSlowPrice(FeedData memory feed) returns (uint256 priceSlow)
```

### 5.2. Volatility & Deviation

```solidity
/// Effective volatility σ = (fastVol + slowVol) / 2
function getSigma(FeedData memory feed) returns (uint32 sigma)

/// Price deviation Δ = max(|fast-slow|, |fast-current|)
function getDelta(FeedData memory feed) returns (uint32 delta)
```

### 5.3. Base Token Feed

```solidity
/// Synthetic feed for base token (always 1.0)
function getBaseFeed() returns (FeedData memory feed)
```

Returns:
- `lastPriceB64`: 1.0 encoded
- Offsets: 0
- Volatility: 0.01% baseline
- TTL: Never expires

## 6. Oracle Selection

### 6.1. Mode Flags

Configured per asset in `OracleConfig`:

```solidity
struct OracleConfig {
    address primary;      // Primary oracle (internal or external)
    address secondary;    // Fallback oracle
    uint16 mode;          // Behavior flags
    uint16 maxDeviation;  // Max price deviation threshold
}
```

| Mode Flag | Bit | Behavior |
|-----------|-----|----------|
| `MODE_USE_INTERNAL` | 0x01 | Prefer internal oracle |
| `MODE_USE_EXTERNAL` | 0x02 | Prefer external oracle |
| `MODE_ALLOW_FALLBACK` | 0x04 | Fall back to secondary on stale/missing |

### 6.2. Selection Logic

```
1. Check mode flags
2. If USE_INTERNAL: query InternalOracle
3. If USE_EXTERNAL: query ExternalOracle
4. If stale and ALLOW_FALLBACK: try secondary
5. If all fail: revert StaleData
```

## 7. Transient Caching (EIP-1153)

For gas optimization, `LibTransientOracle` caches decoded prices in transient storage:

```solidity
/// Cache key: keccak256(abi.encodePacked(token, "price"))
/// Cache value: (priceFast, priceSlow, sigma, delta)
```

**Savings**: ~2,100 gas per cache hit

**Scope**: Single transaction (clears automatically)

## 8. Security Considerations

### 8.1. Manipulation Resistance

1. **Same-block skip**: Prevents multiple updates in one block
2. **TWAP smoothing**: 5min/1hr windows resist flash loan manipulation
3. **Dual-window cross-check**: Large fast/slow divergence triggers higher fees

### 8.2. Staleness Protection

1. **TTL enforcement**: Feeds expire after configurable period
2. **Confidence scoring**: External feeds can signal uncertainty
3. **Fallback cascade**: Secondary oracle protects against primary failure

### 8.3. Deviation Bounds

1. **maxDeviation threshold**: Rejects external pushes with extreme moves
2. **MAX_VOLATILITY cap**: 10,000% ceiling prevents overflow
3. **Offset clamping**: int32 range prevents extreme TWAP divergence

## 9. Usage Examples

### 9.1. Reading Internal Oracle

```solidity
// Get full feed data
IOracle.FeedData memory feed = IOracle(pool).getFeed(token);

// Check freshness
bool fresh = IOracle(pool).isFeedFresh(token, 300); // 5 min max age

// Get just fast TWAP
uint64 fastTWAP = IOracle(pool).getFastTWAP(token);
```

### 9.2. Initializing Oracle

```solidity
// Admin: initialize token feed
IOracle(pool).updateFeed(
    token,           // Asset
    initialPriceB64, // Starting price (B64, token decimals)
    6,               // accDecimals (6 for stables, 12 for ETH, 18 for BTC)
    10_000,          // 0.01% fast vol
    10_000           // 0.01% slow vol
);
```

**Note:** `accDecimals` determines accumulator range vs precision tradeoff. Lower values = more headroom for long-running pools. See [Accumulator Decimals](#accumulator-decimals-accdecimals) section.

### 9.3. External Oracle Push

```solidity
// Keeper: push batch update
oracle.batchPush(
    feedIds,
    fastEMAs,
    slowEMAs,
    fastVolEMAs,
    slowVolEMAs
);
```
