# Treasury Integration

> Protocol treasury operations for fee collection and token management

---

## 1. Overview

The BTR Treasury (`Treasury`) is a standalone contract that manages:
- Protocol fee collection from pools
- BTR token minting (emissions, treasury, vesting)
- TGE initialization and vesting schedules

**Contract**: `contracts/src/Treasury.sol`
**Interface**: `ITreasury`

---

## 2. Protocol Fee Collection

### 2.1. Collect Fees

Anyone can trigger fee collection from a pool. Fees are transferred to the treasury balance.

```solidity
function collectProtocolFees(address pool, address token) external;
```

**TypeScript SDK**:
```typescript
import { Treasury } from '@btr-protocol/sdk';

const treasury = new Treasury(treasuryAddress, signer);
const tx = await treasury.collectProtocolFees(poolAddress, tokenAddress);
```

**Event**:
```solidity
event ProtocolFeesCollected(address indexed pool, address indexed token, uint256 amount);
```

## 3. Token Supply Queries

### 3.1. Supply Information

```solidity
// Get current BTR total supply
uint256 total = treasury.getTotalSupply();

// Get maximum BTR supply (hard cap: 100,000,000)
uint256 max = treasury.getMaxSupply();

// Get remaining mintable BTR
uint256 remaining = treasury.getRemainingMintable();
```

---

## 4. Emissions

### 4.1. Mint Emissions

Owner-only function to mint BTR to the distributor for emissions.

```solidity
function mintEmissionsToDistributor(uint256 amount) external;
```

**Constraints**:
- Only callable by treasury owner
- Enforces `emissionsCap` (65M BTR)
- Total emissions cannot exceed cap

### 4.2. Bridge Emissions

Owner-only function to bridge emissions to remote chains.

```solidity
function bridgeEmissions(uint32 dstEid, uint256 amount, bytes calldata options) external payable;
```

---

## 5. Vesting

### 5.1. Claim Vested Tokens

Beneficiaries can claim their vested tokens after the cliff period.

```solidity
function claimVested() external;
```

**Vesting Schedule**:
- 5 year linear vesting
- 6 month cliff
- 15% unlocked at cliff

### 5.2. Query Claimable

```solidity
uint256 claimable = treasury.getClaimableVested(beneficiary);
```

---

## 6. Error Handling

| Error | Cause | Solution |
|-------|-------|----------|
| `ExceedsMaxSupply` | Minting exceeds 100M cap | Check `getRemainingMintable()` |
| `NotConfigured` | Resource not set up | Verify pool/token addresses |
| `InvalidState` | Operation not allowed | Check if TGE initialized |
