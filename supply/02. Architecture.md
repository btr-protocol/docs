# 7.1. Vault Architecture

> Multi-adapter ERC-4626/7540 vault with a singleton swap router and a single-source-of-truth governance contract.

## 3. Single Source of Truth — `AccessControl`

`AccessControl.sol:14` is the only governance contract. Vaults and Dex adapters do not store their own owner / keeper sets; they read them on every privileged call. Rotation propagates instantly to every consumer.

- **Owner** — Solady 2-step pull pattern; `transferOwnership` and `renounceOwnership` disabled (`AccessControl.sol:60–61`); zero-target completes rejected (`:64`).
- **Keepers** — `isKeeper[address]`, owner-managed (`:69`). Keepers can ratchet defensive parameters and call adapter `kill`, subject to the cluster daily cap.
- **Treasury / Swapper / Factory** — addresses behind timelocks (`ROTATION_TIMELOCK = 7 days`, `FACTORY_TIMELOCK = 14 days`). Bootstrap variants (`bootstrapFactory`, `bootstrapSwapper`) permitted exactly once.
- **Adapter registry** — `isAdapter[a]` set by Factory on `deployDex` (`:101`). Owner can `deregisterAdapter` (`:92`) to retire a compromised clone — vaults using that adapter become unhealthy until the operator calls `removeAdapter`.

## 4. Vault Internals

`Vault.sol:21` is a Solady `ERC20 + Initializable + ReentrancyGuardTransient`. Notable storage:

- `_adapters[8]` — fixed-size array, max 8 adapters (`Vault.sol:60`).
- `adapterState[a] = (lastReported, lastSync)` — cached worst-of mark + sync timestamp.
- `Pending` cohort maps — `pendingRedeem[ctl]` and `claimableRedeem[ctl]`; ERC-7540 ticket aggregation is keyed by **controller**, blended fee-pip is shares-weighted (`:286–304`).
- `hwmAssets`, `hwmSupply`, `hwmInit` — explicit HWM init flag prevents re-arming on full-exit cycles (`:51–53`, `:451`).

## 5. Dex Adapter Internals

`Dex.sol:27` supports six DEX types via the `DexType` enum (`:16`). Multi-holder model (`:92–93`):

- Holders may be vaults, EOAs, multisigs.
- `principalOf[holder]` is **internal LP shares** (semantically — pull mints, withdraw burns), denominated in shares of the adapter's NAV, not raw cost basis.
- `pull(amount)` mints shares pro-rata of current `assetValue(token0)` (`:258`); late-joiners get fair shares without diluting incumbent PnL.
- `vaultAssets(holder) = assetValue(token0) × principalOf[holder] / totalPrincipal` (`:383`).

## 6. Swapper Singleton

`Swapper.sol:19` mediates **all** swap execution for adapters and vault salvage:

- **Atomic** — `swap(...)` calls the pinned `lifiRouter`, balance-delta enforces minOut and zero-residual invariant (`:140–169`).
- **Intent (ERC-7683)** — `openIntent(...)` escrows funds at the pinned `erc7683Settler`; per-tokenOut watermark (`activeOut[token]`, `:52`) prevents settle-order contamination by serializing one active intent per token.
- **Refund paths** — `refundIntent` (soft, settler responsive), `forceRefund` (hard, post `INTENT_GRACE = 6h`), `claimLateRefund` (async settler return).
- **Slippage authority** — adapter pre/post `assetValue` envelope. Swapper enforces only `minOut` + escrow accounting.
- **Rotation** — `ROUTER_TIMELOCK = 7 days` with bootstrap-once; previous Swapper drains gracefully via `forceRefund` / `claimLateRefund`.

## 7. Factory + Clones

`Factory.sol:16` is an immutable triple `(AccessControl, vaultImpl, dexImpl)`:

- `deployVault(...)` — permissionless; the clone is inert (no adapters) until owner adds them.
- `deployDex()` — owner-only; on success, calls `ac.registerAdapter(clone)` so the new clone is trusted by every vault.

To rotate implementations, the owner queues a new Factory via `AC.queueFactory` (14-day timelock). Existing deployments remain functional through the old Factory; only **new** adapters added to vaults must come from the active Factory.

## 8. Invariants Worth Documenting

- `reservedAssets ≥ Σ pendingRedeem.assets` (`Vault.sol:352` — checked subtraction).
- Asset balance ≥ `reservedAssets + claimableAssets` post-`processWithdrawals` (`:360`).
- `exitPip ≥ 1.2 × max_i(deviationBps_i + heartbeatDriftBps_i)` (in pip; `:574–588`) — enforced at `setFees` write time.
- Adapter `lastSync` advances iff `vaultAssets` read succeeded (`:526–535`) — strict, no try/catch in `_sync`.
- `principalOf` shares are zeroed before holder de-registration (`Dex.sol:175`).
- Swapper post-call: `balanceOf(tokenIn, swapper) ≤ inBefore` (`Swapper.sol:166`).
