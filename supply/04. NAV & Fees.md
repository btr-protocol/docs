# 4. NAV & Fees

> How a vault prices its share token and how exit, management, and performance fees accrue.

---

## 1. NAV: Pessimistic Worst-Of

`Vault.totalAssets()` (`Vault.sol:137`) sums:

1. Idle `_asset` balance held at the vault contract.
2. For each adapter, `IDex.vaultAssets(vault)` -the vault's pro-rata share of the adapter's `assetValue(token0)`.

`Dex.assetValue(token)` (`Dex.sol:356`) is the **pessimistic worst-of(pool, oracle)** mark of LP holdings + idle balances, denominated in the requested side:

```text
amt0 = LP_amt0 + idle_token0
amt1 = LP_amt1 + idle_token1

if denom == token0:
    cO = oracle_convert(amt1 → token0)             // Chainlink USD ratio + decimal-adj
    cP = pool_convert(amt1 → token0)               // (sqrtP / Q96)² applied
    return amt0 + min(cO, cP)
else: symmetric
```

This is the protocol's **inline JIT defense**: an attacker pumping the pool to mint cheap shares only succeeds in shifting the pool side; the oracle side is taken instead. The same logic protects exits -pumping the pool to extract value is bounded by `min(pool, oracle)`.

### 1.1. Outage Tolerance

`Vault.totalAssets()` wraps adapter calls in `try/catch`:

```solidity
try IDex(_adapters[i]).vaultAssets(address(this)) returns (uint256 v) { s += v; }
catch { s += adapterState[_adapters[i]].lastReported; }
```

A reverting adapter (e.g., transient oracle stale) does not DoS the vault -the cached `lastReported` is used. The vault, however, transitions to `unhealthy` (see §3) and gates sync deposit/redeem.

### 1.2. Virtual Offset

`_supply() = totalSupply() + OFFSET (= 1e18)` (`Vault.sol:148`). The virtual share offset blunts inflation attacks on first deposit -early depositors cannot mint at exchange rate `0`.

## 2. Exit Fee

A flat fee in **pip** (`PIP_BASE = 100_000`, so `100 pip = 0.1%`) charged on every withdrawal path:

- Sync `redeem` (`Vault.sol:240`): `fee = ⌈g × exitPip / PIP_BASE⌉`, transferred to `ac.treasury()`.
- Sync `withdraw` (`Vault.sol:261`): identical, derived from gross-up.
- Async `claimRedeem` (`Vault.sol:377`): blended cohort `feePip` (shares-weighted on enqueue and merge -`Vault.sol:286–304`, `:343–351`).

### 2.1. Floor Invariant

```text
exitPip ≥ MIN_EXIT_PIP                              // 50 pip = 5 bps
exitPip ≥ 1.2 × max_i(deviationBps_i + heartbeatDriftBps_i)
                                                    // expressed in pip
```

`Vault._exitFloor()` (`Vault.sol:574`) walks every adapter and computes the worst per-adapter `(deviationBps + heartbeatDriftBps)` sum, multiplies by `12` (= `1.2 × 10`, since `1 bps = 10 pip`), and clamps to `uint16`. `setFees` reverts with `ExitFloor` if the requested `exitPip` is below the floor (`:401`).

**Why**: an MEV searcher who exits through a window where pool ↔ oracle deviation is at the configured maximum captures up to `(deviationBps + heartbeatDriftBps)` bps of free value if the exit fee does not exceed it. The 1.2× factor leaves a 20% margin.

## 3. Management Fee

Linear time accrual on `totalAssets`:

```solidity
mgmtAssets = ta × (mgmtPip × elapsed) / (PIP_BASE × YEAR)
mgmtShares = mgmtAssets × supply / (ta − mgmtAssets)   // dilution-equivalent
_mint(treasury, mgmtShares)
```

(`Vault._accrue()`, `:423`.) Bounded `≤ MAX_MGMT_PIP = 50_000 = 50%/yr`. `elapsed` is clamped to `1 year` per accrual to bound a long-dormant vault's first re-accrual. `_accrue()` runs at the **head** of every state-changing call (`deposit`, `redeem`, `withdraw`, `requestRedeem`, `processWithdrawals`, `setFees`, `harvestPerformance`, `syncAdapter`).

## 4. Performance Fee -High-Water Mark

`harvestPerformance()` (`Vault.sol:442`) is the **only** site that updates the HWM. `_accrue` does not. Transient deposit pumps therefore cannot inflate the HWM.

### 4.1. The (assets, supply) Pair

The HWM is stored as the pair `(hwmAssets, hwmSupply)` (`:51–52`). Comparing:

```text
hwmPriceAtCurSup = hwmAssets × supply / hwmSupply
gain             = max(0, totalAssets - hwmPriceAtCurSup)
perfAssets       = ⌈gain × perfPip / PIP_BASE⌉
perfShares       = perfAssets × supply / totalAssets
```

This is mathematically equivalent to comparing share prices `(ta/sup)` vs `(hwmAssets/hwmSupply)` while preserving precision under integer math. The integer-safe form is `ta × hwmSupply > hwmAssets × supply` (`:457`).

### 4.2. Init Flag

`hwmInit` (`:53`) explicitly initializes the HWM on first call. Numeric guards (`hA == 0 || hS == 0`) would re-trigger init when `ta` floors to `1` after a full-exit cycle -and re-arm the HWM at the dust mark. The bool is the canonical first-init signal.

### 4.3. Bound

`MAX_PERF_PIP = 20_000 = 20%`. Charged in shares (dilution), minted to `ac.treasury()`.

## 5. Claim-by-Balance Haircut

When `processWithdrawals(controllers[])` settles a cohort and idle is short:

```solidity
hcBps = available × 10_000 / needed         // available < needed
```

Each cohort ticket is haircut uniformly by `hcBps` (`Vault.sol:340`). Subsequently, when a controller `claimRedeem`s and the contract balance is **still** below `claimableAssets` (e.g., due to a separate cohort competing), the per-claim payout is further capped:

```solidity
share = bal × cl.assets / claimableAssets
g     = min(cl.assets, share)
```

(`:373–376`.) Combined with the cohort `feePip` blending, this produces a fair, regime-stable settlement.

## 6. Salvage

Owner-only `salvage(token, amount, minOut, routerData)` (`Vault.sol:539`) routes a stuck non-asset token through the Swapper into `_asset` and back to the vault. The Swapper recipient invariant (proceeds → caller) ensures funds return to the vault. Reverts if `token == _asset` or `token == address(this)`.

## 7. Worth-Documenting Invariants

- `previewWithdraw(assets) ≥ shares-required` derives `fee = gross - assets` (`Vault.sol:198`) -the rounded shares and rounded fee always agree, no 1-wei drift.
- Cohort `feePip` is **shares-weighted** at enqueue and at processWithdraws merge (`:286–304`, `:343–351`); a user cannot stack claims across regimes to drag the fee down.
- `harvestPerformance` only mints when `ta × hS > hA × sup` (strict gain) AND `perfPip > 0` (`:457`).
- `_accrue` runs before every fee-relevant transition; same-block re-entry early-returns (`:425`).
