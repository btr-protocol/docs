# 7.8. BTR Token & Governance (Forward-Looking)

> **Status: design intent only.** Nothing in this page is implemented in `alm/evm/`. Token utility, distribution, emission schedule, and governance scope are **subject to change** and will be finalized through governance prior to launch. This document scaffolds *only* what aligns with primitives that already exist in the contracts.

---

## 1. What Already Exists On-Chain

The current ALM codebase has **no BTR token contract**. What exists is the surface that a future token can plug into:

| Primitive | Site | Token Hook (forward-looking) |
|-----------|------|------------------------------|
| `AccessControl.owner()` | `AccessControl.sol:14` | DAO-controlled multisig / governor / executor — owner of every chain's AC. |
| `AccessControl.treasury()` | `AccessControl.sol:23` | Recipient of all protocol fees (mgmt, perf, exit). DAO-managed treasury. |
| `AccessControl.isKeeper[]` | `AccessControl.sol:30` | Operator set; eventual rotation may be governance-curated. |
| 7-day rotation timelock | `AccessControl.sol:15` | Treasury / Swapper rotation cannot ship instantly — gives token-holders veto window. |
| 14-day Factory timelock | `AccessControl.sol:16` | Implementation rotation honours the same window. |
| 6-hour feed swap timelock | `PriceProvider.sol:23` | Feed migrations require a public window before take-effect. |

Every existing fee path already routes to `ac.treasury()`:

- Exit fee on `redeem` / `withdraw` / `claimRedeem` (`Vault.sol:243`, `:264`, `:383`).
- Management fee minted as dilution shares (`:434`).
- Performance fee minted as dilution shares (`:463`).
- Salvage proceeds returned to vault (then exit-fee on user withdrawal hits treasury).
- Swapper `ownerRescue` sweeps to treasury (`Swapper.sol:303`).

The **economic plumbing for fee-share is already complete** — only the token + claim contract are missing.

## 2. Token Utility Scope (Subject to Change)

The DAO will determine final scope. Plausible utility surfaces that align with extant primitives:

### 2.1. Treasury Fee Share

A staked BTR position could entitle holders to a share of treasury distributions sourced from the per-vault `mgmtPip`, `perfPip`, and `exitPip` flows already minted / transferred to `ac.treasury()`. This requires only a downstream distributor; no Vault / Dex changes.

### 2.2. Governance Authority

A DAO-controlled executor (Compound Governor / OZ Governor / OZ TimelockController) becomes `AccessControl.owner()` via the 2-step pull pattern (`AccessControl.sol:60`). Scope:

- Set / rotate keepers.
- Queue / execute treasury, swapper, factory rotations.
- Queue / execute risk parameter changes (`deviationBps`, `heartbeatDriftBps`).
- Add / remove / deregister adapters.
- Set vault fees within bounds + floor.
- Set / queue / remove price feeds.

The 7-day, 14-day, 6-hour, and 1-day timelocks are **enforced at the contract layer**, independent of any governor's own delay. A DAO cannot ship a swapper rotation faster than 7 days, even with a unanimous vote.

### 2.3. Adapter / Pool Curation

Future versions may surface community-curated pool whitelists. Today, `Factory.deployDex` is owner-only — an obvious DAO-delegate target.

### 2.4. Boosted / Tiered Vaults

Hypothetical: vaults with reduced exit fees for staked-BTR holders. Implementable by a wrapping contract above the Vault; **not** in-scope for the core Vault logic (per the ERC-4626 strict surface).

## 3. What Will NOT Be Token-Gated

To preempt surprise:

- **Deposit / redeem / withdraw** — remain permissionless on the Vault. Token gating sits at a wrapper layer if at all.
- **`processWithdrawals`** — permissionless settlement, by design (anyone can settle the cohort).
- **`syncAdapter`** — permissionless health restoration after outage.
- **`Swapper.settleIntent`** — permissionless intent settlement.

## 4. Cross-Chain Token Considerations

If BTR ships on multiple chains (see [Cross-chain](/docs/7.7-Cross-chain)):

- Canonical mint chain TBD (likely Ethereum mainnet; alternates: Base, Arbitrum).
- Bridging via a vetted transport (LayerZero OFT / Hyperlane / CCIP) — **not** a BTR-native bridge.
- Each chain's `AccessControl.owner()` becomes the DAO executor reachable via cross-chain governance message — same 7d/14d/1d timelocks honoured locally per chain.

## 5. Distribution / Emissions / Vesting

**Out of scope for this document.** All distribution mechanics — initial allocation, vesting schedules, emission curve, halvings, LP incentives — are not specified in the ALM contracts and will be designed and disclosed separately.

## 6. Items Requiring User / Governance Approval Before Final Doc

These cannot be authoritatively documented from code alone:

- **Final token name / ticker** — "BTR" is a working alias.
- **Treasury policy** — fee-share split between stakers, ops budget, insurance fund.
- **Governor framework** — Compound, OZ, custom.
- **Voting weight model** — token, ve-token, NFT-curated.
- **Quorum / proposal threshold** — numbers TBD.
- **Cross-chain governance topology** — single source of truth chain vs federated.

Until governance ratifies these, copy in this document remains advisory.
