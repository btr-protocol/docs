# AccessControl

> Single Source of Truth (SSoT) for owner / keeper / treasury / swapper / factory roles + isVault / isAdapter registries, with 2-step Solady ownership transfer and timelocked rotations.

---

## 1. Overview

`AccessControl.sol` is the **canonical role registry** used by every protocol contract — ALM Vaults, DEX Pools, Treasury, Distributor, Staking, Bridge. Rather than each contract maintaining its own admin set, all contracts read from this one registry.

**Key principle**: One owner, one treasury, one swapper, one factory across the entire protocol per chain. Any change to those critical addresses passes through a timelock.

---

## 2. Roles & Registries

### 2.1. Singleton Roles (one address each)

| Role | Read via | Description |
|------|----------|-------------|
| `owner()` | Solady `Ownable` | Top-level admin. 2-step transfer + 7d timelock. |
| `treasury()` | `AccessControl.treasury()` | Recipient of ∀ protocol fees (Vault mgmt/perf/exit + DEX swap/flash). |
| `swapper()` | `AccessControl.swapper()` | Whitelisted DEX router for adapter rebalances. |
| `factory()` | `AccessControl.factory()` | Vault / Pool deployer. |

### 2.2. Set Roles (membership registries)

| Registry | Read via | Description |
|----------|----------|-------------|
| `isKeeper[addr]` | `AccessControl.isKeeper(addr)` | Operators authorized for rebalance / harvest / oracle push. |
| `isVault[addr]` | `AccessControl.isVault(addr)` | Whitelist of ALM Vault contracts. |
| `isAdapter[addr]` | `AccessControl.isAdapter(addr)` | Whitelist of DEX adapter contracts. |

## 3. Ownership Transfer (Solady 2-Step Handover)

```solidity
requestOwnershipHandover()                       // newOwner: opens handover (Solady)
completeOwnershipHandover(pendingOwner)          // current owner: finalize (zero-target rejected)
```

`transferOwnership(address)` and `renounceOwnership()` are **disabled** (revert `NotAuth`) — handover is the only path. `completeOwnershipHandover` rejects `address(0)` to close the back-door renounce vector.

**Guarantees**:
- Pull-pattern: `newOwner` must explicitly request before current owner can finalize
- Hijack prevention: hostile owner cannot push ownership to dead address instantly
- Zero-target reject: closes accidental/malicious renounce-to-zero handover

## 4. Timelocked Rotations

Singleton rotations use a **queue → execute** pattern. Each has a matching `cancel*` for clean rollback before execution:

| Function | Delay | Notes |
|----------|-------|-------|
| `queueTreasury(newTreasury)` / `executeTreasury()` / `cancelTreasury()` | 7d (`ROTATION_TIMELOCK`) | Treasury rotation |
| `queueSwapper(newSwapper)` / `executeSwapper()` / `cancelSwapper()` | 7d (`ROTATION_TIMELOCK`) | Swapper rotation |
| `queueStaking(newStaking)` / `executeStaking()` / `cancelStaking()` | 7d (`ROTATION_TIMELOCK`) | Staking rotation |
| `queueFactory(newFactory)` / `executeFactory()` / `cancelFactory()` | **14d** (`FACTORY_TIMELOCK`) | Factory rotation (longer — deployer is highest-impact) |

**Bootstrap one-shots** (pre-rotation seed, callable once while slot is `address(0)`): `bootstrapFactory`, `bootstrapSwapper`, `bootstrapStaking`. Treasury is seeded via constructor.

### 4.1. Cancellation (Phase 44 R-5 fix)

Every pending queued rotation can be **cancelled by the owner** before its ETA:

```solidity
cancelTreasury()
cancelSwapper()
cancelFactory()
cancelStaking()
```

**Rationale (R-5)**: Without explicit cancel, a malicious or accidentally-queued rotation must be left to expire — leaving the protocol in an ambiguous "queued" state for the full timelock window. Cancel provides clean rollback.

## 5. Keeper Management

```solidity
setKeeper(addr, isKeeper)   // owner only, instant
```

Keepers are operational (not custody-holding). No timelock needed — owner can rotate keepers immediately for incident response. Cluster-wide daily kill budget: `KEEPER_KILL_DAILY_CAP = 2` adapter-kills / UTC day (via `chargeKeeperKill`).

## 6. Vault / Adapter Registries

```solidity
registerVault(vault)                  // factory (or owner for migration); monotonic
registerAdapter(adapter)              // factory only
deregisterAdapter(adapter)            // owner only — evict compromised adapter
```

`isVault` is **monotonic** (once true, stays true) — factory rotation does NOT orphan existing vaults' perf-fee bookkeeping. Reads:
- `Treasury.accrue` checks `isVault(msg.sender)` before honouring vault fee accrual
- `Distributor` checks `isVault` / `isAdapter` before accepting fee deposits
- `Bridge` checks `isVault` before allowing bridged Vault-share receipts

## 8. Security Considerations

- **No multi-owner**: deliberate. Multisig sits *behind* `owner()` — the Solady contract is owner-singleton.
- **Cancel auth**: only current owner can cancel a pending request (Phase 44 R-5).
- **Timelock bypass**: none. Even owner cannot skip the 7d delay; if an emergency requires faster action, the Guardian Safe pauses contracts (no rotation needed).
- **Read-only consumers**: Vaults, Pools, Distributor, Bridge all **read** from AccessControl. They never write to it. Bugs in consumers cannot corrupt the registry.
