# 4. BTR Swap -Routing

> Best-of-kind selection across enabled aggregators. Single-pass fan-out, deterministic ranking, slippage envelope applied at the upstream call.

---

## 1. Fan-Out

`fanOutQuote` (`back/services/swap/src/lib/swap.ts:117-133`) is the core routing fn:

```ts
const params = buildParams(req, mode);
const trs = await getAllTimedTr(params);          // parallel, 8s expiry
const filtered = mode === 'keeper'
  ? trs.filter(tr => validateKeeperCalldata(tr, req.fromChain).ok)
  : trs;
return filtered.map(trToRoute);
```

`getAllTimedTr` (in the SDK) issues one HTTP call per aggregator concurrently, with `Promise.allSettled` semantics -a single slow / failing aggregator never blocks the others. Each adapter is given `expiryMs = 8000` (`lib/swap.ts:84`); calls exceeding this budget are dropped from the result.

## 2. Ranking

Ranking is performed inside `getAllTimedTr` before the array is returned. Sort key:

```
netOutput = globalEstimates.output
          − globalEstimates.gasCostUsd
          − globalEstimates.feeCostUsd
```

Where `globalEstimates.output` is denominated in the receive token's USD value (computed by the adapter from the upstream price). The first element of the sorted array is the **best** route and is surfaced as `QuoteResponse.best`.

Routes that do not report `gasCostUsd` or `feeCostUsd` are treated as `0` for those terms -passive aggregators (1inch, Odos, etc.) typically only report gas.

## 3. Slippage Envelope

`maxSlippage` is forwarded to each adapter as basis points (`lib/swap.ts:81`). Default: `100` bps = 1%. Each adapter translates this into the upstream API's expected unit:

| Adapter | Upstream parameter | Unit |
|---------|--------------------|------|
| LiFi | `slippage` | decimal (e.g. `0.005` for 0.5%) |
| 1inch | `slippage` | percent (e.g. `0.5`) |
| 0x | `slippagePercentage` | decimal |
| Odos | `slippageLimitPercent` | percent |
| ParaSwap | `slippage` | bps directly |
| KyberSwap | `slippageTolerance` | bps |
| Squid / Rango / Socket / Unizen | `slippage` | bps |

The adapter is responsible for the unit conversion (`packages/core/src/<Adapter>/index.ts`).

## 4. MEV Considerations

BTR Swap is a **quote relay**, not a private-mempool router. The execution path depends on the caller:

- **User mode (front-end)**: returns standard public-mempool calldata. MEV resistance depends on the user's wallet (e.g. MetaMask + Flashbots Protect). The service does **not** opt-in to private relays on the user's behalf.
- **Keeper mode (vault rebalancer)**: the keeper bot is responsible for submission strategy. Recommended: Flashbots Protect / MEV-Share for chains where supported, or private relays per chain.
- **Intent aggregators (CowSwap, Hashflow -when enabled)**: solver / filler executes on-chain; MEV is internalised to the solver's strategy. Caller signs only the intent.

## 5. Bridge / Exchange Blacklists

`bridgeBlacklist` and `exchangeBlacklist` are forwarded to upstream aggregators that support them (LiFi, Socket). Strings are aggregator-specific (e.g. `"stargate"`, `"hop"`, `"across"`). Aggregators that don't support filtering ignore these fields.

## 6. Cross-Chain vs Monochain

Mode is implicit in `fromChain === toChain`:

- **Monochain** (`fromChain === toChain`): all enabled aggregators are eligible. Passive liquidity aggregators (1inch, Odos, etc.) typically dominate ranking on EVM majors.
- **Cross-chain** (`fromChain !== toChain`): only meta-aggregators (LiFi, Socket, Squid, Rango, Unizen) return non-empty results. Passive ones return `null` from `getTransactionRequest` and are excluded.

The caller does not need to declare cross-chain intent; it falls out of the chain ids.

## 7. Determinism Guarantees

- **No determinism in user mode.** A second `/quote` for identical inputs will yield different routes because upstream prices move. This is by design.
- **Deterministic destination in keeper mode.** Single source (LiFi) + per-chain `to` allowlist (`config.ts:45-55`) ⇒ keeper calldata always lands at the canonical LiFi Diamond. Internal route differences (which underlying DEX LiFi chooses) are still non-deterministic, but the entry point is fixed.

## 8. Latency

Each route includes `latencyMs` (the per-aggregator round-trip). Typical p50:

| Aggregator | p50 ms |
|------------|--------|
| 1inch / 0x / KyberSwap | 200–400 |
| Odos / OpenOcean / ParaSwap | 300–500 |
| LiFi (cross-chain) | 800–1500 |
| Socket | 800–2000 |
| Squid | 1000–2500 |

Numbers are observational, not contractual. The 8-second `expiryMs` is the hard cap.
