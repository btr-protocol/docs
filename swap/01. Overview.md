# 1. BTR Swap -Overview

> **BTR Swap** is the off-chain swap-aggregation surface of the BTR Protocol. It is a meta-aggregator: a single interface that fans out a swap request across a curated set of liquidity and bridge aggregators (LiFi, Socket, Squid, Rango, Unizen, 1inch, 0x, ParaSwap, Odos, KyberSwap, OpenOcean, Firebird) and returns the best route -for either monochain or cross-chain swaps.

---

## 1. Position in the Product Stack

BTR ships three complementary surfaces:

| Surface | Role | Code anchor |
|---------|------|-------------|
| **BTR Swap** | Off-chain aggregator that composes external aggregators. No on-chain logic of its own. | `swap/`, `back/services/swap/` |
| **BTR Dex (AIMM)** | BTR's own AMM -anchor-path-priced, inventory-aware multi-asset pools. | `dex/contracts/src/` |
| **BTR Supply (ALM Vaults)** | ERC-4626/7540 single-asset vaults that allocate to external CL pools and (forward-looking) AIMM. | `alm/evm/src/` |

BTR Swap is the **execution layer** consumed by both human users (front-end) and automated keepers (vault rebalancers). It does **not** introduce new on-chain code -every quote ultimately resolves to calldata produced by an underlying aggregator's smart contract.

## 2. What BTR Swap Provides

| Capability | Description |
|------------|-------------|
| **Single API** | One `POST /quote` and `POST /build` regardless of chain pair, token pair, or routing strategy. |
| **Best-of-kind selection** | Fans out across enabled aggregators in parallel; ranks by net-output (output minus gas + bridge fees). |
| **Calldata validation** | Keeper mode forces a single, deterministic source (LiFi) and validates the resulting `to` address against a per-chain allowlist. |
| **Atomic + Intent** | Supports both atomic on-chain swaps and intent-based settlement (ERC-7683 / RFQ) where the upstream aggregator exposes them. |
| **Rate-limited quoter** | Per-IP rolling 5-minute window; quote-to-execution ratio thresholds gate abusive callers (see [05. Rate Limit](05.%20Rate%20Limit.md)). |

## 3. Composed Aggregators

| Class | Aggregators (active) | Aggregators (planned) |
|-------|----------------------|-----------------------|
| **Cross-chain meta** | LiFi, Socket | Squid, Rango, Unizen (gated on API keys) |
| **Passive liquidity** | 1inch, 0x, ParaSwap, Odos, KyberSwap, OpenOcean, Firebird | -|
| **Intent / RFQ** | -| CowSwap, Hashflow, 1inch Fusion, ParaSwap Delta, Bebop, DeBridge |

Active vs disabled set is governed by `back/services/swap/src/lib/config.ts:19-35`. Disabled adapters remain compiled in `swap/packages/core/src/<Aggregator>/` -only their inclusion in `USER_MODE_AGGS` is gated.

## 5. Non-Goals

- **No on-chain settlement contracts of its own.** All execution flows through aggregator-provided calldata to aggregator-deployed contracts (e.g. LiFi Diamond `0x1231deb6...`).
- **No custody.** The service never holds funds and never signs transactions -it returns calldata for the caller to sign.
- **No price oracle.** Output estimates come from the underlying aggregator's quote response; BTR Swap does not maintain an independent price feed.
- **No order book.** Intent quotes are forwarded as-is from RFQ aggregators (where enabled).
