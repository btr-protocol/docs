# 2. BTR Swap -Architecture

> Two layers: the **SDK** (`@btr-supply/swap`) which encapsulates per-aggregator client code, and the **back service** (`@btr-protocol/swap-service`) which exposes a thin HTTP surface for keepers and the front-end.

---

## 1. Component Map

| Component | Path | Role |
|-----------|------|------|
| `@btr-supply/swap` (core SDK) | `swap/packages/core/src/` | Per-aggregator adapter modules + `getAllTimedTr` fan-out + `IBtrSwapParams` type. Zero-dep, ESM, runs in Node/Bun/Deno/browser. |
| `@btr-supply/swap-cli` | `swap/packages/cli/` | Thin CLI wrapper around the core SDK for scripting / debugging. |
| `@btr-protocol/swap-service` | `back/services/swap/src/` | Bun HTTP server exposing `/quote`, `/build`, `/status/:hash`, `/healthz`. |
| `lib/swap.ts` | `back/services/swap/src/lib/swap.ts` | Hydrates `IBtrSwapParams` from the HTTP shape, applies mode filtering, validates calldata. |
| `lib/config.ts` | `back/services/swap/src/lib/config.ts` | Mode вҶ’ aggregator-set mapping, keeper allowlist, port. |
| `lib/ratelimit.ts` | `back/services/swap/src/lib/ratelimit.ts` | Per-IP rolling-window limiter; tracks quote/exec ratio. |

## 3. HTTP Surface

Defined in `back/services/swap/src/server.ts`.

| Method | Path | Purpose |
|--------|------|---------|
| `POST` | `/quote` | Fan out across configured aggregators, return ranked routes. |
| `POST` | `/build` | Return the single best (or `preferAggId`) route with executable calldata. |
| `GET`  | `/status/:hash` | Placeholder; returns `NOT_FOUND` until per-aggregator txId mapping is wired. |
| `GET`  | `/healthz` | Liveness probe. |

### Request shape (`QuoteRequest`, `back/services/swap/src/lib/types.ts:5-25`)

```ts
{
  fromChain: number;
  toChain:   number;
  fromToken: string;        // address or "native"
  toToken:   string;
  fromAmount: string;       // wei
  fromAddress: string;      // payer
  toAddress?: string;       // receiver, defaults to fromAddress
  fromDecimals?: number;
  toDecimals?: number;
  fromSymbol?: string;
  toSymbol?: string;
  slippageBps?: number;     // default 100 = 1%
  mode?: 'user' | 'keeper';
  integrator?: string;
  referrer?: string;
  exchangeBlacklist?: string[];
  bridgeBlacklist?: string[];
}
```

### Response shape (`QuoteResponseRoute`, `types.ts:32-46`)

```ts
{
  aggId: string;            // 'LIFI' | 'ONE_INCH' | ...
  kind: 'atomic' | 'intent';
  to?: string;
  data?: string;            // 0x-prefixed calldata (omitted for intent kind)
  value?: string;
  approveTo?: string;
  output?: string | number;
  outputWei?: string;
  exchangeRate?: string | number;
  gasCostUsd?: number;
  feeCostUsd?: number;
  latencyMs?: number;
  steps?: number;
}
```

## 4. Modes

Defined in `back/services/swap/src/lib/config.ts`. `resolveMode` (`config.ts:60-70`) consults, in order: `x-btr-mode` header вҶ’ `?lifiOnly=1` query param вҶ’ `body.mode`. Default is `user`.

| Mode | Aggregator set | Calldata validation | Use case |
|------|----------------|---------------------|----------|
| `user`   | `USER_MODE_AGGS` (LiFi, Socket, 1inch, 0x, ParaSwap, Odos, KyberSwap, OpenOcean, Firebird) | None -caller is expected to inspect/sign. | Front-end users, integrators. |
| `keeper` | `KEEPER_MODE_AGGS` (LiFi only) | `validateKeeperCalldata` enforces `to` вҲҲ `KEEPER_TO_ALLOWLIST[chainId]`. | Vault rebalancer keepers. Deterministic source-of-calldata for audit. |

### Keeper calldata validator

`lib/swap.ts:88-99`. Two checks:

1. `tr.to` must be present.
2. `tr.to.toLowerCase()` must be in `KEEPER_TO_ALLOWLIST[chainId]`.

Currently the allowlist contains only the canonical LiFi Diamond address `0x1231deb6f5749ef6ce6943a275a1d3e7486f4eae` (same on all supported EVM chains, per LiFi's deployment doctrine -`config.ts:45-55`). Failures return `422 keeper calldata rejected` with the violated reason.

## 5. Build vs Quote

- `/quote` runs `getAllTimedTr` and serialises every route. The caller picks one.
- `/build` runs `getAllTimedTr` once more (no cache), filters by `preferAggId` if set, falls back to first (best by output), runs the keeper validator if `mode === 'keeper'`, and returns one `BuildResponse`.

`/build` is purposefully not a "render the previously-quoted route" endpoint -quotes are stateless. A new `getAllTimedTr` call ensures calldata is fresh against current upstream prices and the 8-second `expiryMs` (`lib/swap.ts:84`).

## 6. Token Hydration

`lib/swap.ts:53-68` injects a minimal `IToken` stub for each side. Resolution order:

1. Caller-supplied `decimals` / `symbol`.
2. Well-known registry `KNOWN_TOKENS` (USDC/USDT/WETH on chains 1, 10, 137, 8453, 42161 -`swap.ts:28-51`).
3. Fallback: `decimals = 18`, `symbol = ''`.

`address === 'native'` is normalised to `0x0000000000000000000000000000000000000000` before lookup.

## 7. Configuration

Environment:

| Var | Default | Purpose |
|-----|---------|---------|
| `SWAP_PORT` / `PORT` | `8787` | Listen port. |
| (per-aggregator API keys) | -| Read inside individual SDK adapters; absent keys disable that aggregator at runtime. |

Disabled aggregators (no API key configured) are commented out in `USER_MODE_AGGS` (`config.ts:23-27`). Adapter source remains in `swap/packages/core/src/<Aggregator>/`.
